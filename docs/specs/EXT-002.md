# EXT-002: Web Dashboard

**Status**: Implemented
**Priority**: Phase 2 - Ecosystem Expansion
**Note**: Phase 2 feature, not mapped to F-xxx IDs (F-000 to F-019 cover Phase 1 only)

## Overview

A web-based dashboard for users to manage their "Knowledge Library" on a larger screen. Provides comprehensive views of saved content, advanced filtering, bulk actions, and detailed analytics.

## Requirements

### Functional Requirements

1. **Content Library View**
   - Grid and list view options
   - Sort by: recency, platform, title, swipe date
   - Filter by: platform, status (inbox/archived), date range
   - Search by title, author, tags
   - Infinite scroll or pagination

2. **Content Detail View**
   - Full summary display
   - Original content preview (when available)
   - Swipe history
   - Tags/categories
   - Quick actions: keep, discard, delete, share

3. **Bulk Actions**
   - Select multiple items
   - Bulk keep/discard
   - Bulk delete
   - Export selected content

4. **Analytics Dashboard**
   - Swipe statistics (kept vs discarded)
   - Platform distribution
   - Daily/weekly activity
   - Reading streak

5. **Authentication**
   - Google OAuth login (same as mobile/extension)
   - Persistent session
   - Auto-logout on token expiry

6. **Settings**
   - API endpoint configuration
   - Theme (light/dark/system)
   - Default view preferences

### Non-Functional Requirements

1. **Performance**
   - Initial load < 2 seconds
   - Smooth scrolling with 100+ items
   - Responsive design (desktop first, tablet compatible)

2. **Compatibility**
   - Chrome 88+
   - Firefox 88+
   - Safari 14+
   - Edge 88+

3. **Accessibility**
   - WCAG 2.1 AA compliance
   - Keyboard navigation
   - Screen reader support

#### 3. Content Fetching

```typescript
// stores/content.ts
export const useContentStore = defineStore('content', {
  state: () => ({
    items: [] as Content[],
    filters: { status: 'inbox', platform: null } as Filters,
    sort: 'recency' as SortOption,
    page: 1,
    hasMore: true,
  }),

  getters: {
    filteredItems: (state) => {
      return state.items.filter(item => {
        if (state.filters.status && item.status !== state.filters.status) {
          return false;
        }
        if (state.filters.platform && item.platform !== state.filters.platform) {
          return false;
        }
        return true;
      });
    },
  },

  actions: {
    async fetchContent(page = 1) {
      const response = await api.getContent({
        status: this.filters.status,
        platform: this.filters.platform,
        sort: this.sort,
        page,
        limit: 20,
      });
      this.items = page === 1 ? response.items : [...this.items, ...response.items];
      this.hasMore = response.hasMore;
    },
  },
});
```

### API Endpoints Used

|----------|--------|---------|

| `/content` | GET | List all content |
| `/content/pending` | GET | Inbox content |
| `/content/kept` | GET | Archived content |
| `/content/{id}` | GET | Content detail |
| `/content/{id}` | DELETE | Delete content |
| `/swipe` | POST | Record swipe |
| `/platforms` | GET | Platform list |
| `/search` | GET | Search content |
| `/stats` | GET | User statistics |
