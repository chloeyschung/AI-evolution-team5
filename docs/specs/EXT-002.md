**Status**: Implemented | **Created**: 2026-04-14 | **Author**: abraxaspark
**F-xxx Mapping**: F-021 (Web Dashboard) | **Phase**: Phase 2 - Ecosystem Expansion | **Priority**: High

## 1. Overview

**Problem**: Users need a comprehensive, large-screen interface to manage their knowledge library with advanced filtering, bulk actions, and analytics.

**Solution**: Vue 3-based single-page application providing grid/list views, bulk operations, analytics dashboard, and full content management.

**Goals**: Desktop-first content management, advanced filtering and search, analytics visualization, bulk operations

**Non-Goals**: PWA offline support, annotations/highlighting, reading mode (MVP scope)

## 3. User Story / Behavior

As a Briefly user, I want to manage my knowledge library on a large screen, so that I can efficiently organize, filter, and analyze my saved content.

### Key Behaviors

- Dashboard shows content library with grid/list toggle
- Toolbar: search bar, filter dropdowns, sort options
- Infinite scroll or pagination for large libraries
- Click card: open detail view with full summary and metadata
- Select multiple: bulk keep/discard/delete
- Analytics tab: swipe statistics, platform distribution, activity charts
- Settings: API endpoint, theme (light/dark/system)
- Google OAuth login with persistent session
- Auto-logout on token expiry

## 5. API Design

| Endpoint | Method | Description |
|----------|--------|-------------|
| `/auth/status` | GET | Check authentication |
| `/auth/google` | POST | Google login |
| `/auth/logout` | POST | Logout |
| `/api/v1/content` | GET | List all content |
| `/api/v1/content/pending` | GET | Inbox content |
| `/api/v1/content/kept` | GET | Archived content |
| `/api/v1/content/{id}` | GET | Content detail |
| `/api/v1/content/{id}` | DELETE | Delete content |
| `/api/v1/swipe` | POST | Record swipe |
| `/api/v1/platforms` | GET | Platform list |
| `/api/v1/search` | GET | Search content |
| `/api/v1/statistics` | GET | User statistics |

## 6. Implementation

### Files

- `web-dashboard/src/main.ts` - App entry
- `web-dashboard/src/App.vue` - Root component
- `web-dashboard/src/router/index.ts` - Vue Router config
- `web-dashboard/src/stores/auth.ts` - Auth state (Pinia)
- `web-dashboard/src/stores/content.ts` - Content state (Pinia)
- `web-dashboard/src/components/content/ContentList.vue` - Content list component
- `web-dashboard/src/components/content/ContentCard.vue` - Individual card
- `web-dashboard/src/components/analytics/StatsCard.vue` - Statistics display
- `web-dashboard/src/views/Dashboard.vue` - Main dashboard view
- `web-dashboard/src/api/client.ts` - Axios instance
- `web-dashboard/src/api/endpoints.ts` - API calls

### Key Logic

1. **Vue 3 + Pinia Architecture**:
   - Stores: auth, content, settings, UI state
   - Components: composables for shared logic
   - Router: /dashboard, /inbox, /archive, /analytics, /settings

2. **Content List**:
   - Toolbar: SearchBar, FilterDropdown, SortDropdown, ViewToggle
   - Grid/List modes with responsive layout
   - Infinite scroll with intersection observer
   - Skeleton loading state

3. **Authentication**:
   - Google OAuth via popup (same flow as mobile)
   - Token stored in localStorage (or HTTP-only cookies)
   - Auto-refresh before expiry
   - Redirect to login on 401

4. **Bulk Actions**:
   - Checkbox selection on cards
   - Action bar appears on selection
   - Bulk keep/discard/delete with confirmation
   - Optimistic UI updates

5. **Analytics**:
   - Swipe statistics (kept vs discarded pie chart)
   - Platform distribution (bar chart)
   - Daily/weekly activity (line chart)
   - Reading streak display

## 7. Edge Cases

| Scenario | Handling |
|----------|----------|
| No authentication | Redirect to login, block access |
| Token expired | Auto-refresh, then logout if fails |
| Network error | Show error banner, allow retry |
| Empty content | Show "No content" state with hint |
| Large library (>1000 items) | Pagination or virtualized list |
| Browser not supported | Show compatibility message |
