# IOS-006: 홈 탭 카테고리 분류 뷰 (Home Categorization)

**Status**: Draft | **Created**: 2026-05-29 | **Author**: suyoung
**F-xxx Mapping**: Home Screen Categorization | **Phase**: Phase 4 - UX | **Priority**: Medium

> **구현 단계**
> - **Phase 1** (지금 진행 가능): UI 레이아웃 + 로컬 데이터 기반 날짜별·출처별 섹션

## 1. Overview

**Problem**: 현재 홈 탭(`SavedItemsView`)은 저장된 링크를 저장 시간 역순의 단순 리스트로만 보여준다. 콘텐츠가 쌓일수록 "무엇을 저장했는지", "어떤 주제가 많은지"를 한눈에 파악하기 어렵다.

**Solution**: 홈 탭을 3가지 분류 기준(주제별 · 날짜별 · 출처별)의 가로 스크롤 캐러셀 섹션들로 재구성한다. 각 섹션은 독립적으로 데이터를 보여주며 사용자가 저장 맥락을 빠르게 파악하도록 돕는다.

**Goals**:
- 주제별: 서버에서 사전 계산된 동적 클러스터(`GET /api/v1/topics`)를 기반으로 콘텐츠를 묶어 보여준다. 각 클러스터는 콘텐츠의 **AI 요약(summary) · 제목(title) · 키워드(keywords)** 3가지를 조합해 서버가 생성한다. 동적 클러스터가 없으면 `auto_tag_category` 고정 카테고리로 폴백한다
- 날짜별: 오늘 / 이번 주 / 지난주 / 이번 달 등 시간 구간별로 콘텐츠를 묶어 잊혀진 콘텐츠를 재발견하게 한다
- 출처별: 도메인(YouTube, LinkedIn, Medium 등) 기준으로 콘텐츠를 묶어 보여준다
- 각 섹션은 가로 스크롤 캐러셀(`ScrollView(.horizontal)`)로 제공한다

**Non-Goals**:
- 주제 섹션의 동적 클러스터링(예: "최근 바이오 AI 트렌드" 스타일 자동 제목 생성) — IOS-008로 분리
- 앱에서 저장한 콘텐츠의 iOS 자체 auto-tag 처리 — IOS-007로 분리
- 섹션 순서 사용자 직접 설정 — 향후 개선
- 무한 스크롤 또는 섹션별 "더 보기" 상세 화면 — 향후 개선

## 3. User Story / Behavior

**As a** Briefly 사용자,
**I want to** 홈 탭에서 저장한 콘텐츠를 주제 · 날짜 · 출처 별로 묶어서 보고 싶다,
**so that** 무엇을 저장했는지 한눈에 파악하고, 오래전에 저장해 잊혀진 글을 재발견할 수 있다.

### Key Behaviors

```
[홈 탭 진입]
  → HomeViewModel.load() 호출
  → ① 로컬 StorageService에서 즉시 items 로드 (.deleted 제외) → 날짜별·출처별 섹션 즉시 렌더링
  → ② 섹션 순서를 랜덤 셔플 (앱 실행마다 [주제별, 날짜별, 출처별] 순서 변경)
  → ③ 로그인 상태면 GET /api/v1/content 비동기 fetch (discard 제외)
       → 응답 받으면 serverItems 업데이트 → 주제별 섹션 렌더링

[섹션: 주제별 — 로그인 O]
  GET /api/v1/topics fetch → TopicCluster 목록 수신 (IOS-008 사전 계산 결과)
  → 클러스터 있음:
    각 클러스터를 독립 섹션으로 표시
    섹션 제목 = 클러스터 한국어 제목 (예: "AI가 바꾸는 헬스케어")
    서브타이틀 = 대표 키워드 chips (예: "AI · 바이오테크 · 임상시험")
    카드 목록 = content_ids에 해당하는 serverItems (HomeItem으로 매핑)
  → 클러스터 없음 또는 fetch 실패 (폴백):
    serverItems를 auto_tag_category로 그룹핑
    섹션 제목 = 카테고리 한국어 레이블 (Tech → "기술", Business → "비즈니스" 등)
    서브타이틀 = 해당 카테고리의 top keywords

[섹션: 날짜별]
  allItems(로컬 + 서버 병합, 중복 serverContentId 제거)를 savedAt 기준 groupBy
  구간: 오늘 / 이번 주 / 지난주 / 이번 달 / 지난달 / 더 오래됨
  비어있는 구간은 숨김

[섹션: 출처별]
  allItems를 normalizedDomain 기준 groupBy
  정렬: 항목 수 내림차순
  상위 N개 출처만 섹션으로 표시 (예: 최대 8개)

[카드 탭]
  → UIApplication.shared.open(item.url)
  → 로컬 아이템이면 StorageService.updateItem(status: .read)
  → 서버 아이템이면 swipe API (선택적 — IOS-006 범위 외)
```

## 4. Data Models

### Existing Used

- `SavedItem` — 로컬 저장 모델 (`StorageService`)
- `AuthTokenStore` — 로그인 토큰
- `BrieflyAPI` — API 클라이언트

### New (iOS-only)

#### `ServerContent` (새 Codable 모델)

서버 `ContentResponse`를 디코딩하는 iOS 전용 모델.

```swift
struct ServerContent: Codable, Identifiable {
    let id: Int
    let url: URL
    let title: String?
    let platform: String           // 서버가 분류한 출처 (예: "youtube.com")
    let thumbnailURL: URL?
    let summary: String?
    let createdAt: Date            // ISO8601
    let autoTagCategory: String?   // "Tech", "Business", ...
    let autoTagKeywordsEn: [String]
    let autoTagKeywordsOriginal: [String]

    // 표시용 도메인
    var normalizedDomain: String { ... }
}
```

#### `HomeItem` (통합 표현용)

로컬 `SavedItem`과 서버 `ServerContent`를 동일하게 다루기 위한 래퍼.

```swift
enum HomeItem: Identifiable {
    case local(SavedItem)
    case server(ServerContent)

    var id: String { ... }
    var url: URL { ... }
    var displayTitle: String { ... }
    var thumbnailURL: URL? { ... }
    var normalizedDomain: String { ... }
    var savedAt: Date { ... }
    var autoTagCategory: String? { ... }  // IOS-007 sync 후 채워짐; 폴백 모드(고정 카테고리 그룹핑)에서 사용
    var autoTagKeywords: [String] { ... }
}
```

#### `HomeDateBucket` (날짜 구간)

```swift
enum HomeDateBucket: String, CaseIterable {
    case today = "오늘 저장한 따끈따끈한 글"
    case thisWeek = "이번주에 발견한 좋은 글"
    case lastWeek = "놓치지 마세요! 지난주에 담아둔"
    case thisMonth = "이번 달의 링크들"
    case lastMonth = "한달전의 내가 저장한 글"
    case older = "오랜만에 꺼내 읽어 볼까요?"
}
```

### 서버 응답 (기존 ContentResponse)

| 필드 | 타입 | 설명 |
|------|------|------|

| `title` | String? | 제목 |
| `platform` | String | 출처 도메인 |
| `thumbnail_url` | String? | 썸네일 |
| `summary` | String? | AI 요약 |
| `created_at` | String (ISO8601) | 저장 시각 |
| `auto_tag_category` | String? | "Tech", "Business" 등 8개 중 하나 |

### HomeViewModel 핵심 설계

```swift
/// 단일 섹션 — 날짜 구간 / 출처 도메인 / 주제 섹션 모두 동일 타입
struct HomeCardSection: Identifiable {
    enum Kind {
        case date(HomeDateBucket)           // 날짜별
        case source(String)                 // 출처별 (domain)
        case topic(TopicCluster)            // 주제별: 동적 클러스터 (IOS-008, 기본)
        case topicCategory(String)          // 주제별: 고정 카테고리 폴백 (auto_tag_category)
        case topicPlaceholder               // 주제별: Phase 1 로딩 플레이스홀더
    }
    let id: String
    let title: String
    let subtitle: String?                   // 키워드 chips (선택 — 주제별 섹션에서 사용)
    let icon: String                        // 📅 / 🌐 / 📚
    let items: [HomeItem]
    let kind: Kind
}

@MainActor
final class HomeViewModel: ObservableObject {
    @Published var sections: [HomeCardSection] = []   // 모든 섹션 (랜덤 셔플 포함)
    @Published var isLoadingServer = false

    func load() {
        loadLocal()           // 즉시 로컬 반영 → sections 생성 후 shuffled()
        Task { await loadServer() }
    }
}
```

### 폴백: 고정 카테고리 레이블 매핑

동적 클러스터(`GET /api/v1/topics`)가 없거나 fetch에 실패했을 때, `auto_tag_category` 기반 고정 카테고리 그룹핑으로 폴백한다. 이 경우 아래 레이블 표를 사용한다.

| 서버 값 | 표시 레이블 |
|---------|------------|
| Tech | 기술 |
| Business | 비즈니스 |
| Essays | 에세이 |
| Research | 연구 |
| Lifestyle | 라이프스타일 |
| News | 뉴스 |
| Culture | 문화 |
| Other | 기타 |

### 날짜 구간 판별 로직 (기준: 로컬 기기 시간)

```
오늘: savedAt >= 오늘 0시
이번 주: savedAt >= 이번 주 월요일 0시 (오늘 제외)
지난주: savedAt >= 지난 주 월요일 ~ 이번 주 월요일
이번 달: savedAt >= 이번 달 1일 (지난주 이전 제외)
지난달: savedAt >= 지난달 1일 ~ 이번 달 1일
더 오래됨: 나머지
```

## 7. Edge Cases

| Scenario | Handling |
|----------|----------|
| 비로그인 상태 | 주제별 섹션에 안내 문구 표시. 날짜별·출처별은 로컬 데이터로 표시 |
| `apply-auto-tag` 미머지 (auto_tag_category == nil) | 주제별 섹션 미표시 또는 "분류 준비 중" 빈 상태 |
| 로컬 items와 서버 items 중복 | `serverContentId`가 같은 항목은 서버 버전 우선 (더 많은 정보 포함) |
| 서버 fetch 실패 | 에러 로그 후 로컬 데이터만 표시. 주제별 섹션은 숨김 |
| 콘텐츠 0개 | 기존 빈 상태 UI ("아직 저장된 링크가 없어요") |
| 특정 섹션 항목 0개 | 해당 구간/카테고리/출처 섹션 자체를 숨김 |
| 썸네일 없음 | 도메인 이니셜 또는 placeholder 이미지 표시 |
| 매우 긴 제목 | `.lineLimit(2)` 적용 후 말줄임 |

## 9. Testing

### Phase 1 수동 검증 시나리오

1. **레이아웃 확인**
   - 홈 탭 진입 → 3개 섹션이 세로 스크롤로 나열 확인
   - 각 섹션 가로 스크롤 동작 확인
   - 주제별 섹션에 "주제 분석 중" 플레이스홀더 표시 확인

2. **섹션 랜덤 순서**
   - 앱 강제 종료 후 재실행 반복 → 섹션 순서가 매번 다르게 표시 확인

3. **날짜별 — 여러 날짜에 걸쳐 저장**
   - 오늘/지난주/지난달 저장 링크가 각 구간 섹션에 정확히 배치 확인
   - 오늘 항목 없으면 "오늘" 구간 미노출 확인

4. **출처별**
   - YouTube, Medium, LinkedIn URL 저장
   - 홈 탭 → 각 도메인별 카드 확인
   - `www.medium.com` → "medium.com" 정규화 확인

5. **deleted 필터링**
   - Delete 처리한 링크가 모든 섹션에 노출되지 않음 확인

6. **카드 탭**
   - 카드 탭 → Safari 열림 + 읽음 처리 확인

7. **빈 상태**
   - 로컬 항목 없음 → 빈 상태 UI 표시 확인

### Phase 2 수동 검증 시나리오

8. **주제별 — 로그인 O, apply-auto-tag 머지 후**
   - 다양한 카테고리의 링크 저장
   - 홈 탭 진입 → 주제별 섹션에 "기술", "비즈니스" 등 카드 확인

9. **주제별 — 비로그인**
   - 로그아웃 상태에서 홈 탭 → 주제별 섹션에 로그인 안내 문구 확인

10. **서버 데이터 병합**
    - 서버 fetch 후 날짜별·출처별 섹션에 서버 콘텐츠가 추가 노출 확인
    - 로컬과 서버 중복 항목이 한 번만 노출 확인

## 10. Acceptance Criteria

### Phase 1
- [ ] 홈 탭에 주제별 · 날짜별 · 출처별 3개 섹션이 세로 스크롤로 표시된다
- [ ] 각 섹션은 가로 스크롤 캐러셀로 카드를 보여준다
- [ ] 앱 실행 시마다 3개 섹션의 순서가 랜덤하게 바뀐다
- [ ] 주제별 섹션은 "주제 분석 중" 플레이스홀더를 표시한다
- [ ] 날짜별 섹션이 로컬 데이터로 정확한 구간별 그룹핑을 표시한다
- [ ] 출처별 섹션이 로컬 데이터로 도메인별 그룹핑을 표시한다
- [ ] `status == .deleted` 콘텐츠는 모든 섹션에 노출되지 않는다
- [ ] 항목이 없는 구간/출처는 노출되지 않는다
- [ ] 카드 탭 시 해당 URL이 Safari로 열린다

### Phase 2
- [ ] 주제별 섹션이 `GET /api/v1/topics` 동적 클러스터를 기반으로 콘텐츠 카드를 표시한다
- [ ] 동적 클러스터가 없을 때 `auto_tag_category` 고정 카테고리 그룹핑으로 폴백한다
- [ ] 비로그인 시 주제별 섹션에 로그인 안내 문구가 표시된다
- [ ] 날짜별·출처별 섹션에 서버 콘텐츠가 병합 표시된다
- [ ] discard 처리된 서버 콘텐츠는 모든 섹션에 노출되지 않는다
- [ ] 서버 fetch가 로컬 렌더링을 블로킹하지 않는다
