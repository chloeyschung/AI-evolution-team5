# IOS-008: 홈 탭 동적 주제 클러스터링 (Dynamic Topic Clustering)

**Status**: Draft | **Created**: 2026-05-29 | **Author**: suyoung
**F-xxx Mapping**: Home Dynamic Topic Clusters | **Phase**: Phase 5 - Personalization | **Priority**: Low

---

## 1. Overview

**Problem**: IOS-006의 홈 탭 주제별 섹션은 `auto_tag_category`(Tech, Business 등 고정 8개) 기준 그룹핑을 폴백으로 사용한다. 그러나 고정 카테고리만으로는 "최근 바이오 AI 트렌드", "요즘 읽은 스타트업 이야기"처럼 사용자의 실제 관심사 흐름을 반영하는 의미있는 주제를 만들 수 없다. 또한 기존 방식이 `auto_tag_keywords_en`(영어 키워드, 최대 3개)만 입력으로 사용해 콘텐츠의 맥락과 의미를 충분히 반영하지 못한다.

**Solution**: 서버에서 주 2회(매주 월·목 자정) 스케줄 작업을 실행해, 사용자별 콘텐츠의 **AI 요약(summary) · 제목(title) · 키워드(auto_tag_keywords_en)** 3가지를 조합한 입력으로 유사 콘텐츠를 클러스터링하고, LLM으로 한국어 클러스터 제목을 생성한다. iOS 앱은 이 사전 계산된 클러스터를 `GET /api/v1/topics`로 fetch해 주제별 섹션에 표시한다.

**Goals**:
- 콘텐츠의 **AI 요약 · 제목 · 키워드** 3가지를 입력으로 사용해 의미 기반 동적 주제 클러스터 생성 (주 2회 갱신)
- 클러스터 제목은 LLM이 한국어로 생성 — 담백하되 흥미를 이끄는 톤 (예: "AI가 바꾸는 헬스케어")
- iOS 홈 탭 주제별 섹션에서 고정 카테고리 대신 동적 클러스터를 우선 표시 (클러스터 없을 때 폴백)
- 날짜별 · 출처별 섹션명은 하드코딩된 규칙으로 일관성 있게 표시

**Non-Goals**:
- 실시간/즉시 클러스터 재계산 — 스케줄 기반 배치 처리
- 사용자가 클러스터를 직접 수정하거나 이름 변경
- 클러스터 결과의 웹 대시보드 노출 — iOS 전용 (우선)

## 3. User Story / Behavior

**As a** Briefly 사용자,
**I want to** 홈 탭에서 내가 저장한 콘텐츠가 "최근 바이오 AI 트렌드"처럼 의미있는 주제로 묶여 보이길 원한다,
**so that** 내 관심사 흐름을 직관적으로 파악할 수 있다.

### Key Behaviors

```
[서버 스케줄 작업 — 매주 월·목 00:00 KST]
  → 모든 활성 유저 순회
  → 유저별: summary 또는 auto_tag_keywords_en이 있는 콘텐츠 조회
  → 항목별 입력 텍스트 구성:
    format: "{title}. {summary[:300]}. Keywords: {', '.join(keywords_en)}"
    (summary 없으면 title + keywords, keywords도 없으면 title만 사용)
  → 입력 텍스트 기반 TF-IDF 벡터화 후 코사인 유사도 클러스터링
  → 클러스터별 대표 키워드 추출
  → LLM 호출: 대표 키워드 + 콘텐츠 수 → 한국어 클러스터 제목 생성
  → user_topic_clusters 테이블 upsert (user_id, title, content_ids, keywords)
  → 완료 로그 기록

[iOS 홈 탭 진입]
  → HomeViewModel.load()
  → GET /api/v1/topics fetch (로그인 필요)
  → 응답의 클러스터 목록을 주제별 섹션에 표시
  → 클러스터 없거나 실패 → IOS-006 고정 카테고리 폴백

[첫 방문 / 클러스터 미생성 시 (FR-11)]
  → GET /api/v1/topics → {"clusters": []} 즉시 응답
  → 서버가 BackgroundTask로 해당 유저의 클러스터링 즉시 시작 (비동기)
  → iOS는 폴백(고정 카테고리) 표시
  → 클러스터 생성 완료 (약 30~60초) 후 pull-to-refresh 시 동적 클러스터 표시

[클러스터 카드]
  섹션 제목: 클러스터 한국어 제목 (예: "최근 바이오 AI 트렌드")
  섹션 서브타이틀: 대표 키워드 chips (예: "AI · 바이오테크 · 임상시험")
  카드 목록: 해당 클러스터에 속한 콘텐츠 캐러셀
```

## 4. Data Models

### 새 서버 테이블: `user_topic_clusters`

| 컬럼 | 타입 | 설명 |
|------|------|------|
| `id` | Int PK | |
| `user_id` | Int FK | |
| `title_ko` | String | LLM 생성 한국어 클러스터 제목 |
| `keywords_en` | JSON (list[str]) | 대표 키워드 (최대 5개) |

```json
      "content_ids": [101, 205, 312],
      "generated_at": "2026-05-26T15:00:00Z"
    }
  ],
  "generated_at": "2026-05-26T15:00:00Z"
}
```

```swift
struct TopicCluster: Codable, Identifiable {
```

### 클러스터링 알고리즘 (서버)

```
1. 유저의 콘텐츠에서 항목별 입력 텍스트 구성
   format: "{title}. {summary[:300]}. Keywords: {', '.join(keywords_en)}"
   - summary 없으면: "{title}. Keywords: {', '.join(keywords_en)}"
   - keywords도 없으면: "{title}"

2. 입력 텍스트를 TF-IDF로 벡터화한 후 코사인 유사도 계산
   (또는 키워드 co-occurrence 행렬 방식도 가능)

3. 유사도 임계값 이상인 콘텐츠를 동일 클러스터로 묶음

4. 클러스터 크기 < 2 → 제외

5. 클러스터별 상위 키워드 3~5개 추출

6. LLM 호출 → 섹션명 생성 (아래 네이밍 설계 참고)
```

### 섹션 네이밍 설계

#### 주제별 — LLM 생성 (동적)

**톤 가이드**: 담백하되 흥미를 이끄는 한국어. 클릭베이트 지양, 키워드를 자연스럽게 녹인 짧은 문장형.

**LLM 프롬프트 설계**:
```
당신은 콘텐츠 큐레이터입니다.
아래 키워드들로 묶인 콘텐츠 그룹의 섹션 제목을 한국어로 지어주세요.

규칙:
- 15자 이내
- 담백하고 자연스러운 문장 (과장·감탄사 금지)
- 키워드를 직접 나열하지 말고 주제를 함축
- 예시 스타일: "AI가 바꾸는 헬스케어", "조용히 뜨는 기술들", "스타트업의 요즘 고민"

키워드: {keywords_en}
콘텐츠 수: {count}개
JSON으로만 응답: {"title": "..."}
```

**예시**:

| 키워드 | 생성된 섹션명 |
|--------|--------------|
| biotech, AI, clinical trials | AI가 바꾸는 헬스케어 |
| startup, venture capital, fundraising | 스타트업의 요즘 고민 |
| machine learning, GPU, inference | 조용히 뜨는 기술들 |
| productivity, remote work, tools | 일하는 방식의 변화 |
| climate, energy, sustainability | 기후와 에너지 이야기 |

**LLM 실패 시 fallback**: 상위 키워드 2개를 `·`로 연결 (예: "AI · 바이오테크")

### 스케줄 설정

- **실행 시각**: 매주 월요일 · 목요일 00:00 KST (= UTC 15:00 전일)
- **cron 표현식**: `0 15 * * 0,3` (UTC 기준) — 일(0)·수(3) → KST 월·목 00:00
- **구현 방법**: APScheduler, Celery Beat, 또는 Modal cron (서버 인프라에 맞게 선택)

## 7. Edge Cases

| Scenario | Handling |
|----------|----------|
| 유저 콘텐츠가 적어 클러스터 생성 불가 | 빈 clusters 반환 → 폴백 |
| 클러스터링 작업 실패 (특정 유저) | 에러 로그 후 해당 유저 skip, 기존 클러스터 유지 |
| LLM 제목 생성 실패 | 대표 키워드를 join해 fallback 제목 사용 (예: "AI · 바이오테크") |
| iOS fetch 실패 | IOS-006 고정 카테고리 그룹핑으로 폴백 |
| 클러스터 생성 후 콘텐츠 삭제됨 | content_ids 중 deleted 항목은 iOS에서 필터링 |
| 비로그인 상태 | fetch 스킵 → IOS-006 고정 카테고리 폴백 |
