# IOS-015: Dive Deeper — 콘텐츠 심화 질문 (Deep Reflection Questions)

**Status**: Draft | **Created**: 2026-06-10 | **Author**: suyoung
**F-xxx Mapping**: F-007 (Dive Deeper / Reflection) | **Phase**: Phase 5 - Engagement | **Priority**: Medium

---

## 1. Overview

**Problem**: 사용자가 콘텐츠를 저장하고 AI 요약을 읽은 뒤, 더 깊이 생각하거나 적용할 만한 포인트를 발견하기 어렵다. 웹 앱에는 이미 "Dive Deeper" 섹션이 있어 LLM이 생성한 3개의 비판적 사고 질문을 제공하지만, iOS 앱에는 해당 기능이 없다.

**Solution**: `ItemDetailView`의 AI 요약 아래에 **Dive Deeper 섹션**을 추가한다. 항목이 서버와 동기화된 경우(`serverContentId` 보유) `GET /api/v1/content/{id}/reflection-questions` 엔드포인트를 호출해 LLM 생성 질문 3개를 표시한다.

**Goals**:
- iOS 앱과 웹 앱 모두 "Dive Deeper" 용어로 통일 (IOS-009 §2 결정 반영)
- 콘텐츠 세부 화면에서 비판적 사고를 유도하는 질문 3개 표시
- 질문 로드 중 스켈레톤 UI로 자연스러운 로딩 경험 제공
- 질문 캐싱은 서버에서 처리 — 두 번째 로드는 즉시 반환

**Non-Goals**:
- 사용자가 질문에 직접 답변하는 기능 (메모 기능과 별개)
- 질문 생성 실패 시 재시도 버튼 — 실패 시 섹션 숨김
- 오프라인 캐시 — 서버 응답이 없으면 표시 안 함
- 질문 언어 강제 한국어화 — LLM이 콘텐츠 언어에 맞게 생성

## 3. User Story / Behavior

**As a** Briefly 사용자,
**I want to** 기사를 읽고 나서 "이 주제에 대해 어떻게 생각하지?"라는 질문을 받고 싶다,
**so that** 단순 소비가 아니라 내 관점을 형성하는 독서 습관을 기를 수 있다.

### Key Behaviors

```
[ItemDetailView 진입 — serverContentId 있음]
  → AI 요약 섹션 아래에 "Dive Deeper" 섹션이 스켈레톤과 함께 나타남
  → GET /api/v1/content/{serverContentId}/reflection-questions 호출
  → 성공: 스켈레톤 → 질문 3개로 교체
  → 실패 / 빈 배열: 섹션 전체 사라짐

[ItemDetailView 진입 — serverContentId 없음 (서버 미동기화)]
  → Dive Deeper 섹션 표시 안 함

[재진입 (뒤로 갔다가 다시 들어옴)]
  → 이전에 로드된 질문이 있으면 즉시 표시 (ViewModel이 캐시)
  → 없으면 다시 API 호출
```

### 화면 레이아웃 (ItemDetailView 세로 순서)

```
┌──────────────────────────────────┐
│  썸네일 / 헤더 (제목, 도메인, 날짜) │
│                                  │
│  ─── AI 요약 ───────────────────  │
│  (요약 텍스트)                    │
│                                  │
│  ─── Dive Deeper ──────────────  │ ← 신규
│  더 깊이 생각해보기                │
│                                  │
│  • (질문 1)                       │
│  • (질문 2)                       │
│  • (질문 3)                       │
│                                  │
│  [원문 보기]  [Discard]  [Keep]   │
└──────────────────────────────────┘
```

## 4. Data Models

### iOS: `DiveDeeper` 응답 모델

```swift
struct DiveDeeperResponse: Codable {
    let contentId: Int
    let questions: [String]

    enum CodingKeys: String, CodingKey {
        case contentId = "content_id"
        case questions
    }
}
```

### 서버 API (기존 엔드포인트, 변경 없음)

```
GET /api/v1/content/{content_id}/reflection-questions
Authorization: Bearer {access_token}

Response 200:
{
  "content_id": 42,
  "questions": [
    "What second-order effects might this change trigger?",
    "Whose perspective is absent from this account?",
    "How could this apply to your own context?"
  ]
}

Response 200 (AI 생성 실패 시):
{
  "content_id": 42,
  "questions": []
}
```

> 서버는 생성된 질문을 `content.reflection_questions` 컬럼에 캐시한다.
> 두 번째 호출부터는 DB 캐시를 반환하므로 LLM 호출 없이 즉시 응답한다.

## 6. Implementation

### 변경 파일 목록

| 파일 | 변경 내용 |
|------|-----------|
| `Briefly/Services/BrieflyAPI.swift` | `fetchDiveDeeperQuestions(contentId:token:)` 메서드 추가 |
| `Briefly/Views/ItemDetailView.swift` | Dive Deeper 섹션 추가 (AI 요약 아래) |
| `Briefly/ViewModels/ItemDetailViewModel.swift` | `diveDeeperQuestions: [String]`, `isDiveDeeperLoading: Bool` 상태 추가; 진입 시 fetch 트리거 |

> 별도 파일 생성 없이 기존 `ItemDetailViewModel`(또는 동등한 상태 관리)에 통합한다.

### BrieflyAPI 구현 개요

```swift
func fetchDiveDeeperQuestions(contentId: Int, token: String) async throws -> [String] {
    let response: DiveDeeperResponse = try await get(
        "/content/\(contentId)/reflection-questions",
        token: token
    )
    return response.questions
}
```

### ItemDetailView UI 구조

```swift
// AI 요약 섹션 아래에 삽입
if let serverContentId = item.serverContentId {
    DiveDeeperSectionView(
        isLoading: viewModel.isDiveDeeperLoading,
        questions: viewModel.diveDeeperQuestions
    )
    .task {
        await viewModel.loadDiveDeeperIfNeeded(serverContentId: serverContentId)
    }
}
```

### DiveDeeperSectionView 구조

```
DiveDeeperSectionView
├── 헤더: "Dive Deeper" (brieflyTitle2 / semibold) + "더 깊이 생각해보기" (brieflyCaption / secondary)
└── 조건 분기:
    ├── isLoading == true  → SkeletonLine × 3 (shimmer animation)
    ├── questions.isEmpty  → EmptyView (섹션 전체 숨김, isLoading이 false인 경우)
    └── else               → ForEach(questions) { VStack { bullet + Text(question) } }
```

### ViewModel 로직

```swift
// 이미 로드됐거나 serverContentId 없으면 스킵
func loadDiveDeeperIfNeeded(serverContentId: Int) async {
    guard diveDeeperQuestions.isEmpty,
          let token = AuthTokenStore.shared.accessToken else { return }
    isDiveDeeperLoading = true
    defer { isDiveDeeperLoading = false }
    let questions = (try? await BrieflyAPI.shared.fetchDiveDeeperQuestions(
        contentId: serverContentId, token: token
    )) ?? []
    diveDeeperQuestions = questions
}
```

## 7. Edge Cases

| Scenario | Handling |
|----------|----------|
| `serverContentId` 없음 (서버 미동기화 항목) | 섹션 전체 숨김 |
| API 응답 `questions: []` | 섹션 전체 숨김 (로딩 완료 후 사라짐) |
| 네트워크 실패 / 401 | `try?` → empty array → 섹션 숨김 |
| LLM cold-start (~50s) | 로딩 스켈레톤이 표시된 상태로 최대 55s 대기 후 실패 시 숨김 |
| 뒤로 갔다가 재진입 | ViewModel에 캐시된 질문이 있으면 즉시 표시, API 재호출 없음 |
| 항목에 summary도 keywords도 없음 | 서버가 빈 배열 반환 → 섹션 숨김 |

## 8. Design Notes

- **헤더 스타일**: "Dive Deeper" 는 `brieflyTitle2` (또는 `.title3.bold`), "더 깊이 생각해보기"는 `.caption.foregroundColor(.secondary)` 로 표시
- **질문 스타일**: `brieflyBody` 폰트, leading bullet `•`, 줄 사이 padding 12pt
- **섹션 구분**: 기존 AI 요약 섹션과 동일한 카드/배경 스타일로 통일
- **다크모드**: 기존 디자인 시스템 토큰 사용으로 자동 대응

## 9. Testing

### 수동 검증 시나리오

1. **정상 케이스**
   - 서버 동기화된 항목(serverContentId 있음) 진입 → Dive Deeper 섹션 로딩 스켈레톤 표시 확인
   - 1~60초 내 질문 3개 표시 확인 (서버 캐시 유무에 따라 속도 차이)
   - 다시 진입 → 즉시 질문 표시 (재호출 없음)

2. **숨김 케이스**
   - 방금 저장된 새 항목 (서버 미동기화) 진입 → Dive Deeper 섹션 없음 확인
   - IOS-007 서버 동기화 후 재진입 → 섹션 표시 확인

3. **실패 케이스**
   - 백엔드 오프라인 상태에서 진입 → 로딩 스켈레톤 잠시 후 섹션 사라짐 확인
