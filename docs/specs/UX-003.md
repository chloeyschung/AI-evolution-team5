# UX-003: Summary Detail View

## User Story

As a Briefly user, I want to view expanded details about content I've swiped, so that I can access the original source or review metadata when I need more context beyond the 3-line summary.

## Implementation Contract

### Level 1: Natural Language

Detail View is an on-demand expansion interface that displays full information about a specific content item. Users access it from the card stack or from kept/discarded collections.

**Content States:**
- **INBOX:** New/unread content (not yet archived)
- **ARCHIVED:** Content marked as "읽었어요" (read)

The view displays:
- Full content URL with clickable link to source
- Complete metadata (platform, content type, author, timestamp)
- The 3-line AI-generated summary (read-only reference)
- Swipe action history (if applicable: when it was kept/discarded)
- **Content status** (INBOX or ARCHIVED)
- **"읽었어요" button** with state-based behavior:
  - **INBOX entry:** Button active, tap changes status to ARCHIVED
  - **ARCHIVED entry:** Button dimmed with label "이미 읽었어요!" (already read)

The system must:
- Fetch content details by ID from storage
- Display metadata in a structured, readable format
- Provide direct navigation to original source content
- Handle missing metadata gracefully (not all sources provide all fields)
- Support status transition: INBOX → ARCHIVED via "읽어요" button
- **Webapp restriction:** New content save disabled, but status changes allowed

### Level 2: Logic Flow (Pseudocode)

```
INPUT: content_id from card tap or collection item tap, entry_source (inbox_list | archive_list)
PROCESS:
  1. Fetch Content object by ID from storage.
  2. If content not found: show "Content not found" error.
  3. If content exists:
     a. Display full URL as clickable link.
     b. Display platform with icon/badge.
     c. Display content type (article/video/social_post/image).
     d. Display author and timestamp if available.
     e. Display 3-line summary as reference.
     f. Display content status (INBOX or ARCHIVED).
     g. If swipe history exists: display action and timestamp.
     h. Determine button state based on content.status:
        - If status == INBOX:
          - Button active
          - Label: "읽었어요" (Mark as read)
          - On tap: Call update_status(content_id, ARCHIVED)
        - If status == ARCHIVED:
          - Button dimmed/disabled
          - Label: "이미 읽었어요!" (Already read!)
  4. On URL tap: open external browser/link handler.
  5. On "읽었어요" button tap (INBOX only):
     a. Update content status to ARCHIVED.
     b. Refresh view to show disabled button state.
     c. Update list view (if applicable).

OUTPUT: Rendered detail view with all available metadata and status-aware button
ERROR: NotFoundError (content not found), NetworkError (fetch failure), ValidationError (invalid status transition)
```

from dataclasses import dataclass
from typing import Optional
from datetime import datetime
from enum import Enum

class ContentStatus(str, Enum):
    """Content consumption status."""
    INBOX = "inbox"      # New/unread content
    ARCHIVED = "archived"  # Read/archived content

@dataclass
class ContentDetail:
    """Full detail representation of a content item."""
    content_id: int
    url: str
    summary: str
    platform: str
    content_type: str
    title: Optional[str]
    author: Optional[str]
    published_at: Optional[datetime]
    ingested_at: datetime
    status: ContentStatus  # INBOX or ARCHIVED
    swiped_at: Optional[datetime]
    swipe_action: Optional[str]  # "keep" or "discard"

class DetailView:
    """Detail view component for content inspection."""
    
    async def load_detail(self, content_id: int, entry_source: str) -> ContentDetail:
        """
        Fetch full content detail by ID.
        
        Args:
            content_id: The content ID to fetch.
            entry_source: Where user entered from (inbox_list, archive_list, etc.)
            
        Returns:
            ContentDetail object with all available information.
            
        Raises:
            NotFoundError: If content does not exist.
        """
        pass
    
    def render_detail(self, detail: ContentDetail) -> None:
        """
        Render the detail view with all metadata and status-aware UI.
        
        Args:
            detail: ContentDetail object to display.
            - Button state: active if INBOX, dimmed if ARCHIVED
            - Button label: "읽었어요" if INBOX, "이미 읽었어요!" if ARCHIVED
        """
        pass
    
    async def mark_as_read(self, content_id: int) -> ContentDetail:
        """
        Mark content as read (INBOX → ARCHIVED transition).
        
        Args:
            content_id: The content ID to mark as read.
            
        Returns:
            Updated ContentDetail with new status.
            
        Raises:
            NotFoundError: If content does not exist.
            ValidationError: If content is already ARCHIVED.
        """
        pass
    
    def on_url_tap(self, url: str) -> None:
        """
        Handle URL tap to open external source.
        
        Args:
            url: The URL to open.
        """
        pass
    
    def on_not_found(self) -> None:
        """Handle content not found error state."""
        pass

## API Specification

### GET /content/{content_id}

Fetch full detail for a specific content item.

**Request:**
```
GET /content/{content_id}
```

**Response (200 OK):**
```json
{
  "content_id": 1,
  "url": "https://example.com/article/123",
  "summary": "3-line AI-generated summary here...",
  "metadata": {
    "platform": "Medium",
    "content_type": "article",
    "title": "Article Title",
    "author": "John Doe",
    "published_at": "2024-01-15T10:30:00Z"
  },
  "status": "inbox",  // or "archived"
  "ingested_at": "2024-01-15T12:00:00Z",
  "swipe_history": {
    "action": "keep",
    "swiped_at": "2024-01-15T14:00:00Z"
  }
}
```

**Response (404 Not Found):**
```json
{
  "detail": "Content not found"
}
```

### PATCH /content/{content_id}/status

Mark content as read (INBOX → ARCHIVED transition).

**Request:**
```
PATCH /content/{content_id}/status
```

**Body:**
```json
{
  "status": "archived"
}
```

**Response (200 OK):**
```json
{
  "content_id": 1,
  "status": "archived",
  "updated_at": "2024-01-15T15:00:00Z"
}
```

**Response (400 Bad Request):**
```json
{
  "detail": "Content is already archived"
}
```

**Response (404 Not Found):**
```json
{
  "detail": "Content not found"
}
```

## Implementation Notes

- **Backend Only:** This spec defines the backend API. Frontend implementation (React Native component) is out of scope for backend-focused development.
- **Metadata Completeness:** Not all content sources provide all metadata fields. The API should return `null` for missing optional fields rather than omitting them, allowing the frontend to handle display consistently.
- **Swipe History:** The `swipe_history` field is `null` if the content has not been swiped yet (still in pending state).
- **URL Security:** Consider validating URL scheme (http/https only) before allowing external navigation to prevent malicious redirects.
- **Content Status:** The `status` field requires DAT-001 model update. Default status for new content should be `INBOX`.
- **Status Transition:** Only INBOX → ARCHIVED transition is allowed (one-way). ARCHIVED → INBOX requires separate feature.
- **Webapp Mode:** API does not enforce webapp restrictions; this is a frontend concern. Webapp should block new content save but allow status changes.
- **Button State:** Frontend determines button label and state based on `status` field:
  - INBOX: Label="읽어요", State=active
  - ARCHIVED: Label="이미 읽었어요!", State=disabled
