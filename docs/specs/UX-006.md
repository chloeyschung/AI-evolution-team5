# UX-006: Delete Content

**Status**: Implemented | **Created**: 2026-04-14 | **Author**: abraxaspark
**F-xxx Mapping**: F-019 (Delete Content) | **Phase**: Phase 1 - MVP (Mobile Focus) | **Priority**: Medium

---

## 1. Overview

**Problem**: Users need to permanently remove content they no longer want to keep in their library, without the friction of multi-step confirmations.

**Solution**: 1-step confirmation popup for individual content deletion with cascade removal of associated data.

**Goals**: Simple content management, irreversible deletion, data integrity (associated data cleanup)

**Non-Goals**: Trash/recycle bin, undo delete, batch delete, restore deleted content

---

## 2. Requirements

| ID | Requirement | Priority |

| FR-4 | Delete associated swipe history | P0 |
| FR-5 | Action is irreversible (no undo) | P0 |
| FR-6 | User-scoped deletion (can only delete own content) | P0 |
| NFR-1 | Deletion completes within 500ms | P0 |

---

## 3. User Story / Behavior

As a Briefly user, I want to delete content I no longer want, so that my library stays relevant and manageable.

### Key Behaviors

- Delete action available from content card or detail view
- 1-step confirmation: "Delete this content?" with Cancel/Delete options
- On confirm: content removed from all views immediately
- Associated swipe history also deleted
- Brief "Deleted" feedback shown
- No undo option, no trash bin
- Content permanently deleted from database

## 6. Implementation

### Files

- `src/api/routes.py` - DELETE /content/{id} endpoint
- `src/data/repository.py` - ContentRepository.delete() with cascade

### Key Logic

1. **Deletion Order**:
   - Delete SwipeHistory records first (foreign key constraint)
   - Delete Content record
   - Use transaction for atomicity

2. **User Scoping**:
   - Verify content.user_id == current_user.id
   - Raise 404 if content doesn't belong to user (security)
   - Require authentication (Depends(get_current_user))

3. **Cascade Delete**:
   - SQLAlchemy: ondelete="CASCADE" on SwipeHistory.content_id FK
   - Or: explicit delete SwipeHistory before Content

4. **Client-Side Behavior**:
   - User initiates delete (swipe left on card, tap delete button)
   - Show confirmation dialog: "Delete this content?"
   - Options: "Cancel" or "Delete"
   - On confirm: call DELETE API
   - Remove content from UI
   - Show brief "Deleted" toast/snackbar

### Dependencies

- UX-001 - Content exists in library

**Provides** (for future):

## 7. Edge Cases

| Scenario | Handling |
|----------|----------|
| Content not found | 404 error |
| Content belongs to other user | 404 error (security: don't reveal existence) |
| No authentication | 401 error |
| Swipe history exists | Cascade delete removes it |
| Delete from detail view | Navigate back after delete |
| Delete from list view | Remove from list, show next item |
