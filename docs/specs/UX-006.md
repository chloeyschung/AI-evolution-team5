# UX-006: Delete Content

**F-019 Mapping**: Delete Content
**Phase**: Phase 1 - MVP (Mobile Focus)
**Priority**: Medium - User content management

## Overview

Permanently delete individual saved content. 1-step confirmation popup. Irreversible.

## Requirements

### 1. Delete Individual Content
- User can delete any saved content (INBOX or ARCHIVED)
- Deletion removes content from all views
- Deletion removes associated swipe history

### 2. 1-Step Confirmation Popup
- Simple confirmation: "Delete this content?"
- Options: "Cancel" or "Delete"
- No multi-step confirmation (unlike account delete)

### 3. Irreversible Action
- No undo option
- No trash/recycle bin
- Content permanently deleted

### 4. Associated Data Deletion
- Delete content record
- Delete all swipe history for that content
- Do NOT affect user statistics (already calculated)

## API Design

### DELETE /api/v1/content/{content_id}

Permanently delete content and associated swipe history.

**Request:**
```
Authorization: Bearer <access_token>
```

**Response (200 OK):**
```json
{
  "message": "Content deleted successfully"
}
```

**Response (404 Not Found):**
```json
{
  "error": "content_not_found"
}
```

**Response (401 Unauthorized):**
```json
{
  "error": "unauthorized"
}
```

## Implementation Details

### Deletion Order

1. Delete swipe history for content (foreign key constraint)
2. Delete content record

### Client-Side Behavior

1. User initiates delete (e.g., swipe left on content card, tap delete button)
2. Show confirmation popup: "Delete this content?"
3. If user confirms:
   - Call DELETE /content/{id}
   - Remove content from UI
   - Show brief "Deleted" feedback

## Dependencies

- **Depends on**: UX-001 (content exists), AUTH-001 (authentication)
- **Required by**: None

## Testing

1. Delete content → content removed from all views
2. Delete content → swipe history also removed
3. Delete non-existent content → 404
4. Delete without auth → 401
5. Verify content cannot be accessed after deletion
