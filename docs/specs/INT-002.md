# INT-002: LinkedIn/Social Sync

**Status**: Implemented | **Created**: 2026-04-14 | **Author**: abraxaspark
**F-xxx Mapping**: F-023 (LinkedIn/Social Auto-Sync) | **Phase**: Phase 3 - Platform Integrations | **Priority**: Medium

---

## 1. Overview

**Problem**: Users manually save LinkedIn posts and other social media content, missing the convenience of automatic ingestion from their saved items.

**Solution**: LinkedIn API integration with OAuth 2.0, allowing users to connect their account and automatically sync saved posts.

**Goals**: Hands-free content ingestion from social platforms, AI summarization of saved posts, unified content library

**Non-Goals**: Multiple social platforms (MVP is LinkedIn only), post comments sync, real-time sync

## 3. User Story / Behavior

As a Briefly user, I want to automatically sync my LinkedIn saved posts, so that articles and content I've bookmarked are ingested and summarized without manual effort.

### Key Behaviors

- Connect LinkedIn: OAuth 2.0 flow, grant read-only permission
- Saved posts: automatically fetched from LinkedIn "Saved Items"
- Manual sync: trigger immediately via API
- Auto sync: background task runs at configured frequency (Phase 2)
- New posts: fetched, metadata extracted, AI summary generated
- Duplicates: skipped if URL already exists
- Sync log: view history of sync operations with counts/errors
- Post types: article, text post, video, image carousel handled

## 5. API Design

| Endpoint | Method | Description |
|----------|--------|-------------|
| `/integrations/linkedin/status` | GET | Get connection status |
| `/integrations/linkedin/disconnect` | POST | Disconnect LinkedIn account |
| `/integrations/linkedin/sync/configs` | GET | Get sync configurations |
| `/integrations/linkedin/sync/configs` | POST | Create/update sync config |

| `/integrations/linkedin/sync/logs` | GET | Get sync logs |

## 6. Implementation

### Files

- `src/integrations/linkedin/client.py` - LinkedIn API client with OAuth
- `src/integrations/linkedin/sync.py` - Sync logic and orchestration
- `src/integrations/linkedin/models.py` - Pydantic models for LinkedIn data
- `src/integrations/repositories/integration.py` - Integration state repository (shared with INT-001)

### Key Logic

1. **LinkedIn API Client**:
   - Base URL: https://api.linkedin.com/v2
   - OAuth 2.0 bearer token in Authorization header
   - Headers: LinkedIn-Version, X-Restli-Protocol-Version
   - Methods: get_saved_items(), get_post_details(urn)

2. **Sync Service**:
   - Fetch saved items from /me/savedItems endpoint
   - For each item: get post details by URN
   - Filter new posts since last_sync_at
   - For each post:
     - Extract URL from post data
     - Check if URL exists (skip if duplicate)
     - Create Content record with platform='LinkedIn'
     - Determine content_type (article, social_post, video, image)
     - Trigger AI summarization (async)
   - Update last_sync_at timestamp
   - Log sync result

3. **Token Management**:
   - Reuse IntegrationTokens model from INT-001
   - provider='linkedin' for LinkedIn tokens
   - Same encryption and refresh logic

4. **Background Sync** (Phase 2):
   - Reuse Celery task infrastructure from INT-001
   - linkedin_sync_task(user_id)
   - Same APScheduler for due sync checks

### Dependencies

**Requires** (satisfied by):
- AUTH-002 - OAuth infrastructure (token storage, refresh)
- ING-002 - Content ingestion (metadata extraction, upsert)
- AI-001 - Summarization (post summary generation)
- INT-001 - Shared integration models and repository

**Provides** (for future):

## 7. Edge Cases

| Scenario | Handling |
|----------|----------|
| OAuth token expired | Auto-refresh, retry operation |
| API rate limit exceeded | Queue sync, retry after quota reset |
| Post deleted by author | Skip, log warning |
| Post type not supported | Skip, log error type |
| Duplicate URL | Skip, don't re-ingest |
| Network error during sync | Retry with exponential backoff |
| LinkedIn API changes | Update client, handle new response format |
