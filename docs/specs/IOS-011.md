# IOS-011: YouTube URL iOS 로딩 최적화

**Status**: Implemented | **Created**: 2026-06-04 | **Author**: suyoung
**F-xxx Mapping**: F-004 (Content Detail View) | **Phase**: Phase 3 - Bug Fix | **Priority**: High

---

## 1. Overview

**Problem**: YouTube URL을 iOS 앱에서 열면 세 가지 문제가 발생한다.

1. `FetchCoordinator`가 YouTube에 대해 WKWebView 스크래핑을 시도해 10~20초 지연이 발생한다.
2. OG 이미지/제목이 완성된 뒤에도 본문 스크래핑이 끝나야 화면에 반영된다.
3. `ItemDetailView`가 `FetchCoordinator` 업데이트를 실시간으로 반영하지 못해 화면이 굳어 있다.

**Solution**: YouTube URL 감지 → WKWebView 스킵 + oEmbed API로 빠른 메타데이터 취득 + OG 완료 즉시 중간 저장 + ItemDetailView 반응형(`liveItem`) 업데이트.

## 2. 구현 상세

### 2-1. FetchCoordinator — YouTube WKWebView 스킵 (commit `4511637`)

**파일**: `Briefly/Briefly/Services/FetchCoordinator.swift`

YouTube URL에서 2단계(본문 스크래핑)를 건너뛰고 `ogDescription`을 즉시 `articleText`로 사용한다.

```swift
// YouTube는 스크래핑이 불가하므로 ogDescription을 즉시 사용
if Self.isYouTubeURL(item.url) {
    updated.articleText = updated.ogDescription
    updated.fetchStatus = .partial
} else {
    let articleText = try await fetchArticleText(for: item.url)
    updated.articleText = articleText ?? updated.ogDescription
    updated.fetchStatus = (articleText != nil) ? .done : .partial
}

private static func isYouTubeURL(_ url: URL) -> Bool {
    guard let host = url.host?.lowercased() else { return false }
    return host == "youtube.com" || host.hasSuffix(".youtube.com") || host == "youtu.be"
}
```

### 2-2. MetadataService — YouTube oEmbed API (commit `4831265`)

**파일**: `Briefly/Briefly/Services/MetadataService.swift`

전체 HTML 파싱 대신 YouTube oEmbed JSON API로 title / author / thumbnail 을 빠르게 취득한다.

```swift
func fetchYouTubeMetadata(for url: URL) async throws -> PageMetadata {
    let encoded = url.absoluteString.addingPercentEncoding(withAllowedCharacters: .urlQueryAllowed) ?? ""
    guard let oembedURL = URL(string: "https://www.youtube.com/oembed?url=\(encoded)&format=json") else {
        throw URLError(.badURL)
    }
    let (data, _) = try await session.data(from: oembedURL)
    guard let json = try? JSONSerialization.jsonObject(with: data) as? [String: Any] else {
        throw URLError(.cannotParseResponse)
    }
    var meta = PageMetadata()
    meta.ogTitle = json["title"] as? String
    meta.ogDescription = json["author_name"] as? String   // 채널명을 설명으로 활용
    meta.siteName = "YouTube"
    if let thumbURL = json["thumbnail_url"] as? String {
        meta.ogImageURL = URL(string: thumbURL)
    }
    return meta
}
```

`FetchCoordinator`에서 YouTube URL이면 `fetchYouTubeMetadata`를 우선 시도하고, 실패 시 기존 `fetchMetadata`로 fallback한다.

### 2-3. FetchCoordinator — OG 완료 즉시 중간 저장 (commit `4831265`)

OG 메타데이터가 완성되는 즉시 `StorageService.updateItem()` + `fetchCoordinatorDidUpdate` 알림을 발행해 이미지/제목이 본문 로딩 완료 전에도 화면에 표시되도록 한다.

```swift
// OG 완료 즉시 저장 + 알림 — 이미지가 바로 표시되도록
StorageService.shared.updateItem(updated)
NotificationCenter.default.post(name: .fetchCoordinatorDidUpdate, object: nil)
// 이후 본문 스텝 진행…
```

### 2-4. ItemDetailView — liveItem 반응형 업데이트 (commit `3c78af1`)

**파일**: `Briefly/Briefly/Views/ItemDetailView.swift`

`@State var liveItem`을 추가하고 `fetchCoordinatorDidUpdate` 알림 수신 시 `StorageService`에서 최신 항목을 로드해 뷰를 갱신한다.

```swift
@State private var liveItem: SavedItem? = nil

var currentItem: SavedItem { liveItem ?? items[currentIndex] }

.onChange(of: currentIndex) { _ in
    liveItem = nil
}
.onReceive(NotificationCenter.default.publisher(for: .fetchCoordinatorDidUpdate)) { _ in
    let baseItem = items[currentIndex]
    if let updated = StorageService.shared.loadAll().first(where: { $0.id == baseItem.id }) {
        liveItem = updated
    }
}
```

## 5. 알려진 한계

| 항목 | 내용 |
|------|------|
| `articleText` 품질 | YouTube는 `ogDescription`(채널명 수준)이 `articleText`가 되므로 내용이 빈약. 실제 영상 내용은 백엔드 transcript(`ING-003`)에서 처리 |

| ItemDetailView frozen 문제 | `liveItem`으로 부분 해소. `items` 배열 자체가 갱신되지 않아 페이지 전환 후 재진입 시 초기값 고착 가능성 잔존 |
