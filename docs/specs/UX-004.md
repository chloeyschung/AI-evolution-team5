# UX-004: Filter by Platform

**F-013 Mapping**: Filter by Platform
**Phase**: Phase 1 - MVP (Mobile Focus)
**Priority**: Medium - Content discovery

## Overview

Filter saved content by source platform (YouTube, LinkedIn, etc.). Platform list dynamically generated from user's save history.

## Requirements

### 1. Dynamic Platform List
- Platform list generated from user's saved content
- Only shows platforms user has actually saved
- Platform names as displayed (e.g., "YouTube", not "youtube")

### 2. Filter Content by Platform
- Filter pending content by platform
- Filter kept content by platform
- Filter discarded content by platform

### 3. Platform Statistics (Optional)
- Show count of content per platform
- Helps user understand their save distribution

## API Design

### GET /api/v1/platforms

Get list of platforms user has saved content from.

**Request:**
```
Authorization: Bearer <access_token>
```

**Response (200 OK):**
```json
[
  {
    "platform": "YouTube",
    "count": 42
  },
  {
    "platform": "LinkedIn",
    "count": 15
  }
]
```

### GET /api/v1/content/pending?platform=YouTube

Get pending content filtered by platform.

**Request:**
```
Authorization: Bearer <access_token>
Query: platform=YouTube, limit=50
```

**Response (200 OK):**
```json
[
  {
    "id": 1,
    "platform": "YouTube",
    "content_type": "video",
    "url": "...",
    "title": "...",
    ...
  }
]
```

### GET /api/v1/content/kept?platform=YouTube

Get kept content filtered by platform.

### GET /api/v1/content/discarded?platform=YouTube

Get discarded content filtered by platform.

## Implementation Details

### Platform List Generation

1. Query distinct platforms from user's content
2. Count content per platform
3. Return sorted by count (descending)

### Platform Filter

1. Add optional `platform` query parameter to existing endpoints
2. Filter content by platform if parameter provided
3. Case-insensitive platform matching

## Dependencies

- **Depends on**: UX-001 (content exists)
- **Required by**: None

## Testing

1. Get platforms → returns platforms with counts
2. Filter by platform → returns only matching content
3. Filter by non-existent platform → returns empty list
4. Platform list is dynamic (updates as user saves)
