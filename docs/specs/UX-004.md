# UX-004: Filter by Platform

**Status**: Implemented | **Created**: 2026-04-14 | **Author**: abraxaspark
**F-xxx Mapping**: F-013 (Filter by Platform) | **Phase**: Phase 1 - MVP (Mobile Focus) | **Priority**: Medium

---

## 1. Overview

**Problem**: Users need to focus on content from specific sources (e.g., only YouTube videos, only LinkedIn articles) when their library grows large.

**Solution**: Dynamic platform filter that shows only platforms user has saved, with content filtering across all views (pending, kept, discarded).

**Goals**: Content discovery, source-focused consumption, dynamic platform list

**Non-Goals**: Platform blocking/muting, platform-specific settings, custom platform grouping

## 3. User Story / Behavior

As a Briefly user, I want to filter my saved content by source platform, so that I can focus on content from specific sources when I want to.

### Key Behaviors

- Platform list generated dynamically from user's content
- Each platform shows count of saved content
- Tapping platform filters all views (pending, kept, discarded)
- Filter persists across navigation within app session
- "All" option shows content from all platforms
- Platform list sorted by count (descending)

## 6. Implementation

### Files

- `src/api/routes.py` - Platform list endpoint, enhanced content endpoints with platform filter
- `src/api/schemas.py` - PlatformCountResponse schema
- `src/data/repository.py` - ContentRepository.get_platforms(), filter by platform

### Key Logic

1. **Platform List Generation**:
   - Query: SELECT platform, COUNT(*) FROM Content GROUP BY platform
   - Filter: user_id = current_user
   - Order: COUNT(*) DESC (most saved first)
   - Return: platform name and count

2. **Platform Filter**:
   - Add optional `platform` query parameter to existing endpoints
   - Filter: WHERE platform = :platform (case-insensitive)
   - If no platform param: return all platforms
   - Case-insensitive matching: LOWER(platform) = LOWER(:platform)

3. **Endpoint Enhancements**:
   - /content/pending: add platform filter
   - /content/kept: add platform filter
   - /content/discarded: add platform filter
   - Maintain existing pagination (limit/offset)

## 7. Edge Cases

| Scenario | Handling |
|----------|----------|
| No content saved | Platform list is empty |
| Filter by non-existent platform | Returns empty content list |
| Single platform | Shows that platform with count |
| Platform name case variation | Case-insensitive matching |
| New content from new platform | Platform list updates on next fetch |
