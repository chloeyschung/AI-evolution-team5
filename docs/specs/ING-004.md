# ING-004: LinkedIn 콘텐츠 처리 개선 (oEmbed API 기반)

**Status**: Implemented | **Created**: 2026-06-04 | **Author**: suyoung
**F-xxx Mapping**: F-005 (AI Summary) | **Phase**: Phase 3 - Bug Fix | **Priority**: High

---

## 1. Overview

**Problem**: LinkedIn URL을 공유하면 두 가지 문제가 발생한다.

1. **AI 요약 오작동**: 백엔드가 LinkedIn URL에 인증 없는 HTTP 요청을 보내 로그인 리다이렉트 페이지 HTML을 스크래핑한다. AI는 이 페이지를 요약해 "Sign in to LinkedIn account", "Business page creation offered" 같은 boilerplate를 반환한다.
2. **iOS 메타데이터 없음**: `FetchCoordinator`가 LinkedIn OG 태그를 인증 없이 요청하면 LinkedIn이 로그인 페이지로 리다이렉트하거나 차단한다. 결과적으로 제목은 "www.linkedin.com"(도메인만), 이미지는 빈 placeholder로 표시된다.

**Root Cause**: LinkedIn은 인증 없는 요청에 대해 실제 콘텐츠를 제공하지 않는다. 쿠키가 Safari에 있어도 Share Extension / 백엔드 HTTP 요청에는 전달되지 않는다.

**Solution**: LinkedIn URL 감지 시 `https://www.linkedin.com/oembed?url=...` API로 title / author / thumbnail을 취득한다. 백엔드는 oEmbed 결과를 AI 요약 입력으로 사용하고, 실패 시 요약을 건너뛴다(로그인 boilerplate 요약 방지).

## 2. 문제 진단

### 현재 흐름

```
POST /share (LinkedIn URL)
→ _background_summarize()
→ content_extractor.fetch_html_and_text(linkedin.com/...)
→ LinkedIn이 로그인 페이지 HTML 반환 (302 redirect → login)
→ AI.summarize(로그인 페이지 텍스트) → "Sign in to LinkedIn..." 저장
```

```
iOS FetchCoordinator
→ MetadataService.fetchMetadata(linkedin.com/...)
→ LinkedIn이 로그인 페이지 반환 → og:title 없음 → 제목 = 도메인
→ fetchArticleText() → WKWebView도 차단 → articleText 없음
```

### 수정 후 흐름

```
POST /share (LinkedIn URL)
→ _background_summarize()
→ is_linkedin_url(url) 감지
→ fetch_linkedin_text(url) → LinkedIn oEmbed API
  ├─ [성공] "title\nauthor" 텍스트 반환 → AI 요약
  └─ [실패] None 반환 → 요약 건너뜀 (boilerplate 방지)
```

```
iOS FetchCoordinator
→ isLinkedInURL(url) 감지
→ MetadataService.fetchLinkedInMetadata() → oEmbed API
  ├─ [성공] title + thumbnail 표시 (OG 완료 즉시 저장)
  └─ [실패] 기존 fetchMetadata() fallback
→ WKWebView 스크래핑 건너뜀 → fetchStatus = .partial
```

## 4. 구현

### 4-1. 백엔드 — `src/ingestion/linkedin_extractor.py` (신규)

```python
import re
from urllib.parse import urlparse
import httpx

_LINKEDIN_HOST_RE = re.compile(r"(?:^|\.)linkedin\.com$", re.IGNORECASE)
_OEMBED_URL = "https://www.linkedin.com/oembed?url={url}&format=json"

def is_linkedin_url(url: str) -> bool:
    host = urlparse(url).hostname or ""
    return bool(_LINKEDIN_HOST_RE.search(host))

async def fetch_linkedin_text(url: str) -> str | None:
    """LinkedIn oEmbed API로 title + author를 취득합니다. 실패 시 None."""
    try:
        async with httpx.AsyncClient(timeout=8.0) as client:
            r = await client.get(_OEMBED_URL.format(url=url))
            r.raise_for_status()
            data = r.json()
            title = data.get("title", "")
            author = data.get("author_name", "")
            return f"{title}\n{author}".strip() or None
    except Exception:
        return None
```

### 4-2. 백엔드 — `src/api/routers/content.py` (`_background_summarize` 수정)

```python
from src.ingestion.linkedin_extractor import is_linkedin_url, fetch_linkedin_text

# 기존 YouTube 분기 아래에 LinkedIn 분기 추가
if page_text:
    text_content = page_text
elif extract_video_id(url):          # YouTube
    ...
elif is_linkedin_url(url):           # LinkedIn
    linkedin_text = await fetch_linkedin_text(url)
    text_content = linkedin_text or ""   # None → 요약 건너뜀
else:

```

### 4-3. iOS — `MetadataService.swift` (`fetchLinkedInMetadata()` 추가)

YouTube oEmbed과 동일한 패턴으로 LinkedIn oEmbed API 호출.

```swift
func fetchLinkedInMetadata(for url: URL) async throws -> PageMetadata {
    let encoded = url.absoluteString
        .addingPercentEncoding(withAllowedCharacters: .urlQueryAllowed) ?? ""
    guard let oembedURL = URL(string: "https://www.linkedin.com/oembed?url=\(encoded)&format=json") else {
        throw URLError(.badURL)
    }
    let (data, _) = try await session.data(from: oembedURL)
    guard let json = try? JSONSerialization.jsonObject(with: data) as? [String: Any] else {
        throw URLError(.cannotParseResponse)
    }
    var meta = PageMetadata()
    meta.ogTitle = json["title"] as? String
    meta.ogDescription = json["author_name"] as? String
    meta.siteName = "LinkedIn"
    if let thumbURL = json["thumbnail_url"] as? String {
        meta.ogImageURL = URL(string: thumbURL)
    }
    return meta
}
```

### 4-4. iOS — `FetchCoordinator.swift` (LinkedIn 분기 추가)

- 1단계 메타데이터: `isLinkedInURL` 감지 → `fetchLinkedInMetadata()` 시도, 실패 시 `fetchMetadata()` fallback
- 2단계 본문: LinkedIn → WKWebView 건너뜀, `ogDescription`(author명) → `articleText`, `fetchStatus = .partial`
