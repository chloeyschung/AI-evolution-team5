# ADV-001: Personalized Trend Feed

**Status**: Spec | **Created**: 2026-04-14 | **Author**: abraxaspark

---

## 1. Overview

**Problem**: Users have accumulated content in Archive but lack a way to rediscover relevant content based on interests and consumption patterns.

**Solution**: Personalized "Trend Feed" that ranks archived content using:
- Swipe history (kept vs. discarded patterns)
- Interest tags (user-defined) + Content tags (AI-generated)
- Recency and engagement signals

**Non-Goals**: Community trending, external trend integration, external content recommendations (Phase 3+)

## 3. Algorithm

### Relevance Score Formula

```
relevance_score = 0.35×interest_match + 0.30×tag_similarity + 0.20×recency + 0.15×engagement
```

| Component | Formula | Neutral |
|-----------|---------|---------|
| **Interest Match** | `matched_interests / total_user_interests` | 0.5 (no interests) |
| **Tag Similarity** | `Jaccard(content_tags, preferred_tags)` | 0.5 (no preferred tags) |
| **Recency** | `1 / (1 + days_since_kept / 30)` | N/A |
| **Engagement** | `keep_ratio_for_same_tags` | 0.5 (no data) |

**Threshold**: `relevance_score >= 0.1` to appear in feed.

## 5. API

**Endpoint**: `GET /content/trend-feed`

| Parameter | Type | Default | Description |
|-----------|------|---------|-------------|
| `limit` | int | 20 (1-50) | Max items |
| `offset` | int | 0 | Pagination |
| `time_range` | string | "all" | week/month/all |
| `min_score` | float | 0.1 | Min relevance threshold |

**Response**:
```python
TrendFeedResponse = {
    items: List[TrendFeedItem],  # {content, relevance_score, matched_interests, top_tags}
    total: int,
    has_more: bool
}
```

## 7. Edge Cases

| Scenario | Handling |
|----------|----------|
| No archived content | Empty list + "Keep some content first" |
| < 10 archived items | Return all by recency (no scoring) |
| No interest tags | interest_match = 0.5 |
| Content has no tags | tag_similarity = 0.5 |
| All scores below threshold | Empty list |
