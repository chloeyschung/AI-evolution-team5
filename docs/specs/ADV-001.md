## 1. Overview

**Problem**: Users have accumulated content in Archive but lack a way to rediscover relevant content based on interests and consumption patterns.

**Solution**: Personalized "Trend Feed" that ranks archived content using swipe history, interest tag matching, recency, and engagement signals.

**Goals**: Content rediscovery, personalized ranking, increased engagement with archived content

**Non-Goals**: Community trending, external trend integration, external content recommendations (Phase 3+)

## 3. User Story / Behavior

As a Briefly user with a large archive of kept content, I want to see a personalized feed of relevant content, so that I can rediscover articles that match my current interests.

### Key Behaviors

- Trend feed shows archived content ranked by relevance
- Relevance score combines: interest match, tag similarity, recency, engagement
- Time filter: week/month/all time ranges
- Pagination for large archives
- Empty state: "Keep some content first" if no archived items
- Minimum 10 items needed for scoring (below: simple recency sort)

## 5. API Design

| Endpoint | Method | Description |
|----------|--------|-------------|
| `/api/v1/content/trend-feed` | GET | Get personalized trend feed |

### Request/Response Examples

**GET /api/v1/content/trend-feed?limit=20&offset=0&time_range=all&min_score=0.1**

Query Parameters:
- `limit`: 20 (default, 1-1000, hard limit 1000)
- `offset`: 0 (default)
- `time_range`: "all" (default), "week", "month"
- `min_score`: 0.1 (default minimum relevance threshold)

Response (200 OK):
```json
{
  "items": [
    {
      "content": {
        "id": 1,
        "platform": "Medium",
        "content_type": "article",
        "url": "https://medium.com/...",
        "title": "Article Title",
        "summary": "...",
        "kept_at": "2024-01-10T12:00:00Z"
      },
      "relevance_score": 0.85,
      "matched_interests": ["technology", "programming"],
      "top_tags": ["python", "web-development"]
    }
  ],
  "total": 150,
  "has_more": true
}
```

## 6. Implementation

### Files

- `src/ai/trend_analyzer.py` - Relevance scoring algorithm
- `src/data/repository.py` - Query methods for trend feed
- `src/api/schemas.py` - TrendFeedResponse, TrendFeedItem schemas
- `src/api/routes.py` - GET /content/trend-feed endpoint
- `src/constants.py` - TREND_FEED_MAX_LIMIT, TREND_FEED_SAMPLE_SIZE, scoring weights

### Key Logic

1. **Relevance Score Formula**:
   ```
   relevance_score = 0.35×interest_match + 0.30×tag_similarity + 0.20×recency + 0.15×engagement
   ```

2. **Score Components**:
   - **Interest Match**: matched_interests / total_user_interests (0.5 if no interests)
   - **Tag Similarity**: Jaccard(content_tags, preferred_tags) (0.5 if no preferred tags)
   - **Recency**: 1 / (1 + days_since_kept / 30)
   - **Engagement**: keep_ratio_for_same_tags (0.5 if no data)

3. **Threshold**: relevance_score >= 0.1 to appear in feed

4. **Query Optimization**:
   - Cache preferred tags (recalculate on significant swipe history change)
   - Batch queries: get_tags_for_content_ids() to avoid N+1
   - Index ContentTag.tag and InterestTag.tag
   - Hard limit: 1000 items max to prevent memory issues
   - Timezone handling via convert_to_utc() from datetime_utils

5. **Edge Case Handling**:
   - < 10 items: return all by recency (no scoring)
   - No interests: interest_match = 0.5
   - No tags: tag_similarity = 0.5
   - All scores below threshold: empty list

## 7. Edge Cases

| Scenario | Handling |
|----------|----------|
| No archived content | Empty list + "Keep some content first" message |
| < 10 archived items | Return all by recency (no scoring) |
| No interest tags | interest_match = 0.5 (neutral) |
| Content has no tags | tag_similarity = 0.5 (neutral) |
| All scores below threshold | Empty list |
| Large archive (>1000 items) | Hard limit at 1000, pagination required |
