# ING-002: URL Extraction & Cleaning

## User Story
As a Briefly user, I want the system to extract the main text content from a shared URL, so that it can be summarized accurately without noise like ads, scripts, or navigation menus.

## Implementation Contract

### Level 1: Natural Language
The `ContentExtractor` service takes a URL as input, fetches the HTML content asynchronously, and uses a parsing engine to extract the primary textual content. It must strip away "noise" elements such as `<script>`, `<style>`, `<nav>`, `<footer>`, `<header>`, and advertisements, returning a clean, normalized string of text ready for summarization.

### Level 2: Logic Flow (Pseudocode)
```
INPUT: url (string)
PROCESS:
  1. Validate URL format.
  2. Fetch HTML content from the URL using an asynchronous HTTP client.
  3. Parse the HTML using a robust parsing library (e.g., BeautifulSoup).
  4. Remove non-content tags: <script>, <style>, <nav>, <footer>, <header>, <aside>, <form>.
  5. Identify the main content container (e.g., <article>, <main>, or the <body> if no specific container is found).
  6. Extract text from the selected container.
  7. Clean the text: remove excessive whitespace, handle broken characters, and normalize line breaks.
OUTPUT: clean_text (string)
ERROR: ExtractionError (if URL is unreachable, invalid, or no meaningful text is found)
```

## Verification Contract

### Level 1: Scenarios (Natural Language)
- **Success:** A valid blog post URL returns a clean string of the article text.
- **Failure:** An invalid/malformed URL raises an `ExtractionError`.
- **Failure:** A URL that returns a 404 or 500 error raises an `ExtractionError`.
- **Edge:** A page that is almost entirely JavaScript/empty (e.g., a login page) returns an error or indicates no content was found.
- **Edge:** A page with extremely large HTML content is handled without crashing (timeout/limit).

### Level 2: Given-When-Then
```
GIVEN a valid URL to a news article
WHEN the extract_text method is called
THEN the output is a string containing the article text without HTML tags or noise

GIVEN a URL that does not exist (404)
WHEN the extract_text method is called
THEN an ExtractionError is raised
```
