## 1. 개요

**문제**: 웹 대시보드, 브라우저 확장 전반에 걸쳐 인라인 SVG로 임시 제작된 로고가 사용되고 있었으며, 브랜드 일관성이 없고 공식 로고와 달랐음.

**해결**: 공식 로고 에셋(`logo_full.svg`, `logo_short.svg`)을 기반으로 재사용 가능한 React 컴포넌트를 만들고, 노출 맥락(가로형/정방형)에 따라 올바른 로고를 적용.

**목표**:
- 전 플랫폼 브랜드 아이덴티티 통일
- 가로형(wordmark) / 정방형(lettermark) 사용 기준 정립
- 배경색에 따라 흰색·카키색 전환 가능한 구조 도입

**비목표**: iOS 앱 아이콘 교체(xcassets 미구성으로 별도 작업 필요), 브라우저 확장 PNG 래스터 파일 교체.

## 2. 로고 사용 기준

| 맥락 | 로고 종류 | 배경 | 로고 색 |
|------|-----------|------|---------|
| 웹 로그인 페이지 (SignIn) | `LogoFull` (가로형 워드마크) | 카키 (`#3A4229`) | 흰색 |
| 웹 사이드바 뱃지 (NavRail) | `LogoShort` (B 레터마크) | 카키 (`#3A4229`) | 흰색 |
| 웹 회원가입 페이지 (SignUp) | `LogoShort` (B 레터마크) | 카키 (`#3A4229`) | 흰색 |
| 브라우저 탭 파비콘 | `LogoShort` | 카키 (`#3A4229`) | 흰색 |
| 브라우저 확장 아이콘 | `LogoShort` | 카키 (`#3A4229`) | 흰색 |
| 밝은 배경 위 로고 (향후) | `LogoFull` 또는 `LogoShort` | 흰색/투명 | 카키 (`var(--color-primary-600)`) |

## 4. 구현

### 신규 파일

- `web-dashboard/src/components/Logo.tsx` — `LogoFull`, `LogoShort` 두 컴포넌트 export. SVG 경로는 공식 에셋(`images/logo_full.svg`, `images/logo_short.svg`)에서 추출, `fill="currentColor"` 적용.

### 수정 파일

| 파일 | 변경 내용 |

| `web-dashboard/src/pages/SignIn.tsx` | 기존 임시 SVG 아이콘 + 텍스트 → `<LogoFull>` 교체 |

### 컴포넌트 구조

```tsx
// Logo.tsx
export function LogoFull({ className }) { ... }   // viewBox="0 0 1530 546"
export function LogoShort({ className }) { ... }  // viewBox="0 0 557 714"
```

`currentColor` 방식을 사용하므로, 부모 요소의 CSS `color` 속성으로 로고 색상을 제어:
- 카키 배경 컨테이너: `color: #FFFFFF` → 흰색 로고
- 밝은 배경 컨테이너: `color: var(--color-primary-600)` → 카키색 로고
