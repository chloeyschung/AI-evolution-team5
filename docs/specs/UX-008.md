## 1. 개요

**문제**: 웹 대시보드, 브라우저 확장 전반에 걸쳐 인라인 SVG로 임시 제작된 로고가 사용되고 있었으며, iOS 스플래시 화면에도 SwiftUI로 수동 제작된 구버전 로고마크(`BrieflyLogoMark`)가 사용 중이었음. 브랜드 일관성이 없고 공식 로고와 달랐음.

**해결**: 공식 로고 에셋(`logo_full.svg`, `logo_short.svg`)을 기반으로 재사용 가능한 React 컴포넌트를 만들고, 노출 맥락(가로형/정방형)에 따라 올바른 로고를 적용. iOS는 Asset Catalog에 SVG를 template 이미지로 등록해 동일한 공식 워드마크를 사용.

**목표**:
- 전 플랫폼 브랜드 아이덴티티 통일 (웹 + iOS)
- 가로형(wordmark) / 정방형(lettermark) 사용 기준 정립
- 배경색에 따라 흰색·카키색 전환 가능한 구조 도입

**비목표**: 브라우저 확장 PNG 래스터 파일 교체.

### 컴포넌트 구조

```tsx
// Logo.tsx
export function LogoFull({ className }) { ... }   // viewBox="0 0 1530 546"
export function LogoShort({ className }) { ... }  // viewBox="0 0 557 714"
```

`currentColor` 방식을 사용하므로, 부모 요소의 CSS `color` 속성으로 로고 색상을 제어:
- 카키 배경 컨테이너: `color: #FFFFFF` → 흰색 로고
- 밝은 배경 컨테이너: `color: var(--color-primary-600)` → 카키색 로고
