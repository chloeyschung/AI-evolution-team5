# IOS-014: 키워드 / 카테고리 표시 (Keyword & Category Display)

**Status**: Planned | **Created**: 2026-06-08 | **Author**: suyoung  
**F-xxx Mapping**: F-006 (AI Auto-tagging) | **Phase**: Phase 3 | **Priority**: High

---

## 1. Overview

**목표**: 백엔드 AI 자동 태깅(`auto_tagger.py`)이 생성한 키워드·카테고리를  
iOS 앱의 라이브러리 카드와 상세 페이지에 웹 대시보드(`ContentCard.tsx`)와 동일한 방식으로 표시한다.

**백엔드 현황** (변경 없음):
- `auto_tagger.py`가 title + summary 기반으로 LLM 호출 → 카테고리 1개 + 키워드 2–3개(영어·원문) 생성
- DB: `Content.auto_tag_category` (String), `auto_tag_keywords_en` (JSON Text), `auto_tag_keywords_original` (JSON Text)
- `GET /content/{id}` 응답(`ContentResponse`)에 이미 세 필드가 포함됨

**웹 구현 (참조)**:
- `ContentCard.tsx:147–153` — `auto_tag_keywords_en`을 pill 태그 행으로 렌더링 (최대 8개)
- `ContentDrawer.tsx:127–136` — 드로어 상세에서 category + keywords_en 표시
- `.keyword` CSS: 0.7rem, `3px 8px` padding, `border-radius: 999px`, muted 배경·텍스트·보더

## 3. 데이터 흐름

```
백엔드 GET /content/{id}
    ↓ ContentDetail (BrieflyAPI.swift)
      auto_tag_category: String?
      auto_tag_keywords_en: [String]
      auto_tag_keywords_original: [String]
    ↓ StorageService.updateAutoTags(id:category:keywordsEn:keywordsOriginal:)
    ↓ SavedItem (in-memory + UserDefaults)
      autoTagCategory: String?
      autoTagKeywordsEn: [String]
      autoTagKeywordsOriginal: [String]
    ↓ liveItem (ItemDetailView) / viewModel.items (SavedItemsView)
    ↓ UI
```

키워드는 summary 폴링과 동일한 `fetchContentDetail` 응답에서 가져온다.  
별도 API 호출 없이 기존 GET 1회로 category·keywords 3개 필드를 추가 수신한다.

### 4-2. `BrieflyAPI.swift` — `ContentDetail` 확장

**파일**: `Briefly/Briefly/Services/BrieflyAPI.swift`

```swift
struct ContentDetail: Decodable {
    let id: Int
    let summary: String?
    let autoTagCategory: String?
    let autoTagKeywordsEn: [String]
    let autoTagKeywordsOriginal: [String]

    enum CodingKeys: String, CodingKey {
        case id, summary
        case autoTagCategory      = "auto_tag_category"
        case autoTagKeywordsEn    = "auto_tag_keywords_en"
        case autoTagKeywordsOriginal = "auto_tag_keywords_original"
    }

    init(from decoder: Decoder) throws {
        let c = try decoder.container(keyedBy: CodingKeys.self)
        id       = try  c.decode(Int.self,    forKey: .id)
        summary  = try? c.decode(String.self, forKey: .summary)
        autoTagCategory           = try? c.decode(String.self,   forKey: .autoTagCategory)
        autoTagKeywordsEn         = (try? c.decode([String].self, forKey: .autoTagKeywordsEn))    ?? []
        autoTagKeywordsOriginal   = (try? c.decode([String].self, forKey: .autoTagKeywordsOriginal)) ?? []
    }
}
```

`fetchContentDetail` 메서드 자체는 변경 없음 (동일 GET 엔드포인트).

### 4-4. `ItemDetailView.swift` — 폴링 시 키워드 저장 + UI 추가

**파일**: `Briefly/Briefly/Views/ItemDetailView.swift`

#### 폴링 조건 변경 (A안)

summary가 있어도 keywords가 비어있으면 계속 폴링:

```swift
// 변경 전
guard currentItem.summary == nil else { return }

// 변경 후
let needsSummary = (loadedSummary ?? currentItem.summary) == nil
let needsKeywords = currentItem.autoTagKeywordsEn.isEmpty
guard needsSummary || needsKeywords else { return }
```

#### 폴링 task 내 키워드 저장

`fetchContentDetail` 응답마다 keywords를 저장 (summary 여부와 무관하게):

```swift
if let detail = try? await BrieflyAPI.shared.fetchContentDetail(...) {
    // keywords 항상 저장 (빈 배열이 아닌 경우)
    if !detail.autoTagKeywordsEn.isEmpty {
        StorageService.shared.updateAutoTags(
            for: currentItem.id,
            category: detail.autoTagCategory,
            keywordsEn: detail.autoTagKeywordsEn,
            keywordsOriginal: detail.autoTagKeywordsOriginal
        )
        refreshLiveItem()
    }
    // summary는 기존과 동일
    if let summary = detail.summary {
        loadedSummary = summary
        StorageService.shared.updateSummary(for: currentItem.id, summary: summary)
        StorageService.shared.updateSummaryStatus(for: currentItem.id, status: .done)
        return  // summary + keywords 모두 완료
    }
}
```

**종료 조건**: summary를 획득하면 keywords 여부와 무관하게 반환 (keywords는 summary 이후 수 초 내 저장되므로, summary 획득 후 다음 폴링 사이클에서 자연스럽게 수집됨).

#### UI — 키워드 섹션 (`contentSection` 내부)

AI 요약 카드(`brieflyPrimary50` 배경) 아래, 본문 섹션 위에 삽입:

```swift
// keywords 섹션 — 카테고리 뱃지 + 키워드 pill 행
if currentItem.autoTagCategory != nil || !currentItem.autoTagKeywordsEn.isEmpty {
    keywordsSection
}
```

```swift
private var keywordsSection: some View {
    VStack(alignment: .leading, spacing: 8) {
        if let category = currentItem.autoTagCategory {
            Text(category)
                .font(.caption.weight(.semibold))
                .foregroundStyle(Color.brieflyPrimary600)
                .padding(.horizontal, 10)
                .padding(.vertical, 4)
                .background(Color.brieflyPrimary50,
                            in: Capsule())
                .overlay(Capsule().stroke(Color.brieflyPrimary200, lineWidth: 1))
        }

        if !currentItem.autoTagKeywordsEn.isEmpty {
            KeywordPillRow(keywords: currentItem.autoTagKeywordsEn)
        }
    }
}
```

`KeywordPillRow`는 공유 컴포넌트로 신규 파일에 정의 (4-5 참고).

### 4-5. `KeywordPillRow.swift` — 공유 키워드 UI 컴포넌트 (신규)

**파일**: `Briefly/Briefly/Views/Components/KeywordPillRow.swift`  
**Xcode 등록**: Briefly 타겟에만 추가 (ShareExtension에서 미사용)

```swift
import SwiftUI

/// 웹 `.tagRow` / `.keyword` CSS와 동일한 pill 행.
/// keywords가 빈 배열이면 아무것도 렌더링하지 않는다.
struct KeywordPillRow: View {
    let keywords: [String]
    var maxCount: Int = 8

    var body: some View {
        if keywords.isEmpty { EmptyView() } else {
            ScrollView(.horizontal, showsIndicators: false) {
                HStack(spacing: 6) {
                    ForEach(keywords.prefix(maxCount), id: \.self) { kw in
                        Text(kw)
                            .font(.system(size: 11))
                            .foregroundStyle(Color.brieflyInk400)
                            .padding(.horizontal, 8)
                            .padding(.vertical, 3)
                            .background(Color.brieflyInk50,
                                        in: Capsule())
                            .overlay(Capsule().stroke(Color.brieflyInk100, lineWidth: 1))
                    }
                }
            }
        }
    }
}
```

> 웹 `.keyword` 스타일 대응: `border-radius:999px` → `Capsule()`, muted 배경·텍스트·보더.

## 7. 엣지 케이스

| 케이스 | 처리 |
|--------|------|
| 태깅 미완료 (auto_tag_status != "done") | `autoTagKeywordsEn` 빈 배열 → UI 영역 자체가 숨겨짐 |
| 기존 저장 항목 (키워드 필드 없는 JSON) | 커스텀 디코더 → 빈 배열/nil → 재폴링 시 자동 저장 |
| 키워드 1개만 있는 항목 | 정상 표시 (최소 1개부터 렌더링) |
| 키워드 텍스트 길이 초과 | `KeywordPillRow`는 HStack이므로 가로 스크롤 허용 (라이브러리 row는 3개로 제한하여 overflow 방지) |
| category 없고 keywords 있는 항목 | 뱃지 생략, pill 행만 표시 |
| category 있고 keywords 없는 항목 | 뱃지만 표시, pill 행 생략 |
