# IOS-019: 검색 탭 — 빈 상태에 최근에 본 항목 표시

**Status**: Draft | **Created**: 2026-06-13 | **Author**: suyoung  
**Phase**: Phase 3 - Organization | **Priority**: Medium

---

## 1. Overview

**Problem**: 검색 탭에 처음 진입했을 때 아무것도 표시되지 않아, 검색 전까지 탭이 비어있는 느낌을 준다.

**Solution**: 검색어도 필터도 없는 빈 상태에서 "최근에 본 항목" 섹션을 표시한다. 홈 탭과 동일한 `ContentCardView` 카드를 가로 스크롤로 보여주며, 항목을 탭하면 상세 뷰로 이동한다. 최근에 본 항목이 없으면 기존 안내 문구(`emptyPrompt`)로 폴백한다.

**Goals**:
- 검색 탭 빈 상태(검색어 없음 + 필터 없음)에서 "최근에 본 항목" 가로 스크롤 표시
- 홈 탭 `ContentCardView` 카드 스타일 재사용 (160px, 썸네일 + 제목 + 도메인/날짜)
- `ItemDetailView` 진입 시 해당 항목을 최근 본 목록에 자동 기록
- 최근에 본 항목 없으면 기존 `emptyPrompt` 표시

**Non-Goals**:
- 서버 연동 (로컬 UserDefaults 기반)
- 최근 본 항목 수동 삭제 UI
- 앱 재설치 후에도 이력 유지

## 3. User Story / Behavior

```
[검색 탭 최초 진입 — 최근 본 항목 없음]
  → [카테고리 ▼] [플랫폼 ▼]
  → "저장한 콘텐츠를 검색해보세요" (기존 emptyPrompt)

[검색 탭 진입 — 최근 본 항목 있음]
  → [카테고리 ▼] [플랫폼 ▼]
  → "최근에 본 항목"  (섹션 헤더)
  → ←── [Card1] [Card2] [Card3] ... ──→ (가로 스크롤)

[Card 탭]
  → ItemDetailView 이동
  → 해당 항목이 최근 본 목록 맨 앞으로 이동

[검색바에 "AI" 입력]
  → 최근 본 섹션 사라짐
  → 검색 결과 표시 (IOS-018 동작)

[카테고리 필터 선택]
  → 최근 본 섹션 사라짐
  → 필터 결과 표시 (IOS-018 동작)
```

## 4. Implementation

### 변경/추가 파일

| 파일 | 변경 내용 |
|------|-----------|
| `Briefly/Services/RecentlyViewedStore.swift` | 신규 — UserDefaults 기반 최근 본 이력 관리 |
| `Briefly/Views/ItemDetailView.swift` | `.task(id: currentItem.id)`에서 `RecentlyViewedStore.shared.record()` 호출 |
| `Briefly/Views/SearchView.swift` | 빈 상태에 최근 본 섹션 추가, `path` 상태 추가 |

### RecentlyViewedStore

```swift
final class RecentlyViewedStore {
    static let shared = RecentlyViewedStore()
    private let key = "briefly_recently_viewed_ids"
    private let maxCount = 20

    func record(_ id: UUID) {
        var ids = loadIDs()
        ids.removeAll { $0 == id.uuidString }
        ids.insert(id.uuidString, at: 0)
        UserDefaults.standard.set(Array(ids.prefix(maxCount)), forKey: key)
    }

    func recentItems(from all: [SavedItem]) -> [SavedItem] {
        let ids = loadIDs()
        let map = Dictionary(uniqueKeysWithValues: all.map { ($0.id.uuidString, $0) })
        return ids.compactMap { map[$0] }
    }

    private func loadIDs() -> [String] {
        UserDefaults.standard.stringArray(forKey: key) ?? []
    }
}
```

### ItemDetailView — 기록 시점

`ItemDetailView`의 기존 `.task(id:)` 또는 `.onAppear` 에서 호출:

```swift
.onAppear {
    RecentlyViewedStore.shared.record(currentItem.id)
}
```

### SearchView — 빈 상태 교체

```swift
// 기존
if !hasActiveSearch { emptyPrompt }

// 변경
if !hasActiveSearch {
    if recentItems.isEmpty { emptyPrompt }
    else { recentlyViewedSection }
}
```

```swift
private var recentItems: [SavedItem] {
    RecentlyViewedStore.shared.recentItems(from: allItems)
}

private var recentlyViewedSection: some View {
    VStack(alignment: .leading, spacing: 12) {
        Text("최근에 본 항목")
            .font(.brieflyH3)
            .foregroundStyle(Color.brieflyTextPrimary)
            .padding(.horizontal, 16)

        ScrollView(.horizontal, showsIndicators: false) {
            HStack(spacing: 12) {
                ForEach(recentItems) { item in
                    ContentCardView(item: .local(item)) {
                        path.append(item)
                    }
                }
            }
            .padding(.horizontal, 16)
        }
    }
    .padding(.top, 8)
    .frame(maxWidth: .infinity, maxHeight: .infinity, alignment: .topLeading)
}
```

`SearchView`에 `@State private var path = NavigationPath()` 추가,  
`NavigationStack(path: $path)`로 교체.

### 레이아웃

```
NavigationStack(path: $path)
  VStack(spacing: 0)
    filterChipRow
    Group
      if !hasActiveSearch
        → recentItems.isEmpty ? emptyPrompt : recentlyViewedSection
      else if results.isEmpty
        → noResults
      else
        → resultList
      .overlay(alignment: .topLeading) { dropdownContent }
  .searchable(text: $query)
```

## 5. Edge Cases

| Scenario | Handling |
|----------|----------|
| 최근 본 항목이 없음 | `emptyPrompt` 폴백 |
| 최근 본 항목이 삭제됨(`status == .deleted`) | `allItems`가 `.deleted` 제외하므로 자동으로 결과에서 빠짐 |
| 동일 항목 재조회 | 기존 위치 제거 후 맨 앞 삽입 |
| 앱 재진입 시 목록 복원 | UserDefaults 영속 저장으로 유지 |
| 검색어/필터 활성화 | 최근 본 섹션 즉시 숨김, 검색 결과 표시 |

## 6. Acceptance Criteria

- [ ] 검색 탭 최초 진입 시 최근 본 항목 없으면 기존 emptyPrompt 표시
- [ ] ItemDetailView 조회 후 검색 탭 재진입 시 해당 항목이 가로 스크롤에 표시됨
- [ ] 카드가 홈 탭 ContentCardView와 동일한 스타일 (160px, 썸네일, 제목, 도메인)
- [ ] 카드 탭 시 ItemDetailView 이동
- [ ] 동일 항목 재조회 시 목록 맨 앞으로 이동
- [ ] 검색어 입력 또는 필터 선택 시 최근 본 섹션 사라짐
- [ ] 삭제된 항목은 최근 본 목록에서 자동 제외
- [ ] 앱 재시작 후에도 최근 본 목록 유지
