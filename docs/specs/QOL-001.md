## 1. Overview

**Problem**: Codebase had scattered enum definitions, inconsistent type hints, and magic values that reduced maintainability and type safety.

**Solution**: Centralized constants, standardized type syntax, and improved code organization.

**Goals**: Single source of truth for constants, consistent type hints, reduced magic numbers, improved discoverability

## 5. API Design

No API changes. All improvements are internal code organization.

### Error Code Standardization

**Before**: String literals scattered across codebase

**After**: ErrorCode enum
```python
class ErrorCode(StrEnum):
    # Authentication (401)
    UNAUTHORIZED = "unauthorized"
    INVALID_REFRESH_TOKEN = "invalid_refresh_token"
    INVALID_GOOGLE_TOKEN = "invalid_google_token"

    # Not Found (404)
    CONTENT_NOT_FOUND = "content_not_found"
    USER_NOT_FOUND = "user_not_found"

    # Validation (400)
    INVALID_STATE = "invalid_state"
    INVALID_FORMAT = "invalid_format"
```

### Type Hint Changes

**Before**:
```python
from typing import Optional

def get_user(id: int) -> Optional[User]:
    ...
```

**After**:
```python
def get_user(id: int) -> User | None:
    ...
```

### Import Changes

**Before**:
```python
from src.data.models import SwipeAction
```

**After**:
```python
from src.constants import SwipeAction
```
