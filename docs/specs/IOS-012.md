# IOS-012: LinkedIn FetchCoordinator oEmbed Fallback 버그 수정

**Status**: Planned | **Created**: 2026-06-04 | **Author**: suyoung
**F-xxx Mapping**: F-005 (AI Summary), F-003 (Content Metadata) | **Phase**: Phase 3 - Bug Fix | **Priority**: High

---

## 1. Overview

**Problem**: ING-004에서 LinkedIn oEmbed 지원을 추가했지만, oEmbed API 호출이 실패할 경우 iOS FetchCoordinator가 LinkedIn 로그인 페이지의 메타데이터와 설명문을 콘텐츠로 표시하는 회귀 버그가 발생한다.

**Root Cause**: FetchCoordinator 2단계(본문 텍스트)는 `isLinkedInURL` 조건만 보고 무조건 `ogDescription`을 사용한다. 하지만 1단계에서 oEmbed가 실패하면 `ogDescription`은 fallback된 일반 HTML 스크래핑(로그인 페이지)에서 온 값이다.

```
[현재 흐름 — oEmbed 실패 시]
1단계: isLinkedInURL + oEmbed 실패 → liMeta = nil
    → else 분기: fetchMetadata(linkedIn URL) → 로그인 페이지 HTML
    → ogTitle = "LinkedIn: Log In or Sign Up", ogDescription = "로그인 안내..."

2단계: isLinkedInURL == true (oEmbed 성공 여부 무관)
    → articleText = ogDescription  ← 로그인 페이지 설명문 사용 (버그)
```

**Solution**: oEmbed 성공 여부를 추적해서, 실패 시 LinkedIn URL도 일반 URL과 같이 articleText를 비워두거나 생략한다.

---

## 2. 문제 진단

### 현재 코드 (`FetchCoordinator.swift`)

```swift
// 1단계
let meta: PageMetadata
if Self.isYouTubeURL(item.url), let ytMeta = ... {
    meta = ytMeta
} else if Self.isLinkedInURL(item.url),
          let liMeta = try? await MetadataService.shared.fetchLinkedInMetadata(for: item.url) {
    meta = liMeta          // oEmbed 성공 시만 실행
} else {
    meta = try await MetadataService.shared.fetchMetadata(for: item.url)  // ← 로그인 페이지 수신
}

// 2단계
if Self.isYouTubeURL(item.url) {
    updated.articleText = updated.ogDescription
} else if Self.isLinkedInURL(item.url) {           // ← oEmbed 실패해도 여기 진입
    updated.articleText = updated.ogDescription    // ← 로그인 페이지 ogDescription 사용
} else {
    let articleText = try await fetchArticleText(for: item.url)
    ...
}
```

### 문제점

- oEmbed 실패 시 `ogDescription`은 LinkedIn 로그인 페이지의 홍보 문구
- 사용자에게 "LinkedIn에 로그인하세요" 같은 내용이 본문으로 표시됨
- `fetchStatus = .partial`로 설정되어 추가 재시도도 없음

## 3. 수정 방안

oEmbed 성공 여부를 로컬 변수로 추적한다.

### 수정 후 흐름

```
[oEmbed 성공 시]
1단계: liMeta 취득 → ogTitle = 실제 제목, ogDescription = 게시물 설명
2단계: isLinkedIn + oEmbedSucceeded → articleText = ogDescription (실제 내용)

[oEmbed 실패 시]
1단계: fetchMetadata(URL) → 로그인 페이지 메타데이터 (부정확하지만 최선)
2단계: isLinkedIn + !oEmbedSucceeded → articleText = nil (빈 상태)
     → fetchStatus = .partial
```

## 4. 구현 상세

### 변경 파일

- `Briefly/Briefly/Services/FetchCoordinator.swift`

### 변경 내용

`fetchOne(item:)` 내에서 oEmbed 성공 여부를 추적하는 로컬 변수 추가:

```swift
private func fetchOne(item: SavedItem) async {
    var updated = item
    updated.fetchStatus = .fetching
    StorageService.shared.updateItem(updated)

    do {
        // 1단계: OG 메타데이터
        let meta: PageMetadata
        var linkedInOEmbedSucceeded = false   // ← 추가

        if Self.isYouTubeURL(item.url),
           let ytMeta = try? await MetadataService.shared.fetchYouTubeMetadata(for: item.url) {
            meta = ytMeta
        } else if Self.isLinkedInURL(item.url),
                  let liMeta = try? await MetadataService.shared.fetchLinkedInMetadata(for: item.url) {
            meta = liMeta
            linkedInOEmbedSucceeded = true    // ← 추가
        } else {
            meta = try await MetadataService.shared.fetchMetadata(for: item.url)
        }
        // ...

        // 2단계: 본문 텍스트
        if Self.isYouTubeURL(item.url) {
            updated.articleText = updated.ogDescription
            updated.fetchStatus = .partial
        } else if Self.isLinkedInURL(item.url) {
            if linkedInOEmbedSucceeded {      // ← 조건 추가
                updated.articleText = updated.ogDescription
            }
            // oEmbed 실패 시 articleText = nil (빈 상태 유지)
            updated.fetchStatus = .partial
        } else {
            let articleText = try await fetchArticleText(for: item.url)
            updated.articleText = articleText ?? updated.ogDescription
            updated.fetchStatus = (articleText != nil) ? .done : .partial
        }
    }
    // ...
}
```

## 5. 테스트 시나리오

| 시나리오 | 기대 결과 |
|---------|----------|
| LinkedIn 공개 게시물 oEmbed 성공 | 실제 제목 + 게시물 설명 표시 |
| LinkedIn 비공개/oEmbed 실패 | 빈 본문, fetchStatus = .partial (로그인 내용 미표시) |
| YouTube URL | 기존 동작 유지 |
| 일반 URL | 기존 동작 유지 |

## 6. 한계 및 향후 개선

- oEmbed 실패 시 LinkedIn 콘텐츠에 AI 요약이 생성되지 않는 것은 의도된 동작(ING-004 설계)
- 공개 게시물이어도 LinkedIn oEmbed API가 일부 URL 형식에서 실패할 수 있음
- 근본적 해결: ING-005의 `page_text` 전송 구조로 iOS WKWebView가 긁은 텍스트를 백엔드에 전달
