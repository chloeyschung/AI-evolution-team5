# ING-003: YouTube 영상 Transcript 기반 요약

**Status**: Draft | **Created**: 2026-06-04 | **Author**: suyoung
**F-xxx Mapping**: F-005 (AI Summary - Real-time Generation) | **Phase**: Phase 3 - Bug Fix | **Priority**: High

---

## 1. Overview

**Problem**: YouTube URL을 공유하면 AI 요약이 영상 내용이 아닌 YouTube ToS/저작권 boilerplate를 요약한다.

예) "유튜브 콘텐츠는 크리에이터 게시물로 판매됩니다.", "Google LLC가 저작권 관련 정보를 제공합니다."

**Root Cause**: `ContentExtractor.fetch_html_and_text()`가 YouTube URL에 대해 일반 HTML scraping을 수행한다. YouTube 페이지는 JavaScript 렌더링 기반이라 실제 영상 콘텐츠가 HTML에 없고, `<body>` 태그에 ToS/저작권 boilerplate 텍스트만 남는다. 이 텍스트가 그대로 LLM에 전달되어 엉터리 요약이 생성된다.

**Solution**: YouTube URL 감지 시 `youtube-transcript-api` 패키지로 영상 자막(transcript)을 추출하고, 자막이 없으면 YouTube oEmbed API로 영상 설명을 fallback으로 사용한다.

---

## 2. 문제 진단

### 현재 흐름

```
POST /share (YouTube URL)
→ _background_summarize()
→ content_extractor.fetch_html_and_text(youtube.com/watch?v=...)
→ HTML scraping → ToS boilerplate 텍스트 추출
→ summarizer.summarize(boilerplate) → 엉터리 요약 저장
```

### 원하는 흐름

```
POST /share (YouTube URL)
→ _background_summarize()
→ YouTubeTranscriptExtractor.extract(video_id)
  ├─ [성공] transcript 텍스트 반환
  └─ [실패] oEmbed title + description 반환 (fallback)
→ summarizer.summarize(transcript) → 실제 영상 내용 요약 저장
```

| NFR-1 | Transcript 추출은 10초 이내 완료 | P1 |

### 4-2. YouTubeTranscriptExtractor 클래스 신설

**파일**: `src/ingestion/youtube_transcript.py` (신규)

```python
import re
from youtube_transcript_api import YouTubeTranscriptApi, NoTranscriptFound, TranscriptsDisabled
import httpx

YOUTUBE_URL_RE = re.compile(
    r"(?:youtube\.com/(?:watch\?.*v=|shorts/)|youtu\.be/)([A-Za-z0-9_-]{11})"
)
OEMBED_URL = "https://www.youtube.com/oembed?url={url}&format=json"
PREFERRED_LANGS = ["ko", "en"]

def extract_video_id(url: str) -> str | None:
    m = YOUTUBE_URL_RE.search(url)
    return m.group(1) if m else None

async def fetch_youtube_text(url: str) -> str | None:
    """YouTube URL에서 텍스트를 추출한다. 실패 시 None 반환."""
    video_id = extract_video_id(url)
    if not video_id:
        return None

    # 1단계: transcript 추출
    try:
        transcript_list = YouTubeTranscriptApi.list_transcripts(video_id)
        try:
            transcript = transcript_list.find_transcript(PREFERRED_LANGS)
        except NoTranscriptFound:
            transcript = next(iter(transcript_list))  # 첫 번째 자막 사용
        snippets = transcript.fetch()
        text = " ".join(s["text"] for s in snippets)
        if text.strip():
            return text
    except (NoTranscriptFound, TranscriptsDisabled):
        pass
    except Exception:
        pass

    # 2단계: oEmbed fallback
    try:
        async with httpx.AsyncClient(timeout=8.0) as client:
            r = await client.get(OEMBED_URL.format(url=url))
            r.raise_for_status()
            data = r.json()
            title = data.get("title", "")
            author = data.get("author_name", "")
            return f"{title}\n{author}".strip() or None
    except Exception:
        pass

    return None
```

### 4-3. `_background_summarize` 수정

**파일**: `src/api/routers/content.py`

```python
from src.ingestion.youtube_transcript import fetch_youtube_text, extract_video_id

async def _background_summarize(content_id, user_id, url, content_extractor, summarizer, page_text=None):
    ...
    # 기존 text_content 추출 이전에 YouTube 처리 삽입
    text_content = ""
    if page_text:
        text_content = page_text
    elif extract_video_id(url):                          # YouTube URL 감지
        yt_text = await fetch_youtube_text(url)
        if yt_text:
            text_content = yt_text
        # yt_text가 None이면 기존 HTML scraping fallback
    
    if not text_content:                                 # 기존 fallback 경로
        try:
            _, text_content = await content_extractor.fetch_html_and_text(url)
        except Exception as exc:
            logging.warning("Content extraction failed for content %s: %s", content_id, exc)
    ...
```

## 6. 제약 및 한계

| 항목 | 내용 |
|------|------|
| 자막 미제공 영상 | Shorts, 일부 라이브, 자막 비활성화 영상은 oEmbed 설명으로 대체 |
| 자동 생성 자막 품질 | YouTube 자동 생성 자막은 부정확할 수 있으나 boilerplate보다 월등히 유용 |

| 긴 영상 | 2시간+ 영상은 transcript가 매우 길어 summarizer의 context window 초과 가능 → 앞 6000자로 truncate |
