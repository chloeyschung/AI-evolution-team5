# IOS-005: 로그인 후 로컬 저장 항목 서버 업로드 (Post-Login Sync)

**Status**: In Progress | **Created**: 2026-05-09 | **Author**: suyoung
**F-xxx Mapping**: Post-login local→server sync | **Phase**: Phase 3 - Sync | **Priority**: Medium

---

## 1. Overview

**Problem**: 로그인 전에 Share Extension으로 저장한 링크들은 로컬 UserDefaults에만 존재하고 `serverContentId`가 없다. 이후 로그인을 해도 이 항목들은 서버(웹 대시보드)에 보이지 않는다.

**Solution**: 로그인 성공 직후, 로컬에 저장된 항목 중 `serverContentId == nil`인 것들을 `POST /api/v1/share`로 일괄 업로드하고 반환받은 서버 ID를 로컬에 저장한다.

**Goals**:
- 로그인 전 저장된 링크가 로그인 후 웹 대시보드에 표시된다
- Keep/Delete 처리 시 swipe API가 정상 호출된다 (`serverContentId` 보유)
- 이미 서버에 있는 항목은 중복 저장되지 않는다 (서버가 URL 중복 처리)

**Non-Goals**:
- 서버 → 로컬 역방향 동기화 (IOS-006+ 별도 티켓)
- 실시간 동기화 (로그인 시 1회 실행)

## 3. User Story / Behavior

**As a** 로그인 전에 이미 링크를 저장한 Briefly 사용자,  
**I want to** 로그인하고 나면 기존에 저장한 링크가 웹 대시보드에서도 보이길 원한다,  
**so that** 어떤 기기에서든 내 저장 목록을 확인하고 Keep/Delete 처리를 이어서 할 수 있다.

### Key Behaviors

```
[로그인 성공] → AccountView.loginSuccess 콜백
             → SyncService.syncLocalItemsToServer() 호출
             → 로컬 항목 중 serverContentId == nil 필터링
             → 항목별 POST /api/v1/share 호출
             → 성공 시 StorageService.updateItemById(serverContentId 포함)
             → 완료 후 NotificationCenter post → Library 화면 갱신
```

## 5. API Design

### 사용 엔드포인트

| Endpoint | Method | 설명 |
|----------|--------|------|
| `/api/v1/share` | POST | URL 저장 및 서버 content_id 발급 (이미 구현됨) |

### 중복 처리

서버의 `POST /api/v1/share`는 동일 URL에 대해 idempotency key 없이도 저장을 허용한다 (`allow_duplicate_url=True`). 따라서 클라이언트는 중복 여부를 별도 체크하지 않아도 된다.

## 6. Implementation

### Files

- `Briefly/Services/SyncService.swift` — 신규 파일, 로그인 후 업로드 로직
- `Briefly/Views/AccountView.swift` — 로그인 성공 콜백에서 `SyncService.sync()` 호출

### Key Logic

**SyncService.swift (신규)**:
```swift
final class SyncService {
    static let shared = SyncService()
    private init() {}

    func syncLocalItemsToServer(token: String) async {
        let items = StorageService.shared.loadAll()
        let unsynced = items.filter { $0.serverContentId == nil }
        guard !unsynced.isEmpty else { return }

        await withTaskGroup(of: Void.self) { group in
            for item in unsynced {
                group.addTask {
                    guard let result = try? await BrieflyAPI.shared.share(url: item.url, token: token) else { return }
                    var updated = item
                    updated.serverContentId = result.id
                    StorageService.shared.updateItemById(updated)

                    // 로그인 전에 이미 Keep/Delete된 항목이면 swipe 소급 적용
                    if item.status == .kept {
                        try? await BrieflyAPI.shared.swipe(contentId: result.id, action: .keep, token: token)
                    } else if item.status == .deleted {
                        try? await BrieflyAPI.shared.swipe(contentId: result.id, action: .discard, token: token)
                    }
                }
            }
        }

        NotificationCenter.default.post(name: .fetchCoordinatorDidUpdate, object: nil)
    }
}
```

**StorageService.swift — `loadAll()` 추가 필요**:
```swift
func loadAll() -> [SavedItem] {
    guard let defaults else { return [] }
    let main = decode(from: defaults, key: mainKey)
    let inbox = decode(from: defaults, key: inboxKey)
    return main + inbox
}
```

**AccountView.swift — 로그인 성공 후 sync 호출**:
```swift
// loginWithEmail / loginWithGoogle 성공 콜백 내부
Task {
    await SyncService.shared.syncLocalItemsToServer(token: accessToken)
}
```

## 7. Edge Cases

| Scenario | Handling |
|----------|----------|
| 로컬 항목이 없음 | guard 조건으로 early return |
| 일부 항목 업로드 실패 | `try?`로 무시, 나머지 항목은 계속 처리 |
| 이미 서버에 동일 URL 존재 | 서버가 새 row로 저장 (중복 허용), serverContentId 정상 반환 |
| 로그인 전 .kept/.deleted 항목 | 업로드 후 즉시 swipe API 호출로 서버 상태 소급 적용 |
| 업로드 중 토큰 만료 | 실패 항목 무시 (다음 로그인 시 재시도됨) |
| 네트워크 없음 | 모든 항목 실패 무시, 다음 로그인 시 재시도 |

- **수동 검증**:

- **Acceptance**:
  - 로그인 직후 로컬 항목이 웹에서 보임
  - 이후 Keep/Delete가 웹에도 반영됨

## 9. Sensory Verification

- **Visual (시각)**: 로그인 후 Library 화면 reload 시 항목 변동 없음 (로컬 항목 그대로 유지). 웹 새로고침 후 항목 등장.
- **Auditory (청각)**: `syncLocalItemsToServer` 시작/완료 로그. 업로드 항목 수 로그.
- **Tactile (촉각)**: UI 블로킹 없음. 로그인 후 즉시 앱 사용 가능.
