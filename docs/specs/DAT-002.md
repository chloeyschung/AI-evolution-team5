# DAT-002: User Profile & Preferences

**Status**: Implemented | **Created**: 2026-04-14 | **Author**: abraxaspark
**F-xxx Mapping**: N/A (Supporting Feature) | **Phase**: Phase 1 - MVP (Mobile Focus) | **Priority**: High

## 1. Overview

**Problem**: Users need personalized settings and persistent preferences for their content consumption experience.

**Solution**: User profile storage with preferences, statistics calculation, and interest tag management.

**Goals**: Personalized experience, sensible defaults, on-demand statistics, interest-based filtering foundation

**Non-Goals**: Multi-user per device (MVP is single-user), social features, advanced analytics

---

## 3. User Story / Behavior

As a Briefly user, I want my preferences and settings to be stored and synchronized, so that my content consumption experience is personalized and consistent across devices.

### Key Behaviors

- New user gets default profile and preferences on first access
- Profile fields can be updated independently
- Preferences persist with correct defaults
- Statistics calculated from swipe history
- Interest tags are unique and can be added/removed

---

## 4. Data Models

### New Tables

**UserProfile**: id, display_name, avatar_url, bio, created_at, updated_at

**UserPreferences**: user_id (unique), theme, notifications_enabled, daily_goal, default_sort, updated_at

**InterestTag**: id, user_id, tag (unique per user)

**UserStreak**: user_id (unique), current_streak, longest_streak, total_active_days, last_activity_date, created_at, updated_at

### Existing Used

- `SwipeHistory` (for statistics calculation)
- `Content` (for content counts)

## 5. API Design

| Endpoint | Method | Description |
|----------|--------|-------------|
| `/api/v1/profile` | GET | Get user profile |
| `/api/v1/profile` | PATCH | Update user profile |
| `/api/v1/preferences` | GET | Get user preferences |
| `/api/v1/preferences` | PATCH | Update user preferences |
| `/api/v1/statistics` | GET | Get user statistics |
| `/api/v1/interests` | GET | Get all interest tags |

## 6. Implementation

### Files

- `src/data/models.py` - UserProfile, UserPreferences, InterestTag, UserStreak models
- `src/data/repository.py` - UserProfileRepository with CRUD operations
- `src/api/schemas.py` - Profile, preferences, statistics schemas
- `src/api/routes.py` - Profile, preferences, statistics, interests endpoints

### Key Logic

1. **Auto-Create with Defaults**:
   - get_or_create_profile() creates if not exists
   - Default theme: SYSTEM
   - Default notifications: enabled
   - Default daily_goal: 20
   - Default sort: recency

2. **Statistics Calculation**:
   - Count total swipes from SwipeHistory
   - Count keep vs discard actions
   - Calculate retention rate (kept / total)
   - Calculate streak (consecutive days with activity)

3. **Interest Tag Management**:
   - Tags are unique per user (unique constraint)
   - Case-insensitive comparison
   - Trimmed for storage

### Dependencies

**Requires** (satisfied by):
- DAT-001 - Storage engine, SwipeHistory for statistics

**Provides** (for future):
- ADV-002 - Streak data for achievements
- UX-004 - Default sort preference
- ADV-003 - Notification preferences

## 7. Edge Cases

| Scenario | Handling |
|----------|----------|
| New user with no profile | Auto-create with defaults |
| Empty swipe history | Statistics show zeros/nulls |
| Duplicate interest tag | Idempotent (no duplicate created) |
| Case variation in tag | Treated as same tag (case-insensitive) |
