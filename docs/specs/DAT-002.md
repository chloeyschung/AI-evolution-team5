# DAT-002: User Profile & Preferences

## User Story

As a Briefly user, I want my preferences and settings to be stored and synchronized, so that my content consumption experience is personalized and consistent across devices.

## Dependencies

- **depends_on:** `@specs/DAT-001.md` (Storage engine for persistence)
- **depended_by:** `@specs/ADV-002.md` (Gamified Achievements - Phase 3)
- **dependency code:** `src/data/models.py`, `src/data/repository.py`

## Implementation Contract

### Level 1: Natural Language

User Profile & Preferences extends the storage engine to manage user-specific data. It provides:

1. **User Profile**: Basic user information (display name, avatar, bio)
2. **Preferences**: Content consumption settings (themes, notification preferences, default filters)
3. **Swipe Statistics**: Aggregated metrics for achievements and insights
4. **Interest Tags**: Content categories/topics the user is interested in

The system must:
- Store a single user profile per installation (MVP: single-user app)
- Persist preferences with sensible defaults
- Calculate swipe statistics on-demand from swipe history
- Support interest-based content filtering (foundation for Phase 3 features)

### Level 2: Logic Flow (Pseudocode)

```
INPUT: Profile/Preferences update request
PROCESS:
  1. Validate input against schema.
  2. Check if user profile exists.
  3. If exists: update fields.
  4. If not exists: create new profile with defaults.
  5. Persist to database.

INPUT: Statistics request
PROCESS:
  1. Count total swipes from SwipeHistory.
  2. Count keep vs discard actions.
  3. Calculate retention rate (kept / total).
  4. Find most active day/hour patterns.
  5. Return aggregated statistics.

INPUT: Interest tag management
PROCESS:
  1. Add/remove interest tags.
  2. Ensure uniqueness of tags.
  3. Persist updated tag list.

OUTPUT: Profile object, preferences dict, statistics object
ERROR: ValidationError (invalid input), NotFoundError (profile not found)
```

## Verification Contract

### Level 1: Scenarios (Natural Language)

- **Success:** New user gets default profile and preferences on first access.
- **Success:** Profile fields can be updated independently.
- **Success:** Preferences persist with correct defaults.
- **Success:** Statistics are accurately calculated from swipe history.
- **Success:** Interest tags are unique and can be added/removed.
- **Edge:** Statistics handle empty swipe history gracefully (zeros/nulls).

### Level 2: Given-When-Then

```
GIVEN a new user with no existing profile
WHEN get_or_create_profile is called
THEN a default profile is created and returned

GIVEN an existing profile
WHEN update_profile is called with new display_name
THEN the profile is updated and updated_at is refreshed

GIVEN a user with no preferences
WHEN get_preferences is called
THEN preferences with sensible defaults are returned
(theme=SYSTEM, notifications_enabled=true, daily_goal=20)

GIVEN 100 swipes (60 keep, 40 discard)
WHEN get_statistics is called
THEN statistics show total_swipes=100, retention_rate=0.6

GIVEN an existing interest tag "Technology"
WHEN add_interest_tag is called with "Technology"
THEN no duplicate is created (idempotent)

GIVEN empty swipe history
WHEN get_statistics is called
THEN statistics show zeros for counts and null for dates
```

## Implementation Notes

- **Single User MVP:** For MVP, assume single user per installation. User ID is implicitly 1.
- **Auto-create:** Profile and preferences are auto-created on first access with sensible defaults.
- **Statistics Calculation:** Statistics are calculated on-demand from SwipeHistory. Consider caching for large datasets in Phase 2.
- **Streak Calculation:** Streak is calculated as consecutive days with at least one swipe, ending today or yesterday.
- **Interest Tags:** Tags are case-insensitive and trimmed for comparison.
