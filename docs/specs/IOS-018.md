# IOS-018: 검색 탭 — 카테고리·플랫폼 필터

**Status**: Draft | **Created**: 2026-06-13 | **Author**: suyoung  
**Phase**: Phase 3 - Organization | **Priority**: High

---

## 1. Overview

**Problem**: 검색 탭에서 텍스트 키워드만으로 검색할 수 있어, 저장 항목이 많아질수록 원하는 콘텐츠를 좁히기 어렵다.

**Solution**: 검색 탭 콘텐츠 영역 상단에 IOS-017과 동일한 "카테고리"·"플랫폼" 드롭다운 멀티 필터 칩을 고정 배치한다. 텍스트 검색어와 AND 조합으로 동작하며, 필터만 선택해도 결과가 표시된다.

**Goals**:
- 검색 콘텐츠 영역 상단에 "카테고리"·"플랫폼" 필터 칩 두 개 고정 배치
- 텍스트 검색어(없을 수 있음) + 카테고리 필터 + 플랫폼 필터 AND 조합
- 필터만 선택해도 결과 표시 (검색어 없이도 동작)
- 각 필터 내에서는 복수 선택 OR 조합
- IOS-017의 `FilterDropdownChip` 컴포넌트 재사용

**Non-Goals**:
- 탭 전환 시 필터 상태 유지 (SearchView 독립)
- 서버사이드 필터링

## 3. User Story / Behavior

```
[검색 탭 진입]
  → 검색바 (내비게이션 상단)
  → [카테고리 ▼] [플랫폼 ▼]   ← 콘텐츠 영역 상단 고정
  → "저장한 콘텐츠를 검색해보세요" 안내

["카테고리 ▼" 탭 → "기술" 선택]
  → 드롭다운 유지, "기술" 체크마크
  → 칩: [기술 ▼]  (primary 색상)
  → 검색어 없어도 autoTagCategory == "Tech" 항목 목록 표시

[검색바에 "AI" 입력]
  → 텍스트 AND 카테고리 조합
  → Tech 항목 중 "AI" 포함 항목만 표시

["뉴스" 추가 선택]
  → 칩: [기술, 뉴스 ▼]
  → "AI" AND (Tech OR News) 항목

["플랫폼 ▼" → "youtube.com" 선택]
  → "AI" AND (Tech OR News) AND youtube.com 항목

[필터 결과 0개]
  → "조건에 맞는 항목이 없습니다 / 다른 키워드나 필터로 검색해보세요"
```

### 카테고리 언어 매핑 (IOS-017과 동일)

| 값 | 한국어 | 영어 |
|---|---|---|
| Tech | 기술 | Tech |
| Business | 비즈니스 | Business |
| Essays | 에세이 | Essays |
| Research | 연구 | Research |
| Lifestyle | 라이프스타일 | Lifestyle |
| News | 뉴스 | News |
| Culture | 문화 | Culture |
| Other | 기타 | Other |

## 4. Implementation

### 변경 파일

| 파일 | 변경 내용 |
|------|-----------|
| `Briefly/Views/SearchView.swift` | 필터 상태 추가, `results` 로직 확장, 필터 UI 추가 |

### 추가 타입

```swift
private enum FilterDropdown: Equatable { case category, platform }
```

### 추가 상태 변수

```swift
@State private var selectedCategories: Set<String> = []
@State private var selectedDomains: Set<String>    = []
@State private var activeDropdown: FilterDropdown? = nil
```

### 데이터 흐름

```swift
private var hasActiveSearch: Bool {
    !trimmed.isEmpty || !selectedCategories.isEmpty || !selectedDomains.isEmpty
}

private var allItems: [SavedItem] {
    StorageService.shared.loadAll().filter { $0.status != .deleted }
}

private var results: [SavedItem] {
    guard hasActiveSearch else { return [] }
    var items = allItems

    // 1. 텍스트 필터 (검색어 있을 때만)
    if !trimmed.isEmpty {
        let q = trimmed.lowercased()
        items = items.filter { /* 기존 텍스트 매칭 로직 */ }
    }

    // 2. 카테고리 필터 (OR)
    if !selectedCategories.isEmpty {
        items = items.filter { item in
            item.autoTagCategory.map { selectedCategories.contains($0) } ?? false
        }
    }

    // 3. 플랫폼 필터 (OR)
    if !selectedDomains.isEmpty {
        items = items.filter { selectedDomains.contains($0.url.normalizedDomain) }
    }

    return items.sorted { $0.savedAt > $1.savedAt }
}
```

### 레이아웃

```
NavigationStack
  VStack(spacing: 0)
    filterChipRow              ← [카테고리 ▼] [플랫폼 ▼]
    Group(content / noResults / emptyPrompt)
      .overlay(alignment: .topLeading) { dropdownContent }
  .searchable(text: $query)
  .navigationTitle("Search")
```

### noResults 문구 업데이트

| 상황 | 제목 |
|------|------|
| 검색어만 있고 결과 없음 | `'\(trimmed)'에 대한 결과가 없습니다` |
| 필터만 있거나 조합 결과 없음 | `조건에 맞는 항목이 없습니다` |

부제: `다른 키워드나 필터로 검색해보세요` (통일)

## 5. Edge Cases

| Scenario | Handling |
|----------|----------|
| 검색어 없음 + 필터 있음 | 필터 조건만으로 결과 표시 |
| 검색어 있음 + 필터 없음 | 기존과 동일한 텍스트 검색 |
| 모든 조건 없음 | `emptyPrompt` 표시 |
| 카테고리 없는 항목 | 카테고리 필터 선택 시 제외 |
| 필터 결과 0개 | `noResults` 표시 (조건 문구) |

## 6. Acceptance Criteria

- [ ] 검색 콘텐츠 영역 상단에 "카테고리"·"플랫폼" 칩 두 개 항상 표시
- [ ] 칩 탭 시 드롭다운 열림 (다른 드롭다운 자동 닫힘)
- [ ] 항목 복수 선택 가능, 선택/해제 시 드롭다운 유지 + 체크마크 토글
- [ ] "전체" 선택 시 필터 초기화 + 드롭다운 닫힘
- [ ] 선택된 항목 이름들이 칩에 쉼표로 표시됨
- [ ] 필터만 선택해도 결과 표시 (검색어 불필요)
- [ ] 검색어 + 필터 AND 조합 정상 동작
- [ ] 결과 없음 시 상황에 맞는 문구 표시
- [ ] 드롭다운 외부 탭 시 닫힘
- [ ] 시스템 언어(ko/en) 자동 적용
