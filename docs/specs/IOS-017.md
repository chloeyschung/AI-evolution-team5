# IOS-017: 라이브러리 탭 — 카테고리·출처 필터

**Status**: Draft | **Created**: 2026-06-13 | **Author**: suyoung  
**Phase**: Phase 3 - Organization | **Priority**: High

---

## 1. Overview

**Problem**: 저장 항목이 많아질수록 라이브러리 탭에서 원하는 콘텐츠를 찾기 어렵다. 현재 Inbox/Saved/Discarded 탭 구분만 있고 카테고리·출처 기반으로 좁혀보는 수단이 없다.

**Solution**: 제목 헤더 아래에 "카테고리"·"플랫폼" 두 칩을 고정 배치한다. 칩을 탭하면 드롭다운이 열려 선택지를 보여주고, 선택 시 칩 레이블이 선택값으로 바뀐다. 두 필터는 AND 조합으로 동작하며 탭 전환 후에도 유지된다. 시스템 언어(한국어/영어)를 따른다.

**Goals**:
- 제목 헤더 아래 고정 필터 칩 두 개 ("카테고리", "플랫폼")
- 칩 탭 → 드롭다운 열림, 항목 선택 → 드롭다운 닫힘·칩에 선택값 표시
- 카테고리 × 플랫폼 AND 조합 필터
- 탭(Inbox/Saved/Discarded) 전환 후에도 필터 상태 유지
- 시스템 언어(ko/기타) 자동 대응

**Non-Goals**:
- 다중 선택 (각 필터 단일 선택)
- 탭 전환 시 필터 초기화
- 서버사이드 필터링

|----|-------------|----------|
| FR-1 | 제목 헤더 아래 "카테고리"·"플랫폼" 칩 두 개를 항상 표시한다 | P0 |

| NFR-1 | 필터링은 메인 스레드 배열 연산으로 처리 | P0 |

## 3. User Story / Behavior

```
[라이브러리 탭 진입]
  → "Inbox 23" 제목 헤더
  → [카테고리 ▼] [플랫폼 ▼]   ← 고정 칩 두 개

["카테고리 ▼" 탭]
  → 드롭다운 열림
    [전체  ✓]
    [기술     ]
    [비즈니스  ]
    [뉴스     ]

["기술" 선택]
  → 드롭다운 닫힘
  → 칩: [기술 ▼]  (primary 색상)
  → 목록: autoTagCategory == "Tech" 항목만

["플랫폼 ▼" 탭 → "youtube.com" 선택]
  → 목록: Tech AND youtube.com 항목만

[탭 전환 → Saved]
  → 필터 상태 유지 (기술 + youtube.com)
  → Saved 항목 중 조건 일치 항목 표시
```

### 카테고리 언어 매핑

| 값 | 한국어 | 영어 |
|---|---|---|
| Tech | 기술 | Tech |
| Business | 비즈니스 | Business |
| Essays | 에세이 | Essays |
| Research | 연구 | Research |
| Lifestyle | 라이프스타일 | Lifestyle |
| News | 뉴스 | News |
| Culture | 문화 | Culture |
| Other | 기타 | Other |

## 4. Implementation

### 변경 파일

| 파일 | 변경 내용 |
|------|-----------|
| `Briefly/Views/LibraryView.swift` | `FilterDropdown` enum 추가, 드롭다운 상태 관리, `FilterDropdownChip` 컴포넌트 추가 |

### 추가 타입

```swift
private enum FilterDropdown: Equatable { case category, platform }
```

### 추가 상태 변수

```swift
@State private var selectedCategory: String? = nil
@State private var selectedDomain: String?   = nil
@State private var activeDropdown: FilterDropdown? = nil
```

### 데이터 흐름

```swift
private var tabItems: [SavedItem] { /* 탭 기준 필터 */ }

private var displayedItems: [SavedItem] {
    tabItems
        .filter { selectedCategory == nil || $0.autoTagCategory == selectedCategory }
        .filter { selectedDomain   == nil || $0.url.normalizedDomain == selectedDomain }
}

private var availableCategories: [String] { /* viewModel.items 전체 기반 */ }
private var availableDomains: [String]    { /* viewModel.items 전체 기반 */ }
```

### 레이아웃

```
VStack(spacing: 0) {
    titleHeaderView            ← 기존 (카운트는 displayedItems 기준)
    filterChipRow              ← [카테고리 ▼] [플랫폼 ▼]
    ZStack(alignment: .bottom) {
        content / emptyView
        floatingTabBar
    }
    .overlay(alignment: .topLeading) {
        dropdownContent        ← 드롭다운 카드 + 외부 탭 dismiss 레이어
    }
}
```

## 5. Edge Cases

| Scenario | Handling |
|----------|----------|
| 카테고리 없는 항목 | `autoTagCategory == nil` → 카테고리 필터 선택 시 제외, "전체" 시 포함 |
| 필터 결과 0개 | 기존 `emptyView` 표시 |
| 드롭다운 외부 탭 | `activeDropdown = nil` |
| 탭 전환 | 필터 상태 유지, 새 탭에 동일 필터 적용 |
