# IOS-017: 라이브러리 탭 — 카테고리·출처 필터

**Status**: Draft | **Created**: 2026-06-13 | **Author**: suyoung  
**Phase**: Phase 3 - Organization | **Priority**: High

---

## 1. Overview

**Problem**: 저장 항목이 많아질수록 라이브러리 탭에서 원하는 콘텐츠를 찾기 어렵다. 현재 Inbox/Saved/Discarded 탭 구분만 있고 카테고리·출처 기반으로 좁혀보는 수단이 없다.

**Solution**: 제목 헤더(타이틀 + 카운트) 바로 아래에 수평 스크롤 필터 바 한 줄을 추가한다.  
카테고리 칩 그룹과 출처 칩 그룹을 구분선으로 나눠 한 행에 표시하며, 두 필터는 탭 전환 후에도 유지된다.

**Goals**:
- 제목 헤더 아래 수평 스크롤 단일 필터 행 추가
- 카테고리(autoTagCategory) 칩 | 출처(normalizedDomain) 칩 한 줄 구성
- 카테고리 필터 × 출처 필터 AND 조합
- 탭(Inbox/Saved/Discarded) 전환 후에도 필터 상태 유지
- 동적 칩 — 전체 저장 항목에 실제로 존재하는 값만 표시

**Non-Goals**:
- 다중 선택 (카테고리, 출처 각각 단일 선택)
- 탭 전환 시 필터 초기화
- 서버사이드 필터링
- 필터 조합 저장·복원

## 2. Requirements

| ID | Requirement | Priority |
|----|-------------|----------|
| FR-1 | 제목 헤더 아래에 카테고리·출처 칩을 한 줄 수평 스크롤로 표시한다 | P0 |
| FR-2 | 카테고리 칩과 출처 칩 사이에 수직 구분선을 표시한다 | P1 |
| FR-3 | 카테고리 칩 목록은 전체 저장 항목의 `autoTagCategory` 고유값으로 동적 생성한다 | P0 |
| FR-4 | 출처 칩 목록은 전체 저장 항목의 `url.normalizedDomain` 고유값으로 동적 생성한다 | P0 |
| FR-5 | 카테고리 칩 탭 → 해당 카테고리 항목만 표시 (출처 필터와 AND) | P0 |
| FR-6 | 출처 칩 탭 → 해당 도메인 항목만 표시 (카테고리 필터와 AND) | P0 |
| FR-7 | 선택된 칩을 다시 탭하면 해제된다 (토글) | P0 |
| FR-8 | Inbox/Saved/Discarded 탭 전환 후에도 두 필터 상태를 유지한다 | P0 |
| FR-9 | 카테고리/출처 고유값이 1개 이하인 경우 해당 그룹을 숨긴다 | P1 |
| NFR-1 | 필터링은 메인 스레드 배열 연산으로 처리 (별도 async 불필요) | P0 |
| NFR-2 | 칩 행은 수평 스크롤 가능, 스크롤바 미표시 | P0 |

## 3. User Story / Behavior

```
[라이브러리 탭 진입 — Inbox]
  → "Inbox 23" 제목 헤더
  → [기술] [비즈니스] [뉴스] | [youtube.com] [medium.com]  ← 필터 행
  → 아이템 목록 (기존 동일)

[카테고리 "기술" 탭]
  → "기술" 칩 활성 (primary 색상)
  → 목록: autoTagCategory == "Tech" 항목만 표시

[출처 "youtube.com" 추가 탭]
  → "youtube.com" 칩도 활성
  → 목록: Tech AND youtube.com 항목만 표시

[탭 전환 → Saved]
  → 필터 상태 유지 (기술 + youtube.com 선택 유지)
  → Saved 항목 중 조건 일치 항목 표시

["기술" 칩 재탭]
  → 카테고리 필터 해제 → youtube.com 항목만 표시
```

### 카테고리 한글 매핑

| 값 | 표시 |
|---|---|
| Tech | 기술 |
| Business | 비즈니스 |
| Essays | 에세이 |
| Research | 연구 |
| Lifestyle | 라이프스타일 |
| News | 뉴스 |
| Culture | 문화 |
| Other | 기타 |
| (nil) | 칩 목록 미포함 |

## 4. Implementation

### 변경 파일

| 파일 | 변경 내용 |
|------|-----------|
| `Briefly/Views/LibraryView.swift` | 필터 상태 변수 추가, `filteredItems` → `tabItems`+`displayedItems` 분리, `LibraryFilterBar` 추가 |

### 상태 변수 (LibraryView에 추가)

```swift
@State private var selectedCategory: String? = nil   // nil = 필터 없음
@State private var selectedDomain: String?   = nil   // nil = 필터 없음
```

### 데이터 흐름

```swift
// 탭 기준 항목 (기존 filteredItems 역할)
private var tabItems: [SavedItem] { ... }

// 칩 필터 적용 후 최종 표시 목록
private var displayedItems: [SavedItem] {
    tabItems
        .filter { selectedCategory == nil || $0.autoTagCategory == selectedCategory }
        .filter { selectedDomain   == nil || $0.url.normalizedDomain == selectedDomain }
}

// 전체 항목 기반 칩 목록 (탭과 무관하게 전체에서 추출)
private var availableCategories: [String] {
    Array(Set(viewModel.items.compactMap(\.autoTagCategory))).sorted()
}
private var availableDomains: [String] {
    Array(Set(viewModel.items.map { $0.url.normalizedDomain })).sorted()
}
```

> `filteredItems` 참조는 모두 `displayedItems`로 교체.  
> `titleHeaderView`의 카운트는 `displayedItems.count`로 변경.

### LibraryFilterBar 레이아웃

```
ScrollView(.horizontal) {
    HStack {
        // 카테고리 칩 그룹 (count > 1일 때만)
        ForEach(categories) { FilterChip }
        // 구분선 (양쪽 그룹 모두 표시될 때만)
        Divider (height: 16)
        // 출처 칩 그룹 (count > 1일 때만)
        ForEach(domains) { FilterChip }
    }
}
```

#### 칩 스타일

| 상태 | 배경 | 텍스트 색 |
|------|------|-----------|
| 선택됨 | `brieflyPrimary500` | `.white` |
| 비선택 | `brieflyInk100` | `brieflyInk500` |

### 레이아웃 위치

```
VStack(spacing: 0) {
    titleHeaderView          ← 기존
    LibraryFilterBar(...)    ← 신규 (필터 행)
    ZStack(alignment: .bottom) {
        content / emptyView
        floatingTabBar       ← 기존 floating pill
    }
}
```

## 5. Edge Cases

| Scenario | Handling |
|----------|----------|
| 카테고리 없는 항목 | `autoTagCategory == nil` → 카테고리 필터 대상 제외, 필터 해제 시 포함 |
| 필터 결과 0개 | 기존 `emptyView` 표시 |
| 선택 칩 재탭 | 해당 필터 nil (토글 해제) |
| 탭 전환 | 필터 상태 유지, 새 탭 항목에 동일 필터 적용 |
| 카테고리/출처 고유값 ≤ 1 | 해당 그룹 미표시 |

## 6. Acceptance Criteria

- [ ] 필터 행이 제목 헤더 아래 한 줄로 표시됨
- [ ] 카테고리 칩 탭 → 해당 카테고리 항목만 표시
- [ ] 출처 칩 탭 → 해당 도메인 항목만 표시
- [ ] 두 필터 동시 적용 시 AND 조합
- [ ] 선택 칩 재탭 → 필터 해제
- [ ] Inbox → Saved 전환 후 필터 상태 유지
- [ ] 카테고리/출처 고유값 1개 이하인 그룹 미표시
- [ ] 필터 결과 0개일 때 emptyView 정상 표시
