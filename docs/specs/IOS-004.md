# IOS-004: "지금 읽기" — 저장 직후 상세 카드로 바로 이동

**Status**: In Progress (버그 수정 완료, 실 기기 검증 필요) | **Created**: 2026-05-09 | **Updated**: 2026-05-09 | **Author**: suyoung

---

## 1. Overview

**Problem**: 외부 앱에서 링크를 저장한 직후 뜨는 확인창의 "지금 읽기" 버튼을 누르면, 현재는 Briefly 앱이 켜지기만 할 뿐 방금 저장한 콘텐츠의 상세 카드가 바로 보이지 않는다.

**Solution**: "지금 읽기" 탭 시 `briefly://item?url=…` 딥링크로 앱을 열고, Library 탭을 활성화하면서 해당 아이템의 **Inbox 카드 리더 모드**(IOS-003의 Keep/Delete 버튼 포함)로 바로 진입한다.

**Goals**:
- "지금 읽기" → 앱 열림 → 해당 아이템 상세 카드 즉시 표시
- 카드는 Inbox 상태이므로 Keep / Skip Now / Delete 버튼 노출
- 아이템 로드가 아직 안 됐을 때도 정상 동작 (pending 처리)

**Non-Goals**:
- Safari나 외부 브라우저로 원문 직접 열기
- 딥링크 이외의 진입 경로 변경 (홈 위젯, 알림 등은 별도)

## 3. User Story / Behavior

**As a** Briefly 사용자,  
**I want to** 링크 저장 확인창에서 "지금 읽기"를 누르면 바로 그 콘텐츠 카드를 보고 싶다,  
**so that** 저장 후 즉시 콘텐츠를 읽고 Keep/Delete 여부를 결정할 수 있다.

### 현재 vs 목표 흐름

**현재:**
```
[지금 읽기] → briefly://item?url=... → 앱 열림 → Library 탭 활성화
           → SavedItem으로 navigate → ItemDetailView(showActions: false)
           → Keep/Delete 버튼 없음, 카드 진입은 되지만 아이템을 처리할 수 없음
```

**목표:**
```
[지금 읽기] → briefly://item?url=... → 앱 열림 → Library 탭 활성화
           → 해당 아이템이 Inbox에 있으면
             → InboxNavigation(items: [item], startIndex: 0)으로 navigate
             → ItemDetailView(showActions: true) — Keep/Skip/Delete 버튼 노출
           → 아이템 미로드 시 pendingDeepLinkURL 대기 → 로드 후 동일 진입
```

## 6. Implementation

### Files

- `Briefly/Views/LibraryView.swift` — `brieflyOpenItem` 수신 핸들러 수정

### Key Logic

**변경 전** (LibraryView의 deep link 핸들러):
```swift
.onReceive(NotificationCenter.default.publisher(for: .brieflyOpenItem)) { notification in
    guard let url = notification.object as? URL else { return }
    viewModel.reload()
    if let item = viewModel.items.first(where: { $0.url == url }) {
        path.append(item)          // SavedItem → showActions: false
    } else {
        pendingDeepLinkURL = url
    }
}
```

**변경 후**:
```swift
.onReceive(NotificationCenter.default.publisher(for: .brieflyOpenItem)) { notification in
    guard let url = notification.object as? URL else { return }
    viewModel.reload()
    if let item = viewModel.items.first(where: { $0.url == url }) {
        openItemAsInboxReader(item)
    } else {
        pendingDeepLinkURL = url   // 로드 대기
    }
}

// onChange(of: viewModel.items) 핸들러도 동일하게 변경
.onChange(of: viewModel.items) { items in
    if let url = pendingDeepLinkURL,
       let item = items.first(where: { $0.url == url }) {
        openItemAsInboxReader(item)
        pendingDeepLinkURL = nil
    }
}

// 헬퍼
private func openItemAsInboxReader(_ item: SavedItem) {
    guard item.status == .unread else {
        // 이미 처리된 아이템이면 단순 상세 진입
        path.append(item)
        return
    }
    // Inbox 아이템 → 카드 리더 모드 (Keep/Skip/Delete 버튼 노출)
    let inboxItems = viewModel.items.filter { $0.status == .unread }
    let startIndex = inboxItems.firstIndex(where: { $0.id == item.id }) ?? 0
    path.append(InboxNavigation(items: inboxItems, startIndex: startIndex))
}
```

## 7. Edge Cases

| Scenario | Handling |
|----------|----------|
| 아이템이 아직 inbox에서 drain되지 않은 상태 | `pendingDeepLinkURL`로 대기 → `viewModel.items` 변경 시 자동 처리 |
| 이미 Keep/Delete된 아이템의 딥링크 재진입 | `status != .unread` → 단순 상세 뷰 (showActions: false) |
| Inbox에 아이템이 없는 상태 (전부 처리됨) | `inboxItems` 빈 배열 가드 → path.append(item) fallback |
| 앱이 완전 종료 상태에서 딥링크 수신 | `onOpenURL` → `ContentView` 초기화 완료 후 처리 (기존 구조 동일) |

### 시뮬레이터 테스트 방법

딥링크 자체는 아래 명령으로 직접 테스트 가능:

```bash
xcrun simctl openurl <device-id> "briefly://item?url=https%3A%2F%2Fexample.com%2Farticle"
```

→ `onOpenURL` → Library 탭 전환 → 카드 리더 진입 흐름 검증 완료.

### 검증 결과

- `xcrun simctl openurl` 로 딥링크 직접 테스트: **Library 탭 전환 + 카드 리더 진입 정상 동작 확인**
- "지금 읽기" 버튼 → 앱 전환: 시뮬레이터 플랫폼 제한으로 미동작, **실 기기 검증 필요**
