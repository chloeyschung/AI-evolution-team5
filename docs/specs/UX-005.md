# UX-005: Search by Title/Tag

**Status**: Implemented | **Created**: 2026-04-14 | **Author**: abraxaspark
**F-xxx Mapping**: F-016 (Search by Title/Tag) | **Phase**: Phase 1 - MVP (Mobile Focus) | **Priority**: Medium

---

## 1. Overview

**Problem**: Users need to find specific content in their library by title, author, or topic without manually scrolling through all saved items.

**Solution**: Real-time search across content titles, authors, and AI-generated category tags with relevance-based ranking.

**Goals**: Fast content discovery, topic-based search, unified search across all content statuses

**Non-Goals**: Full-text search of content body, search history, saved search queries

## 3. User Story / Behavior

As a Briefly user, I want to search my saved content by title or topic, so that I can quickly find specific articles or content about a particular subject.

### Key Behaviors

- Search bar appears in content library screen
- Results update in real-time as user types (debounced)
- Searches title, author, and AI-generated tags
- Results sorted by relevance (title > author > tag), then recency
- Shows content from both INBOX and ARCHIVED
- Empty results: "No matches found" message
- Minimum 2 characters to trigger search

## 6. Implementation

### Files

- `src/api/routes.py` - Search endpoint with query logic
- `src/api/schemas.py` - SearchResponse schema
- `src/data/repository.py` - ContentRepository.search()

### Key Logic

1. **Search Algorithm**:
   - Title search: `LOWER(title) LIKE LOWER('%query%')`
   - Author search: `LOWER(author) LIKE LOWER('%query%')`
   - Tag search: JOIN ContentTag, `LOWER(tag) LIKE LOWER('%query%')`
   - Combined: UNION or OR of all conditions

2. **Relevance Scoring** (SQLite):
   - Simple ordering: title matches first, then author, then tags
   - ORDER BY:
     - CASE WHEN title LIKE query THEN 1 ELSE 3 END
     - CASE WHEN author LIKE query THEN 2 ELSE 3 END
     - created_at DESC (for recency within same relevance)

3. **Debouncing**:
   - Frontend: 300ms debounce on keystroke
   - Cancel pending requests on new query
   - Show loading state during search

4. **Query Validation**:
   - Minimum 2 characters
   - Trim whitespace
   - Escape special characters (SQL injection prevention)

## 7. Edge Cases

| Scenario | Handling |
|----------|----------|
| Query < 2 characters | No search triggered, show all or hint |
| No results found | Empty list with "No matches found" message |
| Special characters in query | Escape for SQL, treat as literal |
| Very long query | Truncate to 100 chars, still search |
| Content with no tags | Still searchable by title/author |
| Duplicate results (title + tag match) | Deduplicate by content_id |
