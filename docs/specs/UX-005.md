# UX-005: Search by Title/Tag

**F-016 Mapping**: Search by Title/Tag
**Phase**: Phase 1 - MVP (Mobile Focus)
**Priority**: Medium - Content discovery

## Overview

Real-time search across content titles and category tags. Search scope: INBOX + Archive integrated.

## Requirements

### 1. Search Across Titles and Tags
- Search content by title (case-insensitive)
- Search content by author (case-insensitive)
- Search content by AI-generated category tags (case-insensitive)

### 2. Real-Time Search
- Instant results as user types
- Debounced API calls (300ms delay)
- Shows results immediately

### 3. Search Scope: All Content
- Search INBOX content
- Search ARCHIVED content
- No separation between statuses

### 4. Relevance Sorting
- Title matches ranked higher than author matches
- Exact matches ranked higher than partial matches
- Results sorted by relevance, then recency

## API Design

### GET /api/v1/search?q=keyword

Search content by query string.

**Request:**
```
Authorization: Bearer <access_token>
Query: q=keyword, limit=50
```

**Response (200 OK):**
```json
[
  {
    "id": 1,
    "platform": "YouTube",
    "content_type": "video",
    "url": "...",
    "title": "How to Build an App",
    "author": "Tech Channel",
    "status": "inbox",
    "created_at": "2024-01-01T00:00:00Z"
  }
]
```

## Implementation Details

### Search Algorithm

1. **Title Search**: `ILIKE '%query%'` on title field
2. **Author Search**: `ILIKE '%query%'` on author field
3. **Tag Search**: Join with ContentTag (AI-generated tags), `ILIKE '%query%'` on tag field
4. **Combined**: UNION or OR of all search conditions

### Relevance Scoring (Optional)

Use PostgreSQL ranking or simple ordering:
1. Title exact match
2. Title partial match
3. Author match
4. Tag match

### SQLite Implementation

SQLite doesn't have full-text search, so use:
- `LOWER(title) LIKE LOWER('%query%')`
- Simple relevance: ORDER BY title match first, then author
