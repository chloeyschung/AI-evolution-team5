# INT-001: YouTube Auto-Sync

**Status**: Implemented | **Created**: 2026-04-14 | **Author**: abraxaspark
**F-xxx Mapping**: F-022 (YouTube Auto-Sync) | **Phase**: Phase 3 - Platform Integrations | **Priority**: Medium

---

## 1. Overview

**Problem**: Users manually save YouTube videos one-by-one, missing the convenience of automatic ingestion from their "Watch Later" playlist or custom playlists.

**Solution**: YouTube Data API v3 integration with OAuth 2.0, allowing users to connect their account and automatically sync selected playlists.

**Goals**: Hands-free content ingestion, playlist-based organization, AI summarization of video content

**Non-Goals**: Video comments sync, channel subscription tracking, real-time sync (MVP scope)

## 3. User Story / Behavior

As a Briefly user, I want to automatically sync my YouTube "Watch Later" playlist, so that new videos are ingested and summarized without manual effort.

### Key Behaviors

- Connect YouTube: OAuth 2.0 flow, grant read-only permission
- View playlists: "Watch Later" (default) + custom playlists
- Select playlists: multi-select with sync frequency per playlist
- Manual sync: trigger immediately via API
- Auto sync: background task runs at configured frequency (Phase 2)
- New videos: fetched, metadata extracted, AI summary generated
- Duplicates: skipped if video ID already exists
- Sync log: view history of sync operations with counts/errors

## 6. Implementation

### Files

- `src/integrations/youtube/client.py` - YouTube API client with OAuth
- `src/integrations/youtube/sync.py` - Sync logic and orchestration
- `src/integrations/youtube/models.py` - Pydantic models for YouTube data
- `src/integrations/repositories/integration.py` - Integration state repository
- `src/integrations/tasks.py` - Celery tasks for background sync (Phase 2)
- `src/data/models.py` - IntegrationTokens, IntegrationSyncConfig, IntegrationSyncLog

### Key Logic

1. **YouTube API Client**:
   - googleapiclient.discovery.build('youtube', 'v3')
   - OAuth 2.0 with Credentials from stored tokens
   - Auto-refresh token before expiry
   - Methods: get_user_id(), get_playlists(), get_playlist_items()

2. **Sync Service**:
   - Fetch playlist items with pagination (max 50 per page)
   - Filter new videos since last_sync_at
   - For each video:
     - Check if URL exists (skip if duplicate)
     - Create Content record with metadata
     - Trigger AI summarization (async)
   - Update last_sync_at timestamp
   - Log sync result (ingested, skipped, errors)

3. **Token Management**:
   - Store: IntegrationTokens with encrypted access_token, refresh_token
   - Refresh: OAuth 2.0 token refresh flow
   - Rotate: Update tokens on each API call if close to expiry

4. **Background Sync** (Phase 2):
   - Celery task: youtube_sync_task(user_id, playlist_id)
   - APScheduler: check due syncs every hour
   - Retry: exponential backoff on failure

## 7. Edge Cases

| Scenario | Handling |
|----------|----------|
| OAuth token expired | Auto-refresh, retry operation |
| API rate limit exceeded | Queue sync, retry after quota reset |
| Playlist deleted | Log error, mark config inactive |
| Video private/deleted | Skip, log warning |
| Duplicate video ID | Skip, don't re-ingest |
| Network error during sync | Retry with exponential backoff |
