# IOS-016: 검색 탭 — 로컬 콘텐츠 전문 검색

**Status**: Draft | **Created**: 2026-06-10 | **Author**: suyoung
**F-xxx Mapping**: F-008 (Search) | **Phase**: Phase 3 - Organization | **Priority**: High

## 1. Overview

**Problem**: 저장한 콘텐츠가 늘어날수록 특정 기사를 다시 찾기 어렵다. 현재 검색 탭은 "준비 중" 플레이스홀더만 표시된다.

**Solution**: 검색 탭에 로컬 저장 항목 전문 검색을 구현한다. 제목·AI 요약·키워드·본문 등 모든 텍스트 필드를 대상으로 실시간 검색하며, 결과를 탭하면 콘텐츠 상세 화면으로 이동한다.

**Goals**:
- 저장된 모든 콘텐츠를 제목·키워드·본문으로 검색
- 쿼리 입력 즉시 결과 갱신 (실시간)
- 결과 탭 → ItemDetailView 이동

**Non-Goals**:
- 서버사이드 검색 (로컬 데이터만 대상)
- 검색 히스토리 저장
- 퍼지 매칭 / 오타 교정
- 검색 결과 랭킹 정렬 (저장 날짜 최신순 고정)

## 3. User Story / Behavior

```
[검색 탭 진입]
  → 검색 바 포커스, "제목, 키워드, 본문 검색..." placeholder
  → 쿼리 없음: "저장한 콘텐츠를 검색해보세요" 안내

[쿼리 입력]
  → 실시간 로컬 필터링
  → 결과 목록: 저장 최신순

[결과 탭]
  → ItemDetailView(items: [item], startIndex: 0, showActions: true)

[결과 없음]
  → "'{쿼리}'에 대한 결과가 없습니다"
```

### 검색 대상 필드 및 우선순위

| 필드 | 설명 |
|------|------|
| `displayTitle` | 표시 제목 (ogTitle → title → domain 폴백) |
| `ogDescription` | OG 메타 설명 |
| `autoTagKeywordsEn` | AI 추출 영문 키워드 배열 |
| `autoTagKeywordsOriginal` | AI 추출 원문 키워드 배열 |
| `autoTagCategory` | AI 분류 카테고리 |
| `summary` | AI 요약 |
| `articleText` | 본문 텍스트 |
| URL `host` | 도메인 |

## 4. Implementation

### 변경 파일

| 파일 | 변경 내용 |
|------|-----------|
| `Briefly/Views/SearchView.swift` | 전면 재구현 |

### 구조

```
SearchView
├── .searchable(text: $query, prompt: "...")  ← iOS 네이티브 검색 바
├── 쿼리 비어있음 → EmptyPromptView ("저장한 콘텐츠를 검색해보세요")
├── 결과 없음    → NoResultsView ("'{query}'에 대한 결과가 없습니다")
└── 결과 있음    → List { SearchResultRow × n }
    └── 탭 → NavigationLink(value: SavedItem)
        └── .navigationDestination(for: SavedItem.self) → ItemDetailView
```

### 검색 로직

```swift
private var results: [SavedItem] {
    guard !query.trimmingCharacters(in: .whitespaces).isEmpty else { return [] }
    let q = query.lowercased()
    return StorageService.shared.loadAll()
        .filter { $0.status != .deleted }
        .filter { item in
            item.displayTitle.lowercased().contains(q) ||
            (item.ogDescription?.lowercased().contains(q) ?? false) ||
            (item.summary?.lowercased().contains(q) ?? false) ||
            (item.articleText?.lowercased().contains(q) ?? false) ||
            item.autoTagKeywordsEn.contains { $0.lowercased().contains(q) } ||
            item.autoTagKeywordsOriginal.contains { $0.lowercased().contains(q) } ||
            (item.autoTagCategory?.lowercased().contains(q) ?? false) ||
            (item.url.host?.lowercased().contains(q) ?? false)
        }
        .sorted { $0.savedAt > $1.savedAt }
}
```

### SearchResultRow 표시 항목

- 썸네일 (ogImageURL, 없으면 도메인 파비콘)
- 제목 (`displayTitle`, 2줄 제한)
- 도메인 + 저장 날짜
- AI 키워드 pill (최대 3개, 있을 경우)

## 5. Edge Cases

| Scenario | Handling |
|----------|----------|
| 쿼리 공백만 입력 | trimming 후 빈 문자열 취급 → 빈 상태 표시 |
| 저장 항목 0개 | 빈 상태와 동일 ("저장한 콘텐츠를 검색해보세요") |
| 아주 긴 본문 검색 | `contains` 는 O(n) — 수백 개 항목 기준 즉각 반응 가능 |
