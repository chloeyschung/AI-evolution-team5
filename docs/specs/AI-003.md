# AI-003: AI Categorization

**Status**: Implemented | **Created**: 2026-04-14 | **Author**: abraxaspark

## 1. Overview

**Problem**: Users need to organize and search their saved content by topic, but manual tagging is time-consuming and inconsistent.

**Solution**: AI-powered auto-tagging that generates 1-3 relevant category tags from content title and summary.

**Goals**: Accurate topic classification, consistent tag format, non-blocking async processing

**Non-Goals**: User-defined tag taxonomy, manual tag editing, tag merging/deduplication

---

## 3. User Story / Behavior

As a Briefly user, I want my saved content to be automatically categorized with relevant tags, so that I can search and filter by topic without manual effort.

### Key Behaviors

- Content saved via /share endpoint
- System triggers async tag generation
- LLM analyzes title + summary
- Generates 1-3 relevant tags
- Tags stored and available for search/filter

## 6. Implementation

### Files

- `src/ai/categorizer.py` - Categorizer service with LLM integration
- `src/data/models.py` - ContentTag model
- `src/data/repository.py` - Tag CRUD operations

### Key Logic

1. **Tag Generation Prompt**:
   ```
   Analyze this content and generate 1-3 category tags.
   Title: {title}
   Summary: {summary}

   Rules:
   - Max 3 tags
   - Each tag 1-3 words
   - Lowercase, no special characters
   - Most relevant categories only

   Tags:
   ```

2. **Tag Generation Flow**:
   - Content saved via /share endpoint
   - Trigger async tag generation (background task)
   - Call LLM with title + summary
   - Parse response for tags (comma-separated or list)
   - Validate format (lowercase, max 50 chars, no special chars)
   - Save tags to ContentTag table

3. **Error Handling**:
   - LLM call fails → log error, no tags
   - Invalid response → retry once
   - Still fails → content saved without tags (graceful degradation)

### Dependencies

**Requires** (satisfied by):
- AI-001 - Summary exists for tag generation
- ING-002 - Content extracted with title

**Provides** (for future):
- UX-005 - Tag search functionality
- ADV-001 - Interest matching based on tags

## 7. Edge Cases

| Scenario | Handling |
|----------|----------|
| LLM API fails | Log error, content saved without tags |
| Invalid response format | Retry once, then skip |
| Title/summary too short | Generate best-effort tags |
| Duplicate tags in response | Deduplicate before saving |
| Tag exceeds 50 chars | Truncate to 50 chars |
