# AI-003: AI Categorization

**F-006 Mapping**: AI Categorization
**Phase**: Phase 1 - MVP (Mobile Focus)
**Priority**: Medium - Content organization

## Overview

Auto-classify content into AI-generated category tags (max 3 tags per content) using LLM. Free-form tags, no predefined category list.

## Requirements

### 1. AI-Generated Tags
- Use LLM to analyze content title and summary
- Generate 1-3 relevant category tags
- Tags are free-form (no predefined list)
- Tags should be concise (1-3 words)

### 2. Max 3 Tags Per Content
- Limit to maximum 3 tags
- Prioritize most relevant tags
- Tags should be diverse (not overlapping)

### 3. Tag Format
- Lowercase
- Space-separated words (e.g., "machine learning", not "machine_learning")
- No special characters
- Max 50 characters per tag

### 4. Async Processing
- Tag generation happens after content is saved
- Does not block content ingestion
- Can be retried on failure

## API Design

### POST /api/v1/content/{content_id}/categorize

Trigger AI categorization for specific content.

**Request:**
```
Authorization: Bearer <access_token>
```

**Response (200 OK):**
```json
{
  "content_id": 1,
  "tags": ["technology", "programming", "python"]
}
```

### GET /api/v1/content/{content_id}/tags

Get AI-generated tags for content.

**Response (200 OK):**
```json
{
  "content_id": 1,
  "tags": ["technology", "programming", "python"]
}
```

## Data Models

### New: ContentTag table

```python
class ContentTag(Base):
    __tablename__ = "content_tags"

    id = Column(Integer, primary_key=True)
    content_id = Column(Integer, ForeignKey("content.id"), nullable=False)
    tag = Column(String(50), nullable=False)
    created_at = Column(DateTime, default=utc_now)

    # Relationship
    content = relationship("Content", backref="tags")
```

## Implementation Details

### Tag Generation Prompt

```
Analyze this content and generate 1-3 category tags.
Title: {title}
Summary: {summary}

Rules:
- Max 3 tags
- Each tag 1-3 words
- Lowercase, no special characters
- Most relevant categories only

Tags:
```

### Tag Generation Flow

1. Content saved via /share endpoint
2. Trigger async tag generation
3. Call LLM with title + summary
4. Parse response for tags
5. Save tags to ContentTag table

### Error Handling

1. LLM call fails → log error, no tags
2. Invalid response → retry once
3. Still fails → content saved without tags
