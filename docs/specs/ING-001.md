# ING-001: Mobile Share Sheet Integration

**Status**: Implemented | **Created**: 2026-04-14 | **Author**: abraxaspark
**F-xxx Mapping**: F-004 (Mobile Share Sheet Integration) | **Phase**: Phase 1 - MVP (Mobile Focus) | **Priority**: Critical

---

## 1. Overview

**Problem**: Users need to save content to Briefly instantly from any app without context switching or manual URL entry.

**Solution**: OS-level share sheet integration that intercepts share actions and processes various content types (URLs, text, images, deep links) into standardized format.

**Goals**: Single-tap content capture from any app, automatic content type detection, zero manual input required

**Non-Goals**: Browser extension (EXT-001), auto-sync from platforms (INT-001/002), native share sheet UI implementation (handled by React Native app)

## 3. User Story / Behavior

As a time-poor professional reading an article, I want to save it to Briefly with one tap from the share sheet, so that I can capture information instantly without context switching.

### Key Behaviors

- User taps "Share" in any app (browser, messaging, etc.)
- Selects "Briefly" from share sheet options
- System auto-detects content type (URL, text, image, deep link)
- Routes to appropriate processor for metadata extraction
- Returns structured metadata for downstream AI processing

## 6. Implementation

### Files

- `src/ingestion/share_handler.py` - Primary entry point for share data processing
- `src/ingestion/share_types.py` - ShareData and ShareDataType definitions
- `src/ingestion/share_processor.py` - Content type-specific processors (URL, text, image, deep link)
- `src/ingestion/exceptions.py` - ShareDataError and related exceptions
- `src/ingestion/utils.py` - URL extraction, validation utilities

### Key Logic

1. **Content Type Detection**:
   - HTTP/HTTPS URL pattern → URL type
   - Custom scheme (app://, twitter://) → Deep link type
   - Image data (base64, image URL) → Image type
   - Fallback → Plain text type

2. **Processor Routing**:
   - URL → URLShareProcessor (extract text, metadata, generate summary)
   - Plain text → PlainTextProcessor (check for embedded URLs)
   - Deep link → DeepLinkProcessor (identify source app, content type)
   - Image → ImageProcessor (validate format, extract metadata)

3. **Data Flow**:
   - Raw share payload → ShareHandler.process_share()
   - Detect type → Route to processor
   - Processor → ContentMetadata
   - Save to Content table

### Dependencies

**Requires** (satisfied by):
- None (entry point feature)

**Provides** (for future):
- AI-001, AI-002, AI-003 - Content for summarization, metadata extraction, categorization
- ING-002 - URLs for content extraction

## 7. Edge Cases

| Scenario | Handling |
|----------|----------|
| Empty content field | Raise ShareDataError |
| Text with embedded URL | Extract URL, process as URL type |
| Invalid URL format | Fallback to plain text |
| Unknown deep link scheme | Extract scheme as source platform |
| Unsupported image format | Raise ImageProcessingError |
