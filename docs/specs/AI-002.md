# AI-002: Multi-Modal Metadata Extraction

## User Story
As a Briefly user, I want the system to automatically extract metadata (source platform, content type, timestamp) from any ingested content, so that I can organize and filter my knowledge library effectively.

- **dependency code:** `src/ingestion/extractor.py`

## Implementation Contract

### Level 1: Natural Language
The `MetadataExtractor` service analyzes the source URL and extracted content to identify:
- **Source Platform**: The origin platform (e.g., YouTube, LinkedIn, Medium, news site).
- **Content Type**: The type of content (e.g., article, video, image, social post).
- **Timestamp**: When the content was published (if available).
- **Author**: The creator of the content (if available).

This metadata is stored alongside the summary for filtering, search, and display purposes.

### Level 2: Logic Flow (Pseudocode)
```
INPUT: url (string), html_content (string, optional)
PROCESS:
  1. Parse URL to extract domain and path components.
  2. Identify source platform by matching domain against known platforms:
     - youtube.com / youtu.be → "YouTube"
     - linkedin.com → "LinkedIn"
     - medium.com → "Medium"
     - twitter.com / x.com → "Twitter/X"
     - instagram.com → "Instagram"
     - Default: Extract site name from domain or use "Web"
  3. Determine content type:
     - URL path contains "/watch" or "/shorts" → "video"
     - URL path contains "/p/" or "/posts/" → "social_post"
     - URL path contains "/pictures/" or "/reel/" → "image"
     - Default: "article"
  4. Extract timestamp from HTML (if available):
     - Look for <time> tags with datetime attribute.
     - Look for meta tags: og:article:published_time, article:published_time.
     - Parse and normalize to ISO 8601 format.
  5. Extract author from HTML (if available):
     - Look for meta tags: article:author, og:author.
     - Look for <author> or <byline> elements.
OUTPUT: metadata_dict (dictionary with keys: platform, content_type, timestamp, author)
ERROR: MetadataExtractionError (if extraction fails unexpectedly)
```

## Verification Contract

### Level 1: Scenarios (Natural Language)
- **Success (YouTube):** A YouTube URL returns platform="YouTube", content_type="video".
- **Success (Article):** A Medium article URL returns platform="Medium", content_type="article", author, and timestamp.
- **Success (LinkedIn):** A LinkedIn post URL returns platform="LinkedIn", content_type="social_post".
- **Fallback (Unknown):** An unknown domain returns platform="Web", content_type="article".
- **Edge:** A URL without HTML content still extracts platform and content_type from URL alone.

### Level 2: Given-When-Then
```
GIVEN a YouTube video URL "https://www.youtube.com/watch?v=abc123"
WHEN extract_metadata is called
THEN platform is "YouTube" AND content_type is "video"

GIVEN a Medium article URL with HTML content containing <time datetime="2024-01-15">
WHEN extract_metadata is called
THEN timestamp is parsed to datetime(2024, 1, 15)

GIVEN an unknown domain URL "https://randomblog.com/post/123"
WHEN extract_metadata is called without HTML content
THEN platform is "Web" AND content_type is "article"
```

## Implementation Notes
- Use `urlparse` for URL parsing.
- Use `dateutil.parser` for flexible timestamp parsing.
- Use BeautifulSoup (already in dependencies) for HTML metadata extraction.
- Platform detection should be case-insensitive and handle common URL variations (e.g., youtu.be short links).
- The extractor should be resilient: if HTML content is not available, it should still extract what it can from the URL alone.
