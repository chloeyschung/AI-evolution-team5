# EXT-001: Browser Extension (MVP)

**Status**: Implemented
**Priority**: Phase 2 - Ecosystem Expansion
**Note**: Phase 2 feature, not mapped to F-xxx IDs (F-000 to F-019 cover Phase 1 only)

## Overview

A lightweight Chrome/Whale browser extension that allows users to save content from the desktop with a single click. The extension integrates with the existing Briefly backend API to leverage the same ingestion, summarization, and metadata extraction capabilities.

## Requirements

### Functional Requirements

1. **One-Click Save**
   - Click extension icon to save current page
   - Context menu option "Save to Briefly" on right-click
   - Selection context menu "Save Selection to Briefly" for highlighted text

2. **Content Detection**
   - Auto-detect page URL and type (article, video, social post, etc.)
   - Extract page title automatically
   - Support for major platforms: YouTube, LinkedIn, Twitter/X, Medium, news sites, blogs

3. **User Feedback**
   - Toast notification on successful save
   - Error notification on failure (network error, duplicate, etc.)
   - Loading state during processing

4. **Authentication**
   - OAuth login flow via popup window
   - Token storage in browser localStorage
   - Auto-logout on token expiration
   - Link to mobile app for QR-based login (future)

5. **Settings**
   - Toggle auto-extract summary (on/off)
   - API endpoint configuration (for self-hosted instances)
   - Clear cached data option

### Non-Functional Requirements

1. **Performance**
   - Save action completes within 3 seconds (typical page)
   - Minimal memory footprint (<10MB)
   - No blocking of browser UI during save

2. **Compatibility**
   - Chrome 88+ (Manifest V3)
   - Whale browser (Naver)
   - Edge (Chromium-based)

3. **Security**
   - No sensitive data stored in extension
   - HTTPS-only API communication
   - Token refresh before expiration

#### 2. API Client (shared/api.ts)

```typescript
class APIClient {
  private baseUrl: string;
  private token: string | null;

  async shareContent(data: ShareData): Promise<ShareResponse> {
    const token = await this.getAccessToken();
    const response = await fetch(`${this.baseUrl}/api/v1/share`, {
      method: 'POST',
      headers: {
        'Authorization': `Bearer ${token}`,
        'Content-Type': 'application/json'
      },
      body: JSON.stringify(data)
    });
    return response.json();
  }
}
```

#### 3. Content Extractor (utils/extractor.ts)

```typescript
async function extractPageMetadata(): Promise<PageMetadata> {
  return {
    url: window.location.href,
    title: document.querySelector('title')?.textContent ||
           document.querySelector('meta[property="og:title"]')?.content ||
           document.querySelector('h1')?.textContent,
    author: document.querySelector('meta[name="author"]')?.content,
    description: document.querySelector('meta[name="description"]')?.content,
    type: detectContentType(window.location.href)
  };
}
```

```typescript
class AuthManager {
  async login(): Promise<void> {
    // Open login popup
    const { access_token, refresh_token } = await openLoginPopup();
    await this.storeTokens(access_token, refresh_token);
  }

  async getAccessToken(): Promise<string> {
    const token = await this.getStoredAccessToken();
    if (this.isTokenExpired(token)) {
      const newToken = await this.refreshToken();
      return newToken;
    }
    return token;
  }
```

### Data Flow

#### Save Content Flow

```
1. User clicks extension icon or uses context menu
   ↓
2. Content script extracts page metadata (URL, title, author)
   ↓
3. Message sent to background service worker
   ↓
4. Service worker obtains valid access token (refresh if needed)
   ↓
5. POST /api/v1/share with extracted data
   ↓
6. Backend processes: metadata extraction → summarization → storage
   ↓
7. Response with saved content details
   ↓
8. Service worker shows notification to user
```

#### Authentication Flow

```
1. User opens extension (no token found)
   ↓
2. Popup shows "Login with Google" button
   ↓
3. Click opens OAuth consent screen (new window)
   ↓
4. User grants permission, receives authorization code
   ↓
5. Extension exchanges code for tokens via /auth/google endpoint
   ↓
6. Tokens stored in localStorage with expiry tracking
   ↓
7. Extension icon updates to show logged-in state
```

## API Endpoints Used

| Endpoint | Method | Purpose |
|----------|--------|---------|
| `/auth/status` | GET | Check authentication state |
| `/auth/google` | POST | Google OAuth login |
| `/auth/logout` | POST | Logout and revoke tokens |
| `/share` | POST | Save content to Briefly |

## Security Considerations

1. **Token Storage**: Use Chrome storage API with secure flags
2. **API Communication**: HTTPS only, certificate pinning (future)
3. **Content Script Isolation**: No direct access to page DOM without user action
4. **OAuth State Parameter**: Prevent CSRF in OAuth flow
5. **Token Expiry**: Auto-logout on expired tokens
