**Status**: Implemented | **Created**: 2026-04-14 | **Author**: abraxaspark
**F-xxx Mapping**: F-020 (Browser Extension) | **Phase**: Phase 2 - Ecosystem Expansion | **Priority**: High

## 1. Overview

**Problem**: Desktop users need a quick way to save content from their browser without manually copying URLs or switching to the mobile app.

**Solution**: Chrome/Whale browser extension with one-click save, context menu integration, and seamless backend API integration for ingestion and summarization.

**Goals**: Frictionless desktop saving, reuse existing backend infrastructure, minimal extension footprint

**Non-Goals**: Offline queue, content preview in extension, keyboard shortcuts (MVP scope)

## 3. User Story / Behavior

As a Briefly user browsing on desktop, I want to save content with a single click from my browser, so that I can quickly capture articles and videos without leaving my workflow.

### Key Behaviors

- Click extension icon: save current page immediately
- Right-click page: "Save to Briefly" context menu option
- Right-click selection: "Save Selection to Briefly" (future)
- Extension extracts URL, title, author, content type
- POST to /api/v1/share endpoint with extracted data
- Backend processes: metadata extraction → summarization → storage
- Toast notification: "Saved to Briefly!" or error message
- Login required: popup with Google OAuth flow
- Token auto-refresh before expiration

## 5. API Design

| Endpoint | Method | Description |
|----------|--------|-------------|
| `/auth/status` | GET | Check authentication state |
| `/auth/google` | POST | Google OAuth login |
| `/auth/logout` | POST | Logout and revoke tokens |
| `/api/v1/share` | POST | Save content to Briefly |

### Request/Response Examples

**POST /api/v1/share**

Request:
```json
{
  "url": "https://medium.com/article/123",
  "title": "Article Title",
  "author": "John Doe"
}
```

Response (201 Created):
```json
{
  "id": 1,
  "platform": "Medium",
  "content_type": "article",
  "url": "https://medium.com/article/123",
  "title": "Article Title",
  "summary": "AI-generated summary...",
  "created_at": "2024-01-15T12:00:00Z"
}
```

## 6. Implementation

### Files

- `browser-extension/manifest.json` - Extension manifest (V3)
- `browser-extension/src/background/service-worker.ts` - Background service worker
- `browser-extension/src/content/content-script.ts` - Content script for page interaction
- `browser-extension/src/popup/popup.tsx` - Popup UI component
- `browser-extension/src/shared/api.ts` - API client
- `browser-extension/src/shared/auth.ts` - Authentication utilities
- `browser-extension/src/utils/extractor.ts` - Page metadata extraction

### Key Logic

1. **Manifest V3 Setup**:
   - permissions: storage, contextMenus, activeTab
   - host_permissions: <all_urls>
   - background: service worker
   - content_scripts: injected on all URLs
   - action: default popup

2. **Content Extraction**:
   - URL: window.location.href
   - Title: <title>, og:title meta, or h1
   - Author: article:author, og:author meta tags
   - Type: detect from URL patterns (YouTube, LinkedIn, etc.)

3. **API Communication**:
   - Axios/fetch client with auth header
   - Token refresh before API calls if expired
   - Error handling with user-friendly messages

4. **Authentication Flow**:
   - Check localStorage for tokens on load
   - If no token: show "Login with Google" in popup
   - OAuth consent → authorization code → token exchange
   - Store access_token, refresh_token, expires_at
   - Auto-refresh token 5 minutes before expiry

5. **Token Management**:
   - Store: Chrome.storage.local (encrypted)
   - Refresh: POST /auth/refresh before expiry
   - Logout: POST /auth/logout + clear storage

## 7. Edge Cases

| Scenario | Handling |
|----------|----------|
| No authentication | Show login popup, block save |
| Token expired | Auto-refresh, retry save |
| Network error | Show error toast, allow retry |
| Duplicate URL | Backend handles (upsert), return existing |
| JavaScript-heavy page | Extract from meta tags, fallback to URL |
| Extension blocked on site | Respect permissions, show message |
