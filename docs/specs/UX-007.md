# UX-007: Discard Confirmation Modal

**Status**: Implemented | **Created**: 2026-05-01 | **Author**: yoonsoo
**F-xxx Mapping**: F-UX-007 | **Phase**: Phase 1 - MVP | **Priority**: Medium

---

## 1. Overview

**Problem**: Clicking "Discard" on a content card in the web dashboard immediately deletes the item with no confirmation, making accidental deletions unrecoverable without extra steps.

**Solution**: Intercept all discard actions (button click and swipe gesture) with a confirmation modal before executing the delete.

**Goals**: Reduce accidental deletions; give users a clear moment to reconsider before moving an item to trash.

**Non-Goals**: Undo/redo system, batch discard, keyboard shortcut confirmation.

## 3. User Story / Behavior

As a user, I want a confirmation step before discarding a saved item, so that I don't accidentally lose content I meant to keep.

### Key Behaviors

- Clicking the discard button on any content card opens the modal instead of immediately deleting
- Swiping a card to discard also opens the modal instead of immediately deleting
- Clicking "Discard" in the modal proceeds with the deletion
- Clicking "Cancel" or the backdrop closes the modal and leaves the item untouched

## 6. Implementation

### Files

- `web-dashboard/src/components/ui/ConfirmModal.tsx` ŌĆö new reusable modal component
- `web-dashboard/src/components/ui/ConfirmModal.module.css` ŌĆö modal styles using design tokens
- `web-dashboard/src/pages/Dashboard.tsx` ŌĆö added `pendingDiscard` state, wired modal into discard and swipe handlers

### Key Logic

1. `handleDelete` and `handleSwipe` set `pendingDiscard` state instead of calling the API directly
2. `ConfirmModal` renders when `pendingDiscard` is non-null
3. `handleConfirmDiscard` reads `pendingDiscard`, calls the existing delete API, then clears state
4. `handleCancelDiscard` clears `pendingDiscard` with no side effects

### Dependencies

**Requires** (satisfied by):
- Existing discard/swipe logic in Dashboard.tsx

**Provides** (for future):
- `ConfirmModal` ŌĆö generic reusable modal for any confirmation flow

## 7. Edge Cases

| Scenario | Handling |
|----------|----------|
| Backdrop click | Dismisses modal, no deletion |
| Rapid double-click on discard | Modal opens once; second click is blocked by modal overlay |
| Modal open while API is in-flight | Not possible ŌĆö modal only opens before API call |
