# UX-007: 삭제 확인 모달

## 1. 개요

**문제**: 웹 대시보드에서 콘텐츠 카드의 "Discard" 버튼을 클릭하면 확인 절차 없이 즉시 삭제되어, 실수로 삭제한 경우 복구가 번거로움.

**해결**: 삭제 실행 전 확인 모달을 삽입하여 버튼 클릭과 스와이프 제스처 모두에서 사용자의 최종 확인을 받음.

**목표**: 실수로 인한 삭제 감소; 사용자에게 삭제 전 재고할 기회 제공.

**비목표**: 실행 취소/다시 실행 시스템, 일괄 삭제, 키보드 단축키 확인.

## 3. 사용자 스토리 / 동작

사용자로서, 저장된 항목을 삭제하기 전 확인 단계를 원한다. 그래야 실수로 보관하려던 콘텐츠를 잃지 않을 수 있다.

### 주요 동작

- 콘텐츠 카드의 삭제 버튼 클릭 시 즉시 삭제되지 않고 모달이 열림
- 카드 스와이프로 삭제 시도 시에도 모달이 열림
- 모달에서 "Discard" 클릭 시 삭제 진행
- "Cancel" 또는 배경 클릭 시 모달 닫히고 항목 유지

## 6. 구현

### 파일

- `web-dashboard/src/components/ui/ConfirmModal.tsx` — 신규 재사용 가능 모달 컴포넌트
- `web-dashboard/src/components/ui/ConfirmModal.module.css` — 디자인 토큰 기반 모달 스타일
- `web-dashboard/src/pages/Dashboard.tsx` — `pendingDiscard` 상태 추가, 삭제·스와이프 핸들러에 모달 연결

### 핵심 로직

1. `handleDelete`, `handleSwipe`가 API 직접 호출 대신 `pendingDiscard` 상태를 설정
2. `pendingDiscard`가 null이 아닐 때 `ConfirmModal` 렌더링
3. `handleConfirmDiscard`가 `pendingDiscard`를 읽어 기존 삭제 API 호출 후 상태 초기화
4. `handleCancelDiscard`가 부작용 없이 `pendingDiscard` 초기화

### 의존성

**필요 (충족됨)**:
- Dashboard.tsx의 기존 삭제/스와이프 로직

**제공 (향후)**:
- `ConfirmModal` — 모든 확인 흐름에 범용으로 사용 가능한 모달

## 7. 엣지 케이스

| 시나리오 | 처리 방식 |
|----------|-----------|
| 배경 클릭 | 모달 닫힘, 삭제 없음 |
| 삭제 버튼 빠른 이중 클릭 | 모달이 한 번만 열림; 두 번째 클릭은 모달 오버레이에 의해 차단 |
| API 호출 중 모달 열기 | 불가 — 모달은 API 호출 이전에만 열림 |

---

## 8. 테스트

- **단위**: ConfirmModal이 올바른 메시지로 렌더링; onConfirm/onCancel 콜백 정상 동작
- **통합**: 삭제 흐름에서 항목이 사라지기 전 모달이 표시됨
- **인수**: Cancel 클릭 시 항목 목록에 유지; Discard 클릭 시 항목 제거

---

## 9. 감각 검증

- **시각**: 반투명 배경이 대시보드를 덮음; "Move this item to trash?" 메시지, Cancel/Discard 버튼이 있는 모달 카드가 중앙에 표시
- **청각**: 콘솔 오류 없음; Discard 클릭 후 삭제 API가 정확히 한 번 호출됨
- **촉각**: 삭제 트리거 시 모달 즉시 표시; Cancel 클릭 시 즉시 닫힘

---

## 10. 향후 개선사항

1. 삭제 확인 후 실행 취소 토스트 알림
2. 다중 항목 일괄 삭제 시 단일 확인 모달
3. 키보드 단축키 (Escape로 취소, Enter로 확인)
