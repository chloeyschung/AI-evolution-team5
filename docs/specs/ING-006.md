# ING-006: 비유튜브 사이트 AI 요약 안정화

**Status**: In Progress | **Created**: 2026-06-05 | **Author**: suyoung
**F-xxx Mapping**: F-005 (AI Summary) | **Phase**: Phase 3 - Bug Fix | **Priority**: Critical

---

## 1. 문제

유튜브 외 사이트에서 AI 요약이 안정적으로 동작하지 않는다.

### 근본 원인: 3개의 버그가 맞물림

```
[공유] POST /share
    ↓ (즉시 백그라운드)
[서버] SPA 스크래핑 → 플랫폼 설명/로그인 페이지 텍스트
    ↓
[서버] LLM 요약(쓰레기) 저장 + is_ai_summarized=True  ← 여기서 잠김
    ↓ (앱 열릴 때)
[iOS] WKWebView → 실제 본문 수천 자 추출 성공
    ↓
needsSummary = false (요약이 이미 있으므로)
    ↓
rescan 미호출 → 쓰레기 요약 영구 고착
```

**버그 1** (`content.py`): 서버 사이드 스크래핑이 저품질 텍스트(짧거나 로그인/plat boilerplate)를 반환해도 그대로 요약 저장 → `is_ai_summarized=True` 고착

**버그 2** (`FetchCoordinator.swift:96`): `needsSummary = (item.summary == nil)`만 체크 → 쓰레기 요약이 이미 있으면 WKWebView 성공해도 rescan 미호출

**버그 3** (구조): iOS는 기존 요약이 좋은지 나쁜지 판단 불가 → 서버가 품질 보증을 해야 함

---

## 2. 수정 방안

### 백엔드 (`content.py` — `_background_summarize`)

저품질 텍스트 감지 함수 `_is_low_quality_text(text)` 추가:
- 300자 미만
- 로그인/인증 키워드 포함: `sign in`, `log in`, `로그인`, `create account`, `register`
- 해당 시 **요약 저장 않음**, `is_ai_summarized` = False 유지
- → iOS rescan이 정상 작동 가능해짐

### iOS (`FetchCoordinator.swift`)

rescan 조건 변경:
- 기존: `needsSummary = item.summary == nil`
- 변경: WKWebView로 ≥500자 추출 성공 시 → `force=true`로 항상 rescan
- ≥200자 & <500자: 기존대로 `needsSummary` 조건 유지
- → 좋은 본문이 있으면 기존 쓰레기 요약을 반드시 덮어씀
