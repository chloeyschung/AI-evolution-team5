**Status**: Spec

## 1. Overview

### 1.1. Problem Statement

Users lack motivation to consistently engage with their saved content. There's no visual feedback or recognition for building knowledge habits, completing milestones, or maintaining streaks. Gamification can increase daily active users and content consumption.

### 1.2. Solution

An achievement system that rewards users for:
- Maintaining daily consumption streaks
- Reaching content swipe milestones
- Exploring diverse content categories
- Engaging with content from multiple platforms

### 1.3. Goals

- Increase daily active user rate by 20%
- Improve content swipe completion rate by 15%
- Provide visual feedback for user progress
- Encourage diverse content consumption

### 1.4. Non-Goals

- Social leaderboards (Phase 4)
- Competitive elements (Phase 4)
- External rewards or prizes (Out of scope)

class AchievementDefinition(Base):

    is_active = Column(Integer, default=1)  # Can disable achievements

#### UserAchievement

```python
class UserAchievement(Base):
    """User's unlocked achievements."""

    __tablename__ = "user_achievements"

    id = Column(Integer, primary_key=True)
    user_id = Column(Integer, ForeignKey("user_profile.id"), nullable=False)
    achievement_id = Column(Integer, ForeignKey("achievement_definitions.id"), nullable=False)
    unlocked_at = Column(DateTime, default=utc_now, nullable=False)
    metadata = Column(Text)  # JSON: {"streak_days": 7, "platform_count": 5, etc.}

    __table_args__ = (
        UniqueConstraint("user_id", "achievement_id", name="unique_user_achievement"),
    )
```

class UserStreak(Base):

## 5. API Design

### 5.1. Endpoints

#### GET /achievements

```
Get all achievement definitions (optional: filter by type)
```

**Query Parameters**:
- `type`: "streak", "volume", "diversity", "curation" (optional)

**Response**: `{ achievements: [{ id, key, type, name, description, icon, trigger_value, is_unlocked, unlocked_at, progress, progress_percent }] }`

#### GET /achievements/stats

Returns user's achievement statistics:
```json
{ total_unlocked: 5, total_available: 20, completion_percent: 25, streak: { current: 7, longest: 14 } }
```

#### POST /achievements/check

Check and award newly unlocked achievements (called after swipe actions).
```json
{ new_achievements: [{ id: 5, name: "On Fire", icon: "🔥🔥", unlocked_at: "..." }] }
```

### 6.2. Streak Calculation Logic

```python
def calculate_streak(last_activity_date: date, previous_dates: List[date]) -> int:
    """Calculate current streak based on activity dates.

    Args:
        last_activity_date: Most recent activity date
        previous_dates: List of previous activity dates (not including last)

    Returns:
        Current streak count (consecutive days ending at last_activity_date)
    """
    if not previous_dates:
        return 1

    streak = 1
    current = last_activity_date

    for prev_date in sorted(previous_dates, reverse=True):
        if (current - prev_date).days == 1:
            streak += 1
            current = prev_date
        elif (current - prev_date).days > 1:
            break  # Streak broken

    return streak
```

## 7. Edge Cases

| Scenario | Handling |
|----------|----------|
| User has no swipe history | Return empty achievements, 0 progress |
| User unlocks multiple achievements at once | Award all in single transaction |
| User deletes all content | Streak resets, volume achievements remain (historical) |
| Achievement definition disabled | Don't award, but keep existing unlocks |
