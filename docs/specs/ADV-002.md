**Status**: Implemented | **Created**: 2026-04-14 | **Author**: abraxaspark
**F-xxx Mapping**: F-025 (Gamified Achievement System) | **Phase**: Phase 3 - Intelligent Features | **Priority**: Medium

## 1. Overview

**Problem**: Users lack motivation to consistently engage with saved content. No visual feedback for building habits or reaching milestones.

**Solution**: Achievement system rewarding streaks, swipe milestones, platform diversity, and curation efforts with visual badges and progress tracking.

**Goals**: +20% DAU, +15% swipe completion, visual progress feedback, habit formation

**Non-Goals**: Social leaderboards, competitive elements, external rewards (Phase 4+)

## 3. User Story / Behavior

As a Briefly user, I want to see achievements and progress for my engagement, so that I feel motivated to maintain my reading habits and reach milestones.

### Key Behaviors

- 16 achievements across 4 categories: Streak, Volume, Diversity, Curation
- Progress bars show completion percentage
- Badges unlock with visual feedback
- Streak counter updates daily
- Achievement history shows unlock dates
- Check triggered after each swipe action

### Achievements (16 total)

**Streak (5)**:
- First Steps: 1-day streak 🌱
- Building Momentum: 3-day streak 🔥
- On Fire: 7-day streak 🔥🔥
- Unstoppable: 14-day streak 🔥🔥🔥
- Legend: 30-day streak 👑

**Volume (5)**:
- Beginner: 10 swipes 📚
- Enthusiast: 50 swipes 📖
- Scholar: 100 swipes 🎓
- Expert: 500 swipes 🏆
- Master: 1000 swipes 💎

**Diversity (3)**:
- Explorer: 3 platforms 🌍
- Polyglot: 5 platforms 🌐
- Omni: 8+ platforms ✨

**Curation (3)**:
- Curator: 20 kept ⭐
- Collector: 100 kept 📦
- Archivist: 500 kept 🏛️

### Key Logic

1. **Streak Calculation**:
   - Count consecutive days ending at last_activity_date
   - Gap > 1 day breaks streak (resets to 0)
   - Update on each swipe action
   - Track longest_streak and total_active_days

2. **Achievement Check**:
   - Triggered after each swipe (POST /achievements/check)
   - Check all achievements for unlock conditions
   - Award new achievements in transaction
   - Idempotent: unique constraint prevents duplicates

3. **Progress Calculation**:
   - Streak: current_streak / trigger_value
   - Volume: total_swipes / trigger_value
   - Diversity: unique_platforms / trigger_value
   - Curation: kept_count / trigger_value

## 7. Edge Cases

| Scenario | Handling |
|----------|----------|
| No swipe history | Empty achievements, 0 progress |
| Multiple unlocks at once | Award all in transaction |
| Delete all content | Streak resets, volume remains (historical) |
| Definition disabled | Don't award new, keep existing unlocks |
| Concurrent swipe actions | Database unique constraint handles idempotency |
