# AI-001: Core 3-Line Summarizer

**Status**: Implemented | **Created**: 2026-04-14 | **Author**: abraxaspark
**F-xxx Mapping**: F-005 (AI Summary - Real-time Generation) | **Phase**: Phase 1 - MVP (Mobile Focus) | **Priority**: Critical

---

## 1. Overview

**Problem**: Users need to quickly understand the core message of saved content without reading the full text, especially for long articles.

**Solution**: Cloud-based LLM (e.g., Anthropic Claude) generating exactly three concise, high-density lines of text capturing "Who, What, and Why".

**Goals**: Radical brevity, high information density, consistent format, fast generation

**Non-Goals**: Full-text extraction, multi-language summaries, summary editing (MVP scope)

## 3. User Story / Behavior

As a Briefly user, I want to receive a concise 3-line summary of any content I ingest, so that I can quickly understand the core message without reading the full text.

### Key Behaviors

- Content ingested via share sheet or browser extension
- Raw text extracted from URL (ING-002)
- LLM generates 3-line summary with max 300 chars
- Summary stored with content for display
- API failure: retry once, then fallback or error
- Empty/short content: return as-is or minimal summary

## 6. Implementation

### Files

- `src/ai/summarizer.py` - Summarizer service with LLM integration
- `src/ai/exceptions.py` - SummarizationError
- `tests/test_ai_summarizer.py` - Test coverage

### Key Logic

1. **Input Processing**:
   - Clean input text (remove excessive whitespace, boilerplate)
   - Validate content length (raise error if empty)

2. **LLM Prompt Construction**:
   ```
   Role: Expert Summarizer
   Task: Summarize the following content into exactly {max_lines} lines
   Constraint: Maximum {max_chars} characters total, high density, no fluff
   Constraint: No trailing ellipsis or summarization markers
   Content: {raw_content}
   ```

3. **API Call**:
   - Use httpx for asynchronous API calls
   - Call Cloud LLM API (e.g., Anthropic Claude)
   - Implement retry mechanism for transient errors

4. **Response Validation**:
   - Parse response to ensure line count and character constraints
   - If exceeds max_chars: truncate to 300-char boundary (word-safe)
   - Ensure no trailing ellipsis added

5. **Error Handling**:
   - API 500 error → retry once
   - Still fails → raise SummarizationError
   - Fallback: use OG description if available

## 7. Edge Cases

| Scenario | Handling |
|----------|----------|
| LLM API unreachable | Retry once, raise SummarizationError |
| Response exceeds 300 chars | Truncate to 300-char word boundary |
| Input content very short | Return as-is or minimal summary |
| Input content empty | Raise SummarizationError |
| API returns invalid format | Parse error, retry or fallback |
