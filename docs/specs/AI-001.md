# AI-001: Core 3-Line Summarizer

As a Briefly user, I want to receive a concise 3-line summary of any content I ingest, so that I can quickly understand the core message without reading the full text.

## Implementation Contract

### Level 1: Natural Language
The `Summarizer` service takes raw text (or extracted content from a URL) as input and uses a cloud-based LLM (e.g., Anthropic Claude) to generate exactly three concise, high-density lines of text. 

**Constraints:**
- Maximum 300 characters total (including newlines)
- Exactly 3 lines separated by newlines
- High-density content preserving "Who, What, and Why"
- No trailing ellipsis or summarization markers

The output must capture the "Who, What, and Why" of the content within the character limit.

### Level 2: Logic Flow (Pseudocode)
```
INPUT: raw_content (string), max_lines (int, default=3), max_chars (int, default=300)
PROCESS:
  1. Clean input text (remove excessive whitespace, boilerplate).
  2. Construct prompt for LLM:
     - Role: Expert Summarizer.
     - Task: Summarize the following content into exactly {max_lines} lines.
     - Constraint: Maximum {max_chars} characters total, high density, no fluff.
     - Constraint: No trailing ellipsis or summarization markers.
  3. Call Cloud LLM API with prompt and raw_content.
  4. Parse response to ensure it adheres to both line count AND character constraints.
  5. If response exceeds max_chars: truncate to 300-char boundary (word-safe).
  6. Handle API errors (retry logic or fallback to a simpler model).
OUTPUT: summary_text (string)
ERROR: SummarizationError (if API fails or content is unprocessable)
```

## Verification Contract

### Level 1: Scenarios (Natural Language)
- **Success:** A long news article is condensed into three meaningful, distinct lines.
- **Success:** Summary is within 300 characters total (including newlines).
- **Failure:** The LLM API is unreachable; the service raises a `SummarizationError`.
- **Edge:** The input content is extremely short (e.g., a single sentence); the service returns the sentence or a slightly expanded version without exceeding 3 lines or 300 chars.
- **Edge:** The input content is empty or nonsimulated; the service raises an error.

### Level 2: Given-When-Then
```
GIVEN a long article about the history of space exploration
WHEN the summarize method is called
THEN the output is a string with exactly 3 lines of high-density text
AND the total character count is <= 300

GIVEN the LLM API returns a 500 error
WHEN the summarize method is called
THEN a SummarizationError is raised

GIVEN content that would generate a summary exceeding 300 characters
WHEN the summarize method is called
THEN the output is truncated to exactly 300 characters at a word boundary
AND no trailing ellipsis is added
```
