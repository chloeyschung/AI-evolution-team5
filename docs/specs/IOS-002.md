## 1. Overview

**Problem**: IOS-001에서 이메일/비밀번호 로그인만 지원. Google 계정으로 간편 로그인이 불가능.

**Solution**: `GoogleSignIn-iOS` SDK를 사용해 iOS 기본 Google OAuth 플로우를 구현하고,
`POST /api/v1/auth/google` 엔드포인트로 ID 토큰을 검증하여 Briefly JWT를 발급받는다.

**Goals**:
- Google 계정으로 1탭 로그인 (기존 이메일 로그인과 병행)
- 신규/기존 사용자 모두 처리 (백엔드에서 자동 분기)
- `AuthTokenStore`를 그대로 재사용 (IOS-001 토큰 저장소 공유)

**Non-Goals**:
- Sign in with Apple (별도 티켓)
- Google 로그인 후 기기 동기화 읽기 (IOS-003)
- Share Extension에서 Google 재인증 불필요 (토큰은 App Group 공유)

## 3. User Story / Behavior

**Google 로그인 흐름:**
1. Account 탭 → "Google로 로그인" 버튼 탭
2. iOS Google OAuth 시트 표시 (GIDSignIn.sharedInstance.signIn)
3. 사용자가 Google 계정 선택
4. `GIDGoogleUser.idToken.tokenString` 획득
5. `POST /api/v1/auth/google` 호출 (id_token + user_info)
6. 성공: access_token, refresh_token을 App Group UserDefaults에 저장
7. Account 탭에 로그인 상태(이메일) 표시

**취소/실패 시:**
- `GIDSignInError.canceled` → 조용히 무시 (에러 얼럿 없음)
- 네트워크 오류 → "Google 로그인 실패: ..." 에러 얼럿

## 4. API Design

### 사용 엔드포인트

| Endpoint | Method | 설명 |
|----------|--------|------|
| `POST /api/v1/auth/google` | POST | Google ID 토큰 검증 후 Briefly JWT 발급 |

### Request / Response

**POST /api/v1/auth/google**
```json
// Request
{
  "google_id_token": "eyJ...",
  "google_user_info": {
    "id": "1234567890",
    "email": "user@gmail.com",
    "name": "홍길동",
    "picture": "https://lh3.googleusercontent.com/..."
  }
}

// Response 200
{
  "access_token": "eyJ...",
  "refresh_token": "abc...",
  "expires_at": "2026-05-01T10:00:00Z",
  "user": {
    "id": 42,
    "email": "user@gmail.com",
    "display_name": "홍길동",
    "avatar_url": "https://lh3.googleusercontent.com/...",
    ...
  },
  "is_new_user": true
}
```

## 5. Implementation

### Files

- `Briefly/Services/BrieflyAPI.swift` — `loginWithGoogle()` 메서드 + 타입 추가
- `Briefly/Views/AccountView.swift` — Google 로그인 버튼 + `signInWithGoogle()` ViewModel 메서드
- `Briefly/BrieflyApp.swift` — `onOpenURL` → `GIDSignIn.sharedInstance.handle(url)`
- `Briefly.xcodeproj/project.pbxproj` — GoogleSignIn-iOS SPM 패키지 추가
- `Briefly/GoogleService-Info.plist` — Google Cloud Console에서 발급 (수동 추가 필요)

### Xcode 수동 설정 (필수)

#### 1. GoogleSignIn-iOS SPM 추가 (project.pbxproj에 자동 추가됨)
```
File > Add Package Dependencies
URL: https://github.com/google/GoogleSignIn-iOS
Version: Up to Next Major (8.0.0)
Target: Briefly (메인 앱만, Share Extension 제외)
```

#### 2. GoogleService-Info.plist 추가
- [Google Cloud Console](https://console.cloud.google.com) → 프로젝트 선택
- APIs & Services > Credentials > iOS 클라이언트 ID 생성
  - Bundle ID: `com.briefly.app`
- 생성된 `GoogleService-Info.plist` 다운로드
- Xcode에서 `Briefly` 타겟에 추가 (Target Membership: Briefly만)

#### 3. URL 스킴 추가 (Info.plist)
`GoogleService-Info.plist`의 `REVERSED_CLIENT_ID` 값을 URL 스킴으로 추가:
- Xcode → Briefly 타겟 → Info → URL Types
- URL Schemes에 `REVERSED_CLIENT_ID` 값 추가 (예: `com.googleusercontent.apps.XXXX`)

### Key Logic

1. `BrieflyApp.init()`: `GoogleService-Info.plist`에서 CLIENT_ID 읽어 `GIDSignIn` 설정
2. `signInWithGoogle()`: `GIDSignIn.sharedInstance.signIn(withPresenting:)` 호출
3. 성공 콜백: `user.idToken.tokenString` + `user.profile` 정보로 `BrieflyAPI.loginWithGoogle()` 호출
4. 응답 처리: `AuthTokenStore.shared.save()` → IOS-001과 동일한 토큰 저장소

### Dependencies

**Requires**:
- `IOS-001` — AuthTokenStore, BrieflyAPI 기반 구조
- `AUTH-002` — `POST /api/v1/auth/google` 백엔드 엔드포인트

**Provides**:
- `IOS-003` — 토큰 발급 후 다기기 읽기 동기화 (향후)

## 6. Edge Cases

| Scenario | Handling |
|----------|----------|
| 사용자가 Google 시트 취소 | GIDSignInError.canceled → 에러 얼럿 없이 무시 |
| 이미 Google 로그인 상태 | 기존 Briefly 토큰 유지 (재로그인 불필요) |
| 서버 403 (30일 재가입 차단) | "계정 삭제 후 30일 내 재가입 불가" 에러 메시지 |
| GoogleService-Info.plist 없음 | 런타임 경고 출력, Google 버튼 비활성화 |
| 네트워크 없음 | BrieflyAPI.APIError → 에러 얼럿 표시 |
