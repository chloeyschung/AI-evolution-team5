# IOS-007: 앱 저장 콘텐츠 Auto-Tag 연동 (iOS Auto-Tag Integration)

**Status**: Draft | **Created**: 2026-05-29 | **Author**: suyoung
**F-xxx Mapping**: iOS Auto-Tag Pipeline | **Phase**: Phase 4 - UX | **Priority**: Medium

---

## 1. Overview

**Problem**: IOS-006의 홈 탭 주제별 섹션은 서버의 `auto_tag_category` 데이터에 의존한다. 그러나 Share Extension으로 저장된 콘텐츠가 서버에서 자동 태깅(`apply-auto-tag` 파이프라인)이 완료된 후, iOS 앱이 그 결과를 반영하는 흐름이 없다. 또한 `SavedItem` 로컬 모델에 auto-tag 필드가 없어 홈 탭 주제별 섹션에서 로컬 아이템이 누락된다.

**Solution**: 
1. Share Extension → `POST /api/v1/share` 후 서버가 백그라운드로 auto-tag를 실행한다 (`apply-auto-tag` 브랜치 파이프라인, 서버 측 이미 구현).
2. iOS 앱은 포그라운드 진입 시 `GET /api/v1/content`로 서버의 최신 auto-tag 결과를 fetch하여 `HomeViewModel`에 반영한다 (IOS-006에서 이미 fetch 흐름 존재).
3. `SavedItem` 로컬 모델에 `autoTagCategory`, `autoTagKeywordsEn`, `autoTagKeywordsOriginal` 필드를 추가하고, 서버 fetch 결과를 로컬에 병합·저장하여 오프라인에서도 주제별 분류가 유지된다.

**Goals**:
- Share Extension으로 저장한 콘텐츠가 서버 auto-tag 완료 후 홈 탭 주제별 섹션에 정상 노출된다
- 앱 재시작 후 오프라인 상태에서도 마지막으로 fetch한 auto-tag 데이터가 유지된다
- `SavedItem`에 auto-tag 필드가 추가되어 로컬 아이템도 주제별 섹션에 포함될 수 있다

**Non-Goals**:
- iOS 기기에서 직접 LLM API를 호출하는 온디바이스 auto-tag — 서버 의존 방식 유지
- 실시간 태깅 결과 푸시(Push Notification) 수신 — 향후 개선
- auto-tag 결과를 사용자가 직접 수정하는 UI

## 3. User Story / Behavior

**As a** Briefly 앱 사용자,
**I want to** Share Extension으로 저장한 링크가 서버에서 자동 분류된 후 홈 탭 주제별 섹션에 나타나길 원한다,
**so that** 앱으로 저장한 콘텐츠도 웹과 동일하게 주제별로 묶어볼 수 있다.

### Key Behaviors

```
[Share Extension으로 링크 저장]
  → POST /api/v1/share → 서버에 콘텐츠 생성
  → 서버 백그라운드: auto_tagger.tag() 실행 → DB에 auto_tag_* 필드 저장
     (apply-auto-tag 파이프라인, 수 초~수십 초 소요)

[앱 포그라운드 진입 (IOS-006 HomeViewModel.load())]
  → GET /api/v1/content fetch (기존 IOS-006 흐름)
  → 응답의 각 ServerContent를 로컬 SavedItem과 serverContentId로 매칭
  → auto_tag_category, auto_tag_keywords_* 필드를 SavedItem에 병합
  → StorageService.updateItem() 로 로컬 저장
  → HomeViewModel 주제별 섹션에 해당 항목 반영

[오프라인 앱 재시작]
  → StorageService에서 로컬 SavedItem 로드
  → autoTagCategory 필드가 있으면 주제별 섹션에 포함
  → 서버 fetch 없이도 마지막 auto-tag 결과 표시
```

    case .local(let item): return item.autoTagCategory
    case .server(let content): return content.autoTagCategory
    }

### Files 변경 목록

| `Briefly/ViewModels/HomeViewModel.swift` | 서버 fetch 후 `mergeAutoTagFromServer()` 호출 |
| `Briefly/Models/HomeItem.swift` (IOS-006) | `autoTagCategory` 로직 업데이트 |

### `mergeAutoTagFromServer` 핵심 로직

```swift
func mergeAutoTagFromServer(_ serverItems: [ServerContent]) {
    var items = loadAll()
    var updated = false
    for serverItem in serverItems {
        guard let autoTagCategory = serverItem.autoTagCategory,
              let idx = items.firstIndex(where: { $0.serverContentId == serverItem.id }) else { continue }
        items[idx].autoTagCategory = autoTagCategory
        items[idx].autoTagKeywordsEn = serverItem.autoTagKeywordsEn
        items[idx].autoTagKeywordsOriginal = serverItem.autoTagKeywordsOriginal
        updated = true
    }
    if updated { saveAll(items) }
}
```

## 7. Edge Cases

| Scenario | Handling |
|----------|----------|
| `apply-auto-tag` 미머지 (서버가 nil 반환) | `autoTagCategory == nil`이면 로컬 필드 덮어쓰지 않음 |
| `serverContentId` 없는 로컬 아이템 (비로그인 저장) | 매칭 불가 → 주제별 섹션 제외 (IOS-005 sync 후 재시도) |
| auto-tag가 아직 처리 중인 콘텐츠 | 서버 `auto_tag_status == "pending"` → category nil → 주제별 섹션 제외 |
| 기존 `SavedItem` JSON (신규 필드 없음) | optional + 기본값으로 디코딩 성공 |
| 서버 fetch 실패 | 기존 로컬 auto-tag 데이터 유지 |
