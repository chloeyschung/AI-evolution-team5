# IOS-009: iOS 앱 내 용어 통일 (Terminology Unification)

**Status**: Draft | **Created**: 2026-06-04 | **Author**: suyoung
**F-xxx Mapping**: i18n Prep / UX Consistency | **Phase**: Phase 4 - UX | **Priority**: Medium

---

## 1. Overview

**Problem**: iOS · 웹 · 브라우저 확장 · Share Extension이 같은 개념에 서로 다른 용어를 쓰고 있다. i18n 작업 전에 단일 표준을 확립하지 않으면 번역 키가 분산되어 유지보수 비용이 늘어난다.

**Solution**: 팀이 합의한 7개 표준 용어를 iOS 앱 전체에 일괄 적용한다. 코드 내부 식별자(함수명·enum 케이스)는 최소한으로만 건드리고, 사용자에게 노출되는 UI 문자열과 주석을 우선 정리한다.

**Goals**:
- 합의된 7개 표준 용어를 iOS 전체에 적용
- 현재 혼용 중인 `Delete / Deleted` → `Discard / Discarded` 로 통일
- 저장된 항목 탭 `Archived` → `Saved` 로 통일
- 원문 보기 액션 `원문 바로가기` → `원문 보기 / Open original` 로 통일
- `AI Summary` 영문 표기 통일 (한국어 `AI 요약`은 유지)
- 향후 i18n 작업 시 iOS 키가 웹 키와 1:1 매핑 가능한 상태로 준비

**Non-Goals**:
- 실제 i18n(Localizable.strings) 파일 생성 — 후속 스펙으로 분리
- `Dive Deeper` UI 구현 — 미구현 상태이므로 구현 시 IOS-010으로 분리
- 백엔드 API 파라미터 변경 — API는 이미 `action: "discard"` 사용 중, 변경 불필요

## 5. 변경하지 않는 것 (의도적 유지)

| 항목 | 이유 |
|------|------|
| `SavedItem.Status.deleted` enum 케이스명 | UserDefaults JSON 직렬화 키 — 변경 시 기존 데이터 손실 |
| `SavedItem.Status.kept` enum 케이스명 | 동일한 직렬화 이유 |
| `LibraryFilter.archived` enum 케이스명 | 내부 식별자, rawValue(표시 문자열)만 변경 |
| `performDelete()` 함수명 | 내부 식별자, 리팩토링 범위 아님 |
| `SwipeAction.discard` | 이미 올바른 용어 사용 중 ✅ |
| `Label("AI 요약", ...)` 한국어 표기 | 합의된 표준 한국어 — 변경 불필요 ✅ |
| `.discarded` Status 케이스 (기존) | Saved 탭에 쓰이는 별도 케이스, 이번 범위 외 |
| 로그 메시지 `[Delete]` | 개발자용 내부 로그, i18n 범위 아님 |

## 6. 현재 Status enum 혼용 현황 (참고)

`SavedItem.Status`에 `.discarded`와 `.deleted`가 공존하고 있다.

```swift
enum Status: String, Codable {
    case unread, read, discarded   // ← Archived 탭에 포함됨
    case kept                      // ← Archived 탭에 포함됨
    case deleted                   // ← Discarded 탭에 표시됨 (이번 스펙 후)
}
```

- `.discarded`: 현재 Archived 탭에 노출 (`LibraryFilter.archived` 필터 조건 포함) → 이번 스펙 후 탭 이름이 "Saved"로 바뀜
- `.deleted`: 현재 Deleted 탭에 노출 → 이번 스펙 후 탭 이름이 "Discarded"로 바뀜

이 두 케이스의 의미 중복과 정리는 별도 스펙에서 다룬다.

## 7. Testing

### 수동 검증 시나리오

1. **Discard 버튼**
   - Inbox에서 링크 카드 진입 → 하단 버튼이 `Discard / Skip Now / Keep` 으로 표시되는지 확인

2. **스와이프 배지**
   - 카드를 왼쪽으로 스와이프 → `DISCARD` 배지가 표시되는지 확인
   - 카드를 오른쪽으로 스와이프 → `KEEP` 배지 (변경 없음) 확인

3. **Saved 탭**
   - Library 탭 → 필터 탭 레이블이 `Inbox / Saved / Discarded` 로 표시되는지 확인
   - Saved 탭 선택 → navigationTitle이 `Saved` 인지 확인
   - Saved 탭 빈 상태 → `"Saved가 비어있어요"` 확인

4. **Discarded 탭**
   - Discarded 탭 선택 → navigationTitle이 `Discarded` 인지 확인
   - 항목 없는 상태 → `"Discarded가 비어있어요"` / `"Inbox에서 Discard한 링크가 여기에 표시됩니다"` 확인

5. **원문 보기 버튼**
   - 카드 상세 화면 진입 → 원문 보기 버튼 레이블이 `"원문 보기"` 로 표시되는지 확인

## 8. Acceptance Criteria

- [ ] ItemDetailView 하단 버튼 레이블이 `Discard`로 표시된다
- [ ] 왼쪽 스와이프 배지가 `DISCARD`로 표시된다
- [ ] Library 탭 필터 레이블이 `Inbox / Saved / Discarded` 로 표시된다
- [ ] Saved 탭 navigationTitle이 `Saved`로 표시된다
- [ ] Discarded 탭 navigationTitle이 `Discarded`로 표시된다
- [ ] Saved 탭 빈 상태 문구가 `"Saved가 비어있어요"`로 표시된다
- [ ] Discarded 탭 빈 상태 문구에 `Discard` 용어가 사용된다
- [ ] 원문 보기 버튼 레이블이 `"원문 보기"`로 표시된다
- [ ] Keep / Skip Now 레이블은 변경되지 않는다
- [ ] 기존 Keep 항목이 Saved 탭에 정상 노출된다 (데이터 유실 없음)
- [ ] 기존 Discard(Delete) 항목이 Discarded 탭에 정상 노출된다 (데이터 유실 없음)
