## 1. Overview

**Problem**: Users save content but don't consistently consume it. Without reminders, content accumulates unread and engagement drops.

**Solution**: Smart reminder system tracking consumption patterns, sending timely nudges, and learning from user responses.

**Goals**: +25% daily consumption, reduce backlog, personalized non-intrusive reminders

**Non-Goals**: External push notifications (Phase 3), email reminders (Phase 2), social reminders (Phase 4)

## 3. User Story / Behavior

As a Briefly user, I want to receive smart reminders to consume my saved content, so that I maintain my reading habit and don't let content accumulate unread.

### Key Behaviors

- 4 reminder types: Backlog, Streak, Time-Based, Re-engagement
- Priority order: streak > backlog > time_based > reengagement
- Configurable preferences: enabled, preferred time, frequency, quiet hours
- Activity pattern learning: most active hour/day, avg daily swipes
- Action tracking: logged when user acts on or dismisses reminder
- Default: 6 PM daily if no history

## 4. Data Models

### New Tables

**reminder_preferences**: id, user_id (unique), is_enabled, preferred_time, frequency ('daily'/'weekly'/'never'), quiet_hours_start/end, backlog_threshold (default 10), updated_at

**reminder_logs**: id, user_id, reminder_type, message, sent_at, action_taken (bool), action_taken_at, dismissed_at

**user_activity_patterns**: id, user_id (unique), most_active_hour (0-23), most_active_day (0-6, Mon=0), avg_daily_swipes, avg_session_duration (min), last_updated

### Existing Used

- `SwipeHistory` (activity pattern learning)
- `Content` (unread count for backlog)
- `UserStreak` (streak status for streak reminders)

## 6. Implementation

### Files

- `src/ai/reminder_engine.py` - Generate & manage reminders
- `src/data/reminder_preference_repository.py` - Preferences CRUD
- `src/data/reminder_log_repository.py` - Reminder logging
- `src/data/activity_pattern_repository.py` - Pattern learning
- `src/constants.py` - REMINDER_BACKLOG_THRESHOLD, STREAK_CHECK_DAYS
- `src/utils/datetime_utils.py` - is_quiet_hours(), utc_now()

### Reminder Types (4 total)

| Type | Trigger | Example Message |
|------|---------|-----------------|
| **Backlog** | Unread > threshold | "You have 15 unread items. Ready to catch up?" |
| **Streak** | No activity today + streak at risk | "Don't break your 7-day streak! 8 items waiting." |
| **Time-Based** | Preferred consumption time | "Time for your daily knowledge boost! 5 new items..." |
| **Re-engagement** | Inactive N days | "We miss you! 23 items waiting in Briefly." |

### Key Logic

1. **Reminder Generation Flow**:
   - Check if reminders enabled (preferences.is_enabled)
   - Check quiet hours (is_quiet_hours())
   - Check frequency limits (max 1/day default)
   - Generate reminder by priority order (streak > backlog > time_based > reengagement)

2. **Activity Pattern Learning**:
   - Analyze SwipeHistory for most_active_hour (0-23)
   - Determine most_active_day (0-6, Mon=0)
   - Calculate avg_daily_swipes and avg_session_duration
   - Update patterns periodically (not on every swipe)

3. **Frequency Control**:
   - Check last reminder sent time
   - Respect frequency setting (daily/weekly/never)
   - Log all reminders sent (reminder_logs)

4. **Action Tracking**:
   - Log when user acts on reminder (action_taken = true)
   - Log when user dismisses (dismissed_at)
   - Use for future optimization (learn preferences)

**Provides** (for future):

## 7. Edge Cases

| Scenario | Handling |
|----------|----------|
| No swipe history | Default preferences (6 PM daily) |
| Reminders disabled | Return has_reminder: false |
| In quiet hours | Suppress all reminders |
| Just received reminder | Respect frequency limit |
| No unread content | Skip backlog reminder |
| Streak not at risk | Skip streak reminder |
