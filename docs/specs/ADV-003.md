# ADV-003: Smart Reminders

**Status**: Spec | **Created**: 2026-04-14 | **Author**: abraxaspark

---

## 1. Overview

**Problem**: Users save content but don't consistently consume it. Without reminders, content accumulates unread.

**Solution**: Smart reminder system tracking consumption patterns, sending timely nudges, learning from responses.

**Goals**: +25% daily consumption, reduce backlog, personalized non-intrusive reminders

**Non-Goals**: External push notifications (Phase 3), email reminders (Phase 2), social reminders (Phase 4)

## 3. Reminder Types (4 total)

| Type | Trigger | Example Message |
|------|---------|-----------------|
| **Backlog** | Unread > threshold | "You have 15 unread items. Ready to catch up?" |
| **Streak** | No activity today + streak at risk | "Don't break your 7-day streak! 8 items waiting." |
| **Time-Based** | Preferred consumption time | "Time for your daily knowledge boost! 5 new items..." |
| **Re-engagement** | Inactive N days | "We miss you! 23 items waiting in Briefly." |

**Priority Order**: streak > backlog > time_based > reengagement

---

## 4. Data Models

### New Tables

**reminder_preferences**: id, user_id (unique), is_enabled, preferred_time, frequency ('daily'/'weekly'/'never'), quiet_hours_start/end, backlog_threshold (default 10), updated_at

**reminder_logs**: id, user_id, reminder_type, message, sent_at, action_taken (bool), action_taken_at, dismissed_at

**user_activity_patterns**: id, user_id (unique), most_active_hour (0-23), most_active_day (0-6, Mon=0), avg_daily_swipes, avg_session_duration (min), last_updated

### Existing Used
- `SwipeHistory` (activity patterns)
- `Content` (unread count)
- `UserStreak` (streak status)

## 5. API

| Endpoint | Description |
|----------|-------------|
| `GET /reminders/preferences` | Get preferences |
| `PUT /reminders/preferences` | Update preferences |
| `GET /reminders/suggest` | Get current suggestion |
| `POST /reminders/respond` | Log action (acted/dismissed) |

**Suggestion Response**:
```json
{ has_reminder: true, reminder_type: "backlog", message: "...", priority: "high", metadata: { unread_count: 15, streak_days: 7 } }
```

## 7. Edge Cases

| Scenario | Handling |
|----------|----------|
| No swipe history | Default preferences (6 PM daily) |
| Reminders disabled | Return None |
| In quiet hours | Suppress all |
| Just received reminder | Respect frequency limit |
| No unread content | Skip backlog reminder |
