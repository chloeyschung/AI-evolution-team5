# UX-001: Swipe Card Stack

**Status**: Implemented | **Created**: 2026-04-14 | **Author**: abraxaspark
**F-xxx Mapping**: F-008 (Content Card Components), F-009 (Swipe View) | **Phase**: Phase 1 - MVP (Mobile Focus) | **Priority**: Critical

---

## 1. Overview

**Problem**: Users need an intuitive, low-friction interface to consume saved content without the cognitive overhead of traditional list-based or folder-based organization.

**Solution**: Tinder-like card stack presenting one content item at a time with 3-line AI summary and metadata, supporting quick swipe decisions.

**Goals**: Radical brevity, guilt-free experience, fast content triage, muscle memory development

**Non-Goals**: Multi-card preview, card customization, swipe direction configuration

## 3. User Story / Behavior

As a Briefly user, I want to consume my saved content through an intuitive swipe-based card interface, so that I can quickly decide what to keep or discard without cognitive overhead.

### Key Behaviors

- Card stack displays one content item at a time
- Top card shows 3-line summary, platform icon, content type badge, timestamp
- Right swipe: card animates right, commits "keep", next card revealed
- Left swipe: card animates left, commits "discard", next card revealed
- Swipe can be cancelled mid-gesture (card returns to center)
- Empty state: friendly "All caught up!" message when no pending content
- Network error: shows retry option

## 6. Implementation

### Files

- `src/ui/card_stack.dart` - Main swipe card stack component
- `src/ui/card_renderer.dart` - Individual card rendering with summary and metadata
- `src/ui/swipe_handler.dart` - Gesture detection and swipe logic
- `src/data/repository.py` - ContentRepository.get_pending()

### Key Logic

1. **Content Fetching**:
   - Query Content joined with SwipeHistory
   - Filter: content with no SwipeHistory record (pending)
   - Order by created_at DESC (newest first)
   - Paginate with limit/offset for large libraries

2. **Card Rendering**:
   - Summary: 3 lines max, ellipsis overflow
   - Platform: icon + text badge
   - Content type: small badge (article/video/social_post/image)
   - Metadata: author and timestamp if available, hidden if not

3. **Swipe Gesture**:
   - Use react-native-gesture-handler or GestureDetector
   - Track dx for horizontal movement
   - Apply visual transform: rotation (max 15°), opacity overlay (green/red tint)
   - Commit threshold: ~1/3 screen width
   - Cancel: spring animation back to center

4. **Animation**:
   - On commit: animate card off-screen in swipe direction
   - Duration: 300ms with ease-out curve
   - Next card slides in from right with 50ms stagger

5. **Empty State**:
   - Friendly illustration (celebration, not emptiness)
   - Message: "All caught up!" or "Nothing to consume"
   - Optional: "Check back later" or refresh button

## 7. Edge Cases

| Scenario | Handling |
|----------|----------|
| No pending content | Show empty state with friendly message |
| Network error | Show error state with retry option |
| Swipe cancelled (< threshold) | Card springs back to center |
| Summary too long | Truncate to 3 lines with ellipsis |
| Missing metadata | Hide author/timestamp fields |
| Large content library | Paginate with limit/offset |
