# UX-001: Swipe Card Stack

## User Story
As a Briefly user, I want to consume my saved content through an intuitive swipe-based card interface, so that I can quickly decide what to keep or discard without cognitive overhead.

## Dependencies
- **depends_on:** `@specs/AI-001.md` (Summarizer for 3-line summary), `@specs/AI-002.md` (Metadata for display)
- **depended_by:** `@specs/UX-002.md`, `@specs/UX-003.md`
- **dependency code:** `src/ai/summarizer.py`, `src/ai/metadata_extractor.py`

## Implementation Contract

### Level 1: Natural Language
Swipe Card Stack is the primary content consumption interface. It presents ingested content as a stack of cards, where each card displays a concise summary and metadata. Users can swipe left to discard or right to keep content, following the "Radical Brevity" and "Guilt-Free Experience" principles.

Each card displays:
- 3-line AI-generated summary (from AI-001)
- Source platform and content type (from AI-002)
- Timestamp and author (if available)

The component must:
- Fetch content from the storage engine (DAT-001)
- Display one card at a time with smooth animations
- Support swipe gestures (left/right) for quick decisions
- Show visual feedback during swipe interactions
- Handle empty state when no content is available

### Level 2: Logic Flow (Pseudocode)
```
INPUT: Content list from DAT-001 storage
PROCESS:
  1. Fetch pending content (not yet swiped) ordered by recency.
  2. If no content: display empty state ("Nothing to consume").
  3. If content available: render top card with summary + metadata.
  4. On swipe gesture:
     a. Track swipe direction and velocity.
     b. Apply visual transform (rotation, opacity) based on progress.
     c. If swipe threshold exceeded: commit action (keep/discard).
     d. Animate card exit and reveal next card.
  5. Update storage with swipe action (via UX-002).
  6. Repeat from step 2.
OUTPUT: Rendered card stack with swipe interactions
ERROR: NetworkError (fetch failure), EmptyState (no content)
```

## Verification Contract

### Level 1: Scenarios (Natural Language)
- **Success:** Content cards are displayed with 3-line summary and metadata.
- **Success:** Right swipe triggers "Keep" action and reveals next card.
- **Success:** Left swipe triggers "Discard" action and reveals next card.
- **Success:** Empty state is shown when all content has been consumed.
- **Edge:** Swipe gesture can be cancelled mid-swipe (card returns to center).
- **Edge:** Network error shows retry option instead of crashing.

### Level 2: Given-Then-When
```
GIVEN a list of content with summaries and metadata
WHEN the card stack is rendered
THEN the most recent content is displayed as the top card

GIVEN a visible card with summary and metadata
WHEN user swipes right beyond threshold
THEN the card animates out to the right AND the next card is revealed

GIVEN a visible card with summary and metadata
WHEN user swipes left beyond threshold
THEN the card animates out to the left AND the next card is revealed

GIVEN an empty content list
WHEN the card stack is rendered
THEN the empty state message is displayed

GIVEN a swipe gesture that does not exceed threshold
WHEN user releases the gesture
THEN the card returns to center position (cancelled swipe)
```

## Implementation Notes
- This is a React Native component (per MPS tech stack).
- Use `react-native-gesture-handler` for swipe gestures.
- Use `react-native-reanimated` for smooth animations.
- Card layout should respect "Radical Brevity": summary is 3 lines max, metadata is compact.
- Empty state should be guilt-free: celebrate completion, not show emptiness.
- Fetch content in batches to handle large libraries efficiently.
